## 1. 游戏概述

| 项目 | 说明 |
|------|------|
| **中文名** | 命运的织线 |
| **视角** | 第三人称 |
| **舞台** | 归潮镇 — 平凡的海岛小镇 |
| **玩家身份** | 守护小镇的土地公 |
| **类型定位** | 神明模拟 + 关系/命运干预 + AI 原生涌现叙事 |
| **核心命题** | 命运不是被选择的，而是被编织出来的 |

### 1.1 世界观简述

人们看不见命运，却被无数因果与选择牵引向不同人生。玩家作为土地公：

- **不能**直接控制任何人的意志
- **不能**改写已经发生的过去
- **只能**在命运的缝隙中轻轻拨动一根织线 —— 让偶遇更容易发生、让一封信迟到几分钟、让一句鼓起勇气的话被说出口，或让本该发生的契机悄然错过

游戏采用 **AI 原生叙事** 与 **动态因果模拟**：每位居民拥有独立的情绪、记忆、目标与人际关系，并随经历成长变化。玩家不选固定剧情分支，而是通过影响人物、事件、信息与机缘，让世界自行演化。

### 1.2 道德光谱

| 路线 | 行为倾向 | 长期走向 |
|------|----------|----------|
| **守护** | 助人告白、家人重逢、化险为夷 | 正神 / 阳德 |
| **腐化** | 放大嫉妒贪婪恐惧、编织误会冲突 | 邪神 / 阴德 |
| **中立挣扎** | 用尽香火与神力换小镇希望 | 介于天界与冥界之间 |

没有绝对正确的选项，也没有预先写好的单一结局；微小推动可能在未来形成连锁反应。

## 2. 核心体验与卖点

### 2.1 玩家幻想（Fantasy）

「我是看不见却掌控局势的神明」—— 在地图上俯瞰众生，用有限神力撬动人际关系与人生节点。

### 2.2 差异化

1. **干预而非操控**：机制上禁止夺舍式控制，强化策略与道德选择
2. **涌现叙事**：约会、争吵、匿名信、意外事故均可成为关键节点
3. **可视化命运网络**：缘分线 + 命运线双图层，观察—决策—反馈闭环清晰

### 2.3 自述好玩点

1. **核心循环及时反馈**：观察 → 干涉 → 演变 → 升级，正反馈足
2. **故事自由性**：多人物、多结局，勾引好奇心

## 4. 时间结构与一局规模

| 单位 | 长度 | 备注 |
|------|------|------|
| **一天** | 早晨、下午、夜晚（3 时段） | 每时段可观察 + 有限干预 |
| **一周** | 7 天 | 周结算：阴德、阳德 |
| **一局** | 60 游戏日 | 固定绩效期 |

**设计含义**：玩家在每个时段做观察与干预决策；日终/时段末由 AI 模拟 NPC 行为并结算事件。

## 5. 游戏目标与结局

### 5.1 三层目标

| 层级 | 目标 | 说明 |
|------|------|------|
| **表层** | 香火线 | 60 天后香火 ≥ 100，维持神位 |
| **中层** | 人物结局 | 不同 NPC、不同事件的独立结局 |
| **深层** | Meta 领悟 | 解开小镇命运；「人在做天在看，三分天注定七分靠打拼」 |

### 5.2 60天结局判定

**判定优先级**：
1. 香火 < 100 → 失去神格，游戏失败（天界收取神力）
2. 香火 ≥ 100 + 阴德 ≥ 100 + 阳德 < 100 → **邪神结局**
3. 香火 ≥ 100 + 阳德 ≥ 100 + 阴德 < 100 → **天界善神结局**
4. 香火 ≥ 100 + 阴德 < 100 + 阳德 < 100 → **中性结局**（勉强维持神位，在小镇继续混吃等死）
5. 香火 ≥ 100 + 阴德 ≥ 100 + 阳德 ≥ 100 → **传奇结局**（超脱于善恶之外，成为传说）

| 结局类型 | 条件 | 结果 |
|----------|------|------|
| **失败结局** | 香火 < 100 | 失去神格，被贬为畜生 |
| **邪神结局** | 香火 ≥ 100 + 阴德 ≥ 100 + 阳德 < 100 | 与天界决裂，获邪神之力，小镇地狱化 |
| **天界善神结局** | 香火 ≥ 100 + 阳德 ≥ 100 + 阴德 < 100 | 获更高天界之力，爱与正义传播至镇外 |
| **中性结局** | 香火 ≥ 100 + 阴德 < 100 + 阳德 < 100 | 勉强维持神位，在小镇继续生活 |
| **传奇结局** | 香火 ≥ 100 + 阴德 ≥ 100 + 阳德 ≥ 100 | 超脱于善恶，成为传说 |

### 5.3 强硬不幸结局清单

**触发条件**：以下事件一旦发生，游戏立即结束，**不可挽回**

| 事件 | 描述 | 结局 |
|------|------|------|
| **寺庙被烧毁** | 寺庙彻底损毁，神位根基尽失 | 失去神格，被贬为畜生 |
| **异教徒入侵** | 异教徒入侵小镇，岛民改信邪教 | 天界剥夺神位，小镇陷入混乱 |
| **寺庙重大事故** | 寺庙发生重大事故且被天界发现 | 天界问责，神位被降级或剥夺 |
| **岛民集体叛教** | 超过 50% 岛民停止供奉 | 香火来源枯竭，神位维持失败 |

### 5.4 游戏结束触发

1. 时间到达 60 天 → 触发 §5.2 结局判定
2. 触发 §5.3 强硬不幸结局 → 立即结束游戏
3. **命运事件** 触发的特殊终局 → 触发预设剧情结局

## 6. 核心循环（Core Loop）

```
观察命运 → 选择干预（消耗神力）→ 时间推进 → AI 模拟人物行为
    → 事件结算（命运硬币）→ 命运/缘分变化 → 周结算（香火/功德/阴德）→ 解锁能力/剧情
```

| 阶段 | 玩家行为 | 系统行为 |
|------|----------|----------|
| **观察命运** | 查看居民、关系、事件、危险征兆 | 展示当前命运网络 |
| **选择干预** | 消耗神力进行有限干预 | 修改部分命运倾向 |
| **时间推进** | 结束当前时段 | AI 模拟人物行为 |
| **事件结算** | 观看关键事件结果 | 命运硬币结算 |
| **命运变化** | — | 获得新信息与线索；更新人物状态与命运线 |
| **周结算** | — | 获得香火、功德、阴德；解锁能力与剧情 |

## 8. 业线系统（定稿）

> 业线是连接 NPC 结局与阴德/阳德的核心机制

### 8.1 业线结构

| 元素 | 说明 |
|------|------|
| **终点目标** | 角色的核心愿望/理想（如小帅：离开小镇生活） |
| **进度** | 0% ~ 100%，表示业线完成度 |
| **节点数** | 5~10 个关键事件节点 |
| **可跳关** | 突发事件可直接跳过多个节点或直接结束业线 |

### 8.2 进度计算

- 每个节点对应一定进度增量
- 触发节点事件 → 业线进度推进
- 达到 100% = 业线完成，触发结局

**示例（小帅业线）**：
| 节点 | 事件 | 进度 |
|------|------|------|
| 1 | 产生离开小镇的愿望 | 10% |
| 2 | 开始努力学习备考 | 25% |
| 3 | 参加高考 | 50% |
| 4 | 拿到大学通知书 | 90% |
| 5 | 从港口离开小镇 | 100% |

### 8.3 结局判定

**判定三要素**：
1. **角色愿望达成度**：是否实现了核心理想
2. **幸福度**：主观感受（系统维护的幸福度数值）
3. **正常三观**：客观道德评判（基于常识价值观）

**结局类型**：
| 类型 | 判定条件 | 阴德 | 阳德 |
|------|----------|------|------|
| **好结局** | 愿望达成 + 幸福度高 + 符合三观 | 0 | +5~20 |
| **坏结局** | 愿望破碎 + 幸福度低 + 违背三观 | +5~20 | 0 |
| **中性结局** | 愿望部分达成或有代价 | +5 | +5 |

**示例对比**：
| 结局 | 情况 | 阴德 | 阳德 |
|------|------|------|------|
| 好结局 | 考上大学离开小镇 | 0 | +10 |
| 坏结局 | 被拐卖到缅甸 | +10 | 0 |
| 中性 | 放弃大学留在小镇（因土地公干预伤害了小美） | +5 | +5 |

### 8.4 跳关机制

突发事件可直接跳过多个节点或直接结束业线：
- **被拐卖**：跳到业线结束，触发坏结局
- **意外身亡**：跳到业线结束，触发坏结局
- **奇迹般的机遇**：跳过中间节点，快速推进进度

跳关可能导致负面或意外的结局。

### 8.5 NPC 业线配置

| NPC | 主业线 | 支线数 | 机会业线 |
|------|--------|--------|----------|
| 每人 | 1 条 | 2~3 条 | 随机刷新（单次事件） |

**业线类型**：
- 主业线：角色的核心人生目标（如事业、爱情、家庭）
- 支线：次要愿望或人际关系（如友情、复仇、救赎）
- 机会业线：单次事件（如解决一起纠纷、帮助一个路人）

### 8.6 NPC 记忆系统（方案 C）

> NPC 数据结构采用分层存储：静态属性 + 动态状态 + Agent Memory + 缘线 + 历史互动数据库

#### 8.6.1 NPC 数据结构总览

```
NPC 角色
├── 【静态层】（固定不变）
│   ├── 基础信息：姓名、年龄、职业、外貌
│   ├── 故事背景：出身、家庭、经历
│   ├── 固定属性：心智、信仰、体魄、魅力
│   └── 性格模型：五维性格（善良、激进、感性、理性、好奇心）
│
├── 【动态层】（每时段更新）
│   ├── 当前位置：山林/沙滩/学校/...
│   ├── 状态值：幸福度、精力、情绪值
│   ├── 当前目标：短中期目标
│   └── 业线进度：主线/支线进度百分比
│
├── 【Agent Memory 层】（游戏故事感知）
│   ├── 完整事件链：按时间顺序记录所有参与的事件
│   ├── 关键记忆：标记为"重要"的事件（可权重排序）
│   ├── 情感印象：对其他角色的情感印象（好感度、信任度）
│   └── 溢出处理：当 memory 溢出时，精简后重新存储
│
├── 【缘线层】（关系状态）
│   ├── 关系性质：红/金/蓝/灰/黑
│   ├── 线条粗细：1~5（根据历史事件+性格计算）
│   ├── 光效强度：0~100（活跃度）
│   └── 动态变化：每日/事件结算时重新计算
│
└── 【历史互动层】（事件数据库）
    └── JSON 格式存储：时间、地点、参与角色、事件类型、结果
```

#### 8.6.2 静态层示例

```json
{
  "name": "小帅",
  "age": 18,
  "occupation": "学生",
  "background": "归潮镇本地人，家境贫寒，梦想考上大学离开小镇",
  "attributes": {
    "mind": 7,
    "faith": 5,
    "physique": 6,
    "charm": 6
  },
  "personality": {
    "kindness": 0.7,
    "aggression": 0.3,
    "sensibility": 0.8,
    "rationality": 0.6,
    "curiosity": 0.9
  }
}
```

#### 8.6.3 Agent Memory 层示例

```json
{
  "eventChain": [
    { "day": 5, "event": "与小美在海边聊天", "importance": 8, "emotion": "开心" },
    { "day": 12, "event": "拿到大学通知书", "importance": 10, "emotion": "激动" }
  ],
  "impressions": {
    "小美": { "affinity": 85, "trust": 70, "emotion": "爱慕" }
  }
}
```

#### 8.6.4 缘线层示例（小帅↔小美）

```json
{
  "from": "小帅",
  "to": "小美",
  "type": "红",           // 爱情
  "thickness": 3,         // 1~5（青梅竹马）
  "glow": 50,             // 0~100 活跃度
  "calculation": {
    "baseValue": 3,       // 基础值
    "eventBonus": 0.5,    // 历史事件加分
    "personalityMatch": 0.8  // 性格匹配度
  }
}
```

#### 8.6.5 历史互动层示例（独立数据库）

```json
{
  "interactionId": "evt_001",
  "time": "第12天下午",
  "location": "沙滩",
  "participants": ["小帅", "小美"],
  "eventType": "对话",
  "description": "小帅向小美告白",
  "result": "成功",
  "impact": {
    "小帅": { "happiness": +20, "memoryImportance": 9 },
    "小美": { "happiness": +15, "memoryImportance": 8 }
  }
}
```

#### 8.6.6 溢出处理策略

| 触发条件 | 处理方式 |
|----------|----------|
| Memory 条数 > 50 | 按重要性权重排序，保留前30条，其余合并为摘要 |
| Memory 总token > 10k | LLM 生成精简摘要，替换旧记录 |

#### 8.6.7 数据库存储建议

- **缘线层**：图数据库（如 Neo4j）存储 NPC 间关系
- **历史互动层**：JSON 文件或 NoSQL 数据库（如 MongoDB）
- **Agent Memory**：向量数据库（如 Milvus）支持语义检索

| 游客/外来人 | C 群体NPC | 1 | 港口旅客、临时访客 | 港口事件、新闻传播、氛围变化 | 无完整业线 |

**Agent级别说明**：
- **S 完整Agent**：完整静态属性、动态状态、Agent Memory、主线+支线业线、长期缘线变化。
- **A 半完整Agent**：保留关键记忆与业线，但减少主动模拟频率；只在相关事件/地点/关系触发时更新。
- **B 规则驱动**：使用固定状态机与有限记忆摘要，不做完整 Agent 推演。
- **C 轻量剧情NPC**：传统剧情角色或群体状态，只提供固定反应、提示、线索或氛围，不进入 Agent Memory。

#### 8.7.3 轻量剧情 NPC 使用规则

轻量剧情 NPC 用于降低 AI 成本，同时补足世界可信度。它们不进入完整 Agent Memory，不参与每日自由模拟，只在特定剧情、地点或事件中出现。

| 类型 | 使用方式 | 示例 |
|------|----------|------|
| 固定剧情角色 | 在开局、周结算、终局等关键节点提供固定反应 | 天界使者、邪神低语 |
| 功能触发角色 | 用于启动事件、提供线索、推进地点功能 | 学校老师、镇长/村委代表 |
| 线索载体 | 不作为行动角色，只作为文本、病历、遗物、录音出现 | 老医生遗留记录 |
| 群体状态 | 以整体数值或氛围表现，不拆成单个角色 | 普通信徒群体、邪教随从、游客 |

**设计限制**：
- 不拥有独立业线。
- 不拥有长期 Agent Memory。
- 不参与每时段全量模拟。
- 可以拥有固定台词、固定反应、固定事件条件。
- 可以影响香火、邪教传播度、庙会支持度等群体变量。

## 10. 玩家能力与成长

### 10.1 玩家属性

| 属性 | 初始值 | 说明 |
|------|--------|------|
| 香火数 | 50 | 每日 -1，60 天后 ≥100 获胜 |
| 阴德 | 0 | 只增不减，≥100 触发邪神路线 |
| 阳德 | 0 | 只增不减，≥100 触发正神路线 |
| 神力 | 10 | 每日恢复至满，上限动态调整 |

### 10.2 技能表（定稿）

| 等级 | 技能名 | 消耗 | 限制 | 效果 |
|------|--------|------|------|------|
| **0** | 观测 | 0 | 无 | 查询并展示命运线、缘分线，不进行AI推导，只做数据展示 |
| **1** | 托梦 | 3 | 每天夜晚可执行 | 间接影响NPC认知与运势，改变次日决策倾向 |
| **1** | 赐福 | 2 | 遇见事件时可执行 | 增强命运硬币（提高成功率 50%→70%，或增减 1 枚硬币） |
| **2** | 引导机缘 | 5 | 无 | 引导事件向特定方向发展，创造或强化机会 |
| **3** | 遮蔽/推迟信息 | 8 | 无 | 阻断或延迟信息传递，制造误解或延迟决策 |
| **4** | 剪断/建立命运线 | 10 | 无 | 对缘线/命运线的结构级操作，彻底改变人际关系或人生轨迹 |

### 10.3 神力上限机制（周结算）

**每周结束后**，根据玩家表现调整神力上限：

| 表现维度 | 权重 | 影响 |
|----------|------|------|
| 香火增长 | 40% | 香火越高，上限↑ |
| 干预成功率 | 30% | 成功率越高，上限↑ |
| 阴德/阳德积累 | 30% | 道德资源越高，上限↑ |

**计算公式**：
```
新上限 = 当前上限 + (香火增量 × 0.1) + (干预成功率 - 0.5) × 2 + (阴德+阳德增量 × 0.05)
新上限 = Clamp(新上限, 5, 20)
```

**示例**：
- 第 1 周初始上限 10
- 香火增长 3 点 → +0.3
- 干预成功率 60% → +0.2
- 阴德+阳德增长 8 点 → +0.4
- 新上限 = 10 + 0.3 + 0.2 + 0.4 = **10.9 → 11**（向上取整）

### 10.4 技能解锁条件

| 技能等级 | 解锁条件 |
|----------|----------|
| Lv.0 观测 | 初始解锁 |
| Lv.1 托梦/赐福 | 初始解锁 |
| Lv.2 引导机缘 | 阴德+阳德 ≥ 10 或 神力上限 ≥ 12 |
| Lv.3 遮蔽/推迟信息 | 阴德 ≥ 30 或 阳德 ≥ 30 或 神力上限 ≥ 15 |
| Lv.4 剪断/建立线 | 阴德 ≥ 50 或 阳德 ≥ 50 或 神力上限 ≥ 18 |

> **设计意图**：Lv.0-1 用于熟悉系统，Lv.2-4 需要一定积累才能解锁，形成成长节奏。

## 11. 资源与经济

### 11.1 香火与神力获取

- 提升岛民信仰
- 每日岛民供奉（系统自动结算）
- 实现他人愿望后的回报（付出—回报结构）

### 11.2 香火机制（定稿）

- **初始值**：50 点
- **每日消耗**：-1 点（维持神格基础消耗）
- **胜利条件**：60 天后 ≥ 100 点
- **失败条件**：60 天后 < 100 点（失去神格）

### 11.3 命运硬币系统（定稿）

**基本规则**：

事件结算时，根据角色对应的能力值进行判定：
- 角色能力值（如 5 点体魄）
- 事件需求值（如篮球赛胜利需 3 点）
- 基础成功概率：50%

**硬币机制**：

| 操作 | 效果 |
|------|------|
| **抛硬币** | 抛出 N 枚硬币，其中 N = 能力值 - 需求值；正面数量 ≥ 1 则成功 |
| **神力影响** | 消耗神力可提高正面概率（50% → 70%）或直接增减硬币数量 |
| **玩家干预** | 可选择增减硬币或调整概率，但消耗神力 |

**示例**：
- 角色体魄 5，事件需求 3 → 抛 2 枚硬币
- 基础每枚硬币 50% 正面
- 消耗神力提升至 70% 正面
- 至少 1 枚正面则成功

## 13. 剧情大纲与周阶段排期

### 13.1 叙事原则

《命运的织线》采用 **周阶段 + 可变事件池** 的剧情结构，而不是完全固定的逐日线性剧本。

| 原则 | 说明 |
|------|------|
| **周阶段推进** | 60天约等于9周，每周定义大致局势、主线压力和事件池 |
| **事件池触发** | 关键事件根据NPC状态、地点、缘线、业线、玩家干预触发 |
| **少量锚点事件** | 开局通知、第60天天灾等事件固定发生，保证主线框架 |
| **AI结局变体** | 主结局由系统判定，NPC个人后日谈可由AI根据最终状态生成 |
| **敏感内容暗示** | 暴力、性侵害、死亡、邪教等内容使用线索、传闻、状态变化表达，不露骨描写 |

玩家作为归潮镇土地公，需要在60天内恢复香火、引导岛民命运，并决定第60天的庙会到底会成为守护小镇的避难仪式、邪神入主的献祭仪式，还是被人祸与天灾共同吞没。

| 启动 | 顾沉舟偷渡上岛，江雪仪接应 |
| 发展 | 拉拢不满者，尤其陈海生 |
| 扩散 | 秘密聚会、新信仰话术、广场活动 |
| 冲突 | 许明川、周行知、赵守正逐渐接近真相 |
| 收束 | 邪教失败、邪教入主、广场邪教集会替代庙会 |

#### 13.5.3 侦探凶情线：真相 vs 沉默

| 阶段 | 内容 |
|------|------|
| 启动 | 周行知发现港口偷渡线索 |
| 发展 | 许明川接触异常案件 |
| 升级 | 失踪、伤害、诊所异常、寺庙账目 |
| 冲突 | 证据不足、岛民沉默、妹妹涉险 |
| 收束 | 成功逮捕/驱逐邪教核心，或证据链断裂导致庙会失败 |

### 13.6 终局与 AI 后日谈

结局分为两层：

1. **主结局框架**：由系统根据香火、阴德、阳德、关键事件和地点状态判定。
2. **AI后日谈变体**：由AI根据NPC业线、缘线、关键事件和玩家干预风格生成个性化收束。

| 终局状态 | 关键条件 | 主结局倾向 |
|----------|----------|------------|
| 大多数村民在山顶寺庙庙会 | 庙会成功、邪教失败、寺庙未起火 | 好结局1：天灾伤亡低 |
| 天灾被阻止或削弱 | 高香火、高阳德、关键业线达成 | 好结局2：庙会正常幸福 |
| 广场举行邪教活动 | 邪教传播成功、庙会被替代 | 坏结局1：海啸伤亡惨重 |
| 寺庙庙会发生纵火+地震 | 庙会举办但陈海生/邪教/火灾隐患恶化 | 坏结局2：最惨重 |
| 60天前资源崩盘 | 香火/神力不足，玩家无力干预 | 不幸结局：岛民自生自灭 |

**AI后日谈输入**：

| 输入 | 影响文本 |
|------|----------|
| 每个NPC业线结局 | 谁活着、谁离岛、谁留下、谁改变 |
| 缘线状态 | 谁在一起、谁决裂、谁和解 |
| 关键事件 | 是否抓住顾沉舟、寺庙是否烧毁、庙会是否举办 |
| 地点状态 | 寺庙/广场/港口是否受损 |
| 道德资源 | 结局语气偏正神、邪神、中性或传奇 |
| 玩家干预风格 | 被称为仁慈土地公、恶神、懒神或织命者 |

**示例变体**：
- 如果陈远舟离岛成功：他在灾后寄回第一封大学来信。
- 如果林潮音成为自愿巫女：她继续主持重建后的春祭。
- 如果陈海生救赎：他在废墟中背着苏婉逃上山。
- 如果顾沉舟未被捕：传闻有人在外岛看见过他的背影。

## 18. 缺漏与改进建议

### 18.1 必须补清的硬设计

1. **统一时间轴**：建议定稿为「60 游戏日 = 绩效期」，每日 3 时段 = 180 次决策窗口上限（干预次数另限）
2. **三线资源公式**：例：行为标签 → 日终 ±阴德/阳德/香火；100 点阈值触发的结局互斥表
3. **神力经济表**：每个干预动作的消耗、失败是否退款、周回复量
4. **命运硬币**：建议定义为「事件结算的随机/权重修正器」，并显示给玩家
5. **AI 安全与品质**：爱情/暴力/死亡/未成年内容的规则过滤与人工审核样例库
6. **NPC MVP 规模**：建议首版 6～12 人，每人 1 核心 + 2～3 缘线，验证后再扩

### 18.2 系统文档缺口

| 系统 | 现状 | 建议产出 |
|------|------|----------|
| 业线 | UI 有、机制无 | 独立 GDD 小节：触发、显示、与道德资源挂钩 |
| 干预技能 | 仅等级名 | 技能表：目标、消耗、CD、成功率、示例 |
| 邪神/天界 | 仅终局描述 | 中期事件链、UI 提示、背叛/皈依节点 |
| 强硬不幸结局 | 未定义 | 列表 + 不可逆标记 + 是否可挽回 |
| 存档 | 未提 | 日终 autosave + 关键节点分支标记 |

### 18.3 体验与范围控制

- **正反馈具象化**：周结算解锁什么（新干预、地图信息、NPC 秘密）逐条列出
- **MVP 切割**：第一条可玩竖切 = 3 NPC 三角关系 + 1 核心命运节点 + 7 游戏日
- **第三人称实现**：建议默认俯视地图；关键事件再进事件面板演出

### 18.4 技术预研（若用 Unity 6 + 本工作室流程）

- AI 叙事服务：本地规则引擎 vs 云端 LLM 的分工
- 因果图数据结构：缘线/命运线的存储与回放
- 成本：每时段 LLM 调用次数上限与缓存策略
