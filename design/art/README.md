# art 美术策划目录

> 文档状态：整理版  
> 更新日期：2026-06-12  
> 用途：美术风格、资产需求、视觉参考和低成本制作边界

---

## 1. 目录职责

`art/` 用于存放美术方向、资产清单、风格规范和交付标准。

本目录回答的问题是：

- 游戏整体看起来是什么风格？
- 美术资产优先做哪些？
- 每类资产需要什么尺寸、格式和表现方式？
- 哪些视觉效果属于 MVP，哪些应推迟？
- UI、美术、研发之间如何对齐资源需求？

---

## 2. 当前美术基准

| 项目 | 当前方向 |
|---|---|
| 引擎 | Godot |
| 美术风格 | 简约 2D 像素风 |
| 氛围 | 温暖小镇 + 轻微神秘 + 命运压力 |
| 成本策略 | 少量高复用场景、角色头像、图标和 UI 组件 |
| UI 方向 | 暖木色、纸张感、像素边框、高对比文本 |
| 表现重点 | 归潮镇、土地庙、香火、海雾、缘线、灾难征兆 |

## 4. 美术交付格式建议

| 类型 | 建议格式 |
|---|---|
| 像素图 | PNG，透明背景 |
| UI 图标 | PNG，透明背景，必要时附 SVG 源文件 |
| 场景底图 | PNG |
| 色板 | PNG 或 Markdown 色值表 |
| 动画 | PNG 序列或 Godot 可用 SpriteFrames |
| 源文件 | Aseprite / PSD / Krita / SVG，根据制作工具保留 |

---

## 5. 优先级规则

| 优先级 | 说明 |
|---|---|
| P0 | MVP 必须使用，缺失会阻塞首个可玩版本 |
| P1 | 完整体验需要，但不阻塞 MVP |
| P2 | 增强表现、氛围或长线内容 |

## 6. 成本边界

MVP 阶段不做：

- Live2D
- 复杂角色全身动画
- 大量逐帧战斗动画
- 复杂天气系统
- 复杂动态光照
- 全场景昼夜重绘
- 大型关系网动态特效
- 高成本结局演出动画

MVP 阶段优先做：

- 可读性强的主地图
- 少量核心地点图标
- 3-6 名核心 NPC 头像
- 通用 UI 面板和按钮
- 资源 / 风险 / 事件图标
- 少量可复用氛围特效，如香火烟雾、海雾、警告闪烁
