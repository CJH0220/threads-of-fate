# 美术风格指南

> 文档状态：初版  
> 更新日期：2026-06-12  
> 项目：《命运的织线》  
> 引擎方向：Godot  
> 美术方向：简约 2D 像素风

---

## 1. 总体风格定位

《命运的织线》的美术目标是在低成本条件下，建立一个温暖、清晰、略带神秘感的海岛小镇。

关键词：

```text
温暖小镇
土地庙
香火
海雾
命运织线
像素边框
纸张与木牌
轻微民俗神秘
克制的灾难预兆
```

视觉不追求华丽复杂，而是优先保证：

- 玩家一眼能看懂 UI 信息。
- 地点和 NPC 有足够辨识度。
- 命运、神明、邪教、天灾等主题有氛围但不过度恐怖。
- 资产可以复用、替换和逐步扩展。

## 2. 画面基调

| 类型 | 方向 |

## 3. 色彩规范

### 3.1 基础色板

| 用途 | 色彩方向 | 参考色 |
|---|---|---|
| 纸张 / 面板底 | 米黄、浅羊皮纸 | `#f1d9aa` |
| 木质边框 | 暖棕 | `#8a5a32` |
| 深描边 | 深褐 | `#3b2a1f` |
| 普通按钮 | 赤棕 | `#b8753c` |
| 焦点 / Hover | 亮橙棕 | `#d08a4a` |
| 禁用状态 | 低饱和棕灰 | `#714432` |
| 背景夜色 | 深蓝灰 | `#243447` |

### 3.2 系统色

| 系统 | 色彩方向 | 说明 |
|---|---|---|
| 香火 | 橙红 / 烟火色 | 表示信仰、供奉、神位稳定 |
| 神力 | 蓝紫 / 青蓝 | 表示神明能力和消耗 |
| 阳德 | 金黄 / 暖绿 | 表示正向、守护、救助 |
| 阴德 | 暗红 / 紫黑 | 表示操控、遮蔽、邪路 |
| 危险 | 红色 + 图标 | 不能只靠颜色表达 |
| 已完成 | 灰化 / 低饱和 | 保留可读性 |

## 4. 像素规则

### 4.1 分辨率基准

首批 UI 参考以 1280 × 720 为基准。资产制作应考虑 16:9 显示比例。

建议像素单位：

| 类型 | 建议尺寸 |
|---|---|
| 小图标 | 16×16 / 24×24 |
| UI 常用图标 | 32×32 |
| NPC 头像 | 64×64 / 96×96 |
| 地点图标 | 96×96 / 128×128 |
| 小型场景插图 | 320×180 |
| 主地图底图 | 960×540 或 1280×720 |

### 4.2 绘制规则

- 使用清晰轮廓，不做过细线条。
- 轮廓优先使用深褐或深蓝灰，不用纯黑铺满。
- 色块简洁，少量阴影即可。
- 避免过度抖动和复杂材质。
- UI 面板边框保持一致厚度。
- 图标应在 32×32 下仍能识别。

## 5. UI 视觉语言

UI 应继承“命运织线器 + 土地庙文书”的混合感：既像神明管理系统，又像纸卷、木牌和供桌上的记录。

### 5.1 推荐元素

- 木质横条
- 纸张面板
- 像素描边
- 香火图标
- 小旗帜事件标记
- 命运线条装饰
- 朱印 / 符纸式危险标记

### 5.2 禁止或慎用

- 高科技霓虹 UI
- 大量玻璃拟态
- 复杂 3D 面板
- 过多粒子特效
- 需要精细鼠标操作的小控件
- 只靠颜色表达风险或阵营

## 6. 场景方向

### 6.1 主地图

主地图应像一张可点击的小镇俯视图。MVP 阶段可以是一张固定 2D 地图底图，地点以按钮或热点方式实现。

重要地点：

| 地点 | 视觉关键词 |
|---|---|
| 寺庙 | 山顶、鸟居或庙门、香炉、石阶 |
| 学校 | 操场、教学楼、黄昏窗光 |
| 港口 | 木栈桥、渔船、海雾、灯塔 |
| 诊所 | 白色招牌、药箱、温柔但冷清 |
| 咖啡店 | 暖灯、招牌、日常社交 |
| 酒屋 | 暗暖灯光、木门、成年人秘密 |
| 海鲜店 | 鱼箱、蓝布、渔网 |
| 广场 | 公告栏、集会空间、节庆装饰 |

### 6.2 氛围变化

| 状态 | 表现 |
|---|---|
| 日常 | 暖光、清晰色彩、轻量动态 |
| 危险 | 地点旁出现红色图标、暗色覆盖或警告边框 |
| 海雾 | 低饱和蓝灰遮罩 |
| 邪教渗透 | 紫黑色边缘、异常符号、小范围暗红 |
| 香火旺盛 | 寺庙暖光增强、香火烟雾更明显 |
| 香火衰弱 | 香炉变暗、灰烟、面板低饱和 |

## 7. 角色方向

MVP 阶段优先做头像，不要求全身立绘。

### 7.1 头像要求

- 至少能区分年龄、职业、情绪基调。
- 头像应适配人物面板、事件面板和列表卡片。
- 表情可先做默认表情，P1 再扩展情绪差分。

### 7.2 角色辨识关键词

| 角色 | 视觉关键词 |
|---|---|
| 林潮音 | 高中生、灵力敏感、柔和但焦虑 |
| 陈远舟 | 高中生、认真、离岛梦想、书包或书本 |
| 顾沉舟 | 外来游客、优雅、危险、过度亲和 |
| 江雪仪 | 新医生、冷静、白衣、温柔控制感 |
| 许明川 | 年轻民警、疲惫但正直 |
| 陈海生 | 渔民、粗糙、疲惫、愤怒积累 |
| 慧圆 | 寺庙管理者、表面慈和、内在腐败 |

## 8. 特效方向

MVP 只做少量复用特效：

| 特效 | 用途 | 优先级 |
|---|---|---|
| 香火烟雾 | 香火状态、寺庙氛围 | P0 |
| 命运线闪烁 | 缘线 / 业线变化提示 | P0 |
| 危险警告闪烁 | 高风险事件 | P0 |
| 海雾遮罩 | 天灾和异常氛围 | P1 |
| 金色赐福光点 | 正向干预反馈 | P1 |
| 紫黑阴影扩散 | 邪路 / 阴德反馈 | P1 |

避免大型全屏特效，减少制作和性能成本。

---

## 9. 可访问性要求

美术资产必须配合 UI 可访问性规范：

- 危险信息必须有图标或文字，不只靠红色。
- 资源类型不能只靠颜色区分，应有图标或文字标签。
- UI 文本与背景保持高对比。
- 焦点状态必须明显。
- 减少动画模式下，闪烁和漂浮动效应降低或关闭。

## 10. MVP 美术验收标准

- [ ] 主地图地点在 1280×720 下清晰可辨。
- [ ] P0 UI 面板、按钮、资源标签风格统一。
- [ ] 至少 3 名 MVP NPC 有头像。
- [ ] 香火、神力、阳德、阴德有可识别图标或标签。
- [ ] 危险事件有非颜色提示。
- [ ] 事件面板有可复用场景插图或占位图规则。
- [ ] 所有像素图在目标缩放下不模糊。
- [ ] 资产命名与 `asset-list.md` 一致。
