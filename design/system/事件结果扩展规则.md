# 事件结果扩展规则

> 文档状态：初版  
> 更新日期：2026-06-13  
> 用途：定义事件选项、干预修正、资源变化、NPC 状态变化、缘线业线变化和后续事件配置规则

---

## 1. 系统目标

事件结果扩展规则用于把 `事件配置表.csv` 中的事件，落到可测试、可调参、可复盘的结算结果。

本规则回答：

- 一个事件有哪些可能结果？
- 玩家干预如何改变结果？
- 资源、NPC 状态、缘线、业线和镇域状态如何变化？
- 结果如何写入历史？
- 哪些结果可以触发后续事件或 Game Over？

核心原则：事件文本可以变化，但结算字段必须结构化。

---

## 2. 数据来源

| 数据文件 | 用途 |
|---|---|
| `data/事件配置表.csv` | 定义事件何时、何地、由谁触发 |
| `data/事件结果表.csv` | 定义事件的可结算结果和变化 |
| `data/干预技能表.csv` | 定义干预消耗、目标和默认倾向 |
| `data/资源数值表.csv` | 定义资源初始值、上下限和 UI 优先级 |
| `data/缘线初始关系表.csv` | 定义缘线 ID 与基础关系 |
| `data/业线节点表.csv` | 定义关键 NPC 的业线节点 |
| `data/本地化文本表.csv` | 定义结算摘要和 UI 文案 key |

---

## 3. 事件结果结构

每条结果使用 `OutcomeId` 唯一标识。

```text
EventOutcome
- OutcomeId
- EventId
- OutcomeName
- TriggerCondition
- ResourceDelta
- BondDelta
- KarmaDelta
- NpcStateDelta
- TownStateDelta
- FollowUpEventIds
- HistorySummaryKey
- Description
```

### 3.1 字段说明

| 字段 | 说明 |
|---|---|
| `OutcomeId` | 结果唯一 ID |
| `EventId` | 所属事件 ID，必须存在于事件配置表 |
| `OutcomeName` | 策划可读名称 |
| `TriggerCondition` | 触发条件，可引用干预、阈值、状态标签 |
| `ResourceDelta` | 资源变化，格式如 `yang_de:+3;divine_power:-2` |
| `BondDelta` | 缘线变化，格式如 `bond_chaoyin_yuanzhou:+5` |
| `KarmaDelta` | 业线变化，格式如 `chaoyin_witch_line:+8` |
| `NpcStateDelta` | NPC 状态变化，格式如 `lin_chaoyin:stress+5` |
| `TownStateDelta` | 镇域状态变化，如 `public_order:-3` |
| `FollowUpEventIds` | 后续事件 ID，多个用 `;` 分隔 |
| `HistorySummaryKey` | 本地化摘要 key |
| `Description` | 策划说明，不直接等同游戏正文 |

## 4. 结算优先级

同一事件可能有多个结果候选，按以下顺序判定：

1. 强制失败结果，如火灾失控、香火崩塌。
2. 明确干预结果，如托梦、赐福、显灵警示。
3. 阈值结果，如缘线强度、业线进度、资源阈值。
4. 默认结果。
5. 兜底结果。

若多个结果同时满足，只取优先级最高的一个主结果；次要变化应通过结果字段合并，而不是同时触发多个主结果。

## 5. 干预修正规则

### 5.1 托梦

| 作用 | 规则 |
|---|---|
| 信息提示 | 给 NPC 一个模糊预感或犹豫标签 |
| 典型变化 | 降低高风险业线推进速度 |
| 成本 | 消耗神力 |
| 风险 | 不能直接让 NPC 知道完整真相 |

示例：

```text
TriggerCondition: dream_hint
ResourceDelta: divine_power:-2;yang_de:+2
NpcStateDelta: chen_haisheng:warning_dream
```

### 5.2 赐福

| 作用 | 规则 |
|---|---|
| 正向缓冲 | 降低压力、提高关系、小幅提高阳德 |
| 典型变化 | 使坏结果变成中性或轻微好结果 |
| 风险 | 不能消除根因，只能缓冲当次事件 |

### 5.3 牵引机缘

| 作用 | 规则 |
|---|---|
| 改变遇见的人或地点 | 触发机会事件或改变后续事件条件 |
| 典型变化 | 提高缘线、发现线索、开启后续事件 |
| 风险 | 可能错过原本路径上的其他事件 |

### 5.4 遮蔽灾兆

| 作用 | 规则 |
|---|---|
| 隐藏线索或风险 | 让 NPC 避开真相或错过调查 |
| 典型变化 | 增加阴德，降低秩序或提高邪教传播 |
| 风险 | 可短期避险，长期恶化结局倾向 |

### 5.5 显灵警示

| 作用 | 规则 |
|---|---|
| 高成本强提醒 | 阻止高危事件或降低致命风险 |
| 典型变化 | 大幅消耗神力，提高阳德或恐惧 |
| 风险 | 可能让暗处敌人知道神明介入 |

## 6. 资源变化规则

资源变化必须遵守 `资源数值表.csv` 的上下限。

| 资源 | 变化来源 | 说明 |
|---|---|---|
| 香火 | 周结算、重大事件、信任变化 | 低于 100 会进入失败风险 |
| 神力 | 周恢复、香火转化、干预消耗 | 不足时不能使用干预 |
| 阳德 | 守护、救助、诚实揭露 | 推动正神和传奇路线 |
| 阴德 | 遮蔽、操控、牺牲他人 | 推动邪神路线 |
| 邪教传播值 | 顾沉舟事件、遮蔽线索 | 影响邪教结局和 Game Over |
| 小镇秩序 | 调查、公开事件、灾前准备 | 影响终局生还质量 |
| 寺庙信任 | 慧圆线、巫女线、账目线 | 影响最终聚集地点和旧信仰路线 |

## 7. NPC 状态变化规则

NPC 状态分为数值型、标签型和剧情锁定型。

| 类型 | 示例 | 用法 |
|---|---|---|
| 数值型 | `stress+5`, `fire_risk-20` | 可累计，可用于阈值判断 |
| 标签型 | `clinic_suspicion`, `warning_dream` | 表示是否获得线索或经历 |
| 剧情锁定型 | `cult_committed`, `left_cult` | 影响后续事件和结局 |

状态变化要写入存档，不应只存在于事件文本中。

---

## 8. 缘线变化规则

缘线强度建议范围为 `0-100`。

| 范围 | 含义 |
|---|---|
| 0-20 | 破裂或敌意 |
| 21-40 | 紧张、疏离、互不信任 |
| 41-60 | 普通关系或仍可修复 |
| 61-80 | 强关系、互相信任 |
| 81-100 | 命运强绑定 |

缘线变化应满足：

- 单次普通事件变化建议 `±1-5`。
- 关键事件变化建议 `±5-12`。
- 致命或不可逆事件可以超过 `±15`。
- UI 显示不必展示精确数值，可显示阶段。

---

## 9. 业线变化规则

业线用于表示角色长期命运方向。

| 变化幅度 | 使用场景 |
|---|---|
| `+1-3` | 日常事件轻微推动 |
| `+4-8` | 关键事件推进 |
| `+9-15` | 锚点事件或不可逆选择 |
| `-1-10` | 玩家成功阻断风险路线 |

业线节点到达阈值后，可以解锁：

- 新事件。
- 新 UI 线索。
- 新结局候选。
- 新干预窗口。

---

## 10. 后续事件规则

`FollowUpEventIds` 只表示候选后续事件，不保证立即触发。

后续事件仍需检查：

1. 当前天数和时段。
2. NPC 是否可参与。
3. 地点是否可用。
4. 必要状态是否满足。
5. 是否已触发或被阻断。

---

## 11. 历史记录规则

事件结果写入历史时分三层：

| 层级 | 内容 |
|---|---|
| 系统历史 | `EventId`, `OutcomeId`, 时间、参与 NPC、结构化变化 |
| 玩家摘要 | `HistorySummaryKey` 对应的本地化文本 |
| 调试记录 | 完整条件、判定顺序、被拒绝候选结果 |

玩家历史不显示隐藏条件、精确权重和未解锁结局公式。

---

## 12. AI 文本边界

AI 只能在结果已经确定后润色：

- 事件正文。
- 结算摘要。
- NPC 短句。
- 结局后日谈。

AI 不能改写：

- `OutcomeId`。
- 资源变化。
- NPC 状态变化。
- 缘线业线变化。
- 后续事件。
- 结局路线。

## 14. 边界情况

| 情况 | 处理 |
|---|---|
| 结果没有匹配 | 使用默认结果，若无默认结果则记录配置错误 |
| 资源变化越界 | 按上下限截断，并记录调试信息 |
| 后续事件不存在 | 忽略该后续事件并记录配置错误 |
| NPC ID 不存在 | 阻止该结果写入并报配置错误 |
| `HistorySummaryKey` 缺失 | 使用兜底摘要 key |
| AI 生成与结果冲突 | 丢弃 AI 文本，显示本地模板 |

---

## 15. 验收标准

- [ ] 每个 P0 事件至少有一个默认结果。
- [ ] 每个可干预 P0 事件至少有一个干预结果。
- [ ] 所有 `EventId` 能在 `事件配置表.csv` 中找到。
- [ ] 所有 `HistorySummaryKey` 能在 `本地化文本表.csv` 中找到。
- [ ] 所有资源变化字段引用 `资源数值表.csv` 中存在的资源。
- [ ] 所有缘线变化字段引用 `缘线初始关系表.csv` 中存在的缘线。
- [ ] 所有 NPC 状态变化字段引用存在的 NPC ID。
- [ ] 事件结果不依赖 AI 决定核心状态。
