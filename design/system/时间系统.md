# 时间系统

> 文档状态：初版  
> 更新日期：2026-06-12  
> 所属阶段：P0  
> 用途：定义 60 天结构、时段推进、周结算、终局触发和时间相关边界

---

## 1. 系统目标

时间系统负责驱动《命运的织线》的主循环。玩家所有观察、事件触发、NPC 行动、资源变化和结局判定，都必须落在明确的时间结构中。

系统目标：

1. 给玩家明确的倒计时压力。
2. 让小镇事件按照天数和时段推进。
3. 为事件系统、NPC 行为、资源系统和结局系统提供统一时间基准。
4. 支持周阶段剧情和第 60 天终局结算。

---

## 2. 时间结构

### 2.1 总时长

| 项目 | 规则 |
|---|---|
| 游戏总长度 | 60 天 |
| 起始日 | Day 1 |
| 终局日 | Day 60 |
| 教学序章 | 第 0 夜，不计入 60 天 |
| 周阶段 | 约 8 个完整周阶段 + 终局阶段 |

### 2.2 每日时段

默认每日分为三个可推进时段：

| 中文 | 数据枚举 | 说明 |
|---|---|---|
| 早 | `Morning` | 日常开始、学校、寺庙、诊所活动较多 |
| 午 | `Afternoon` / `Noon` | 学校、咖啡店、港口、调查事件较多 |
| 晚 | `Night` | 酒屋、港口、秘密聚会、梦境和危险事件较多 |

数据表可使用更细枚举：

```text
Morning
Noon
Afternoon
Night
Any
Daytime
```

UI 面向玩家时应简化显示为：

```text
第 {day} 天 · 早 / 午 / 晚
```

## 5. 时间推进流程

```text
玩家点击“推动时间”
→ 检查未查看危险事件
→ 必要时弹出危险确认
→ 锁定当前时段玩家操作
→ 结算当前时段未处理事件
→ 推进到下一个时段
→ 触发新时段 NPC 行动与事件检查
→ 生成变化摘要
→ 按跳转优先级进入事件 / 周总结 / 终局 / 主地图
```

### 5.1 时段推进规则

| 当前时间 | 推进后 |
|---|---|
| Day N 早 | Day N 午 |
| Day N 午 | Day N 晚 |
| Day N 晚 | Day N+1 早 |
| Day 60 晚 | 进入终局结算 |

### 5.2 推动时间前置检查

推动时间前必须检查：

- 是否有未查看的致命风险事件。
- 是否有强制锚点事件未播放。
- 是否有确认弹窗正在打开。
- 当前是否处于事件结算中。
- 当前是否已经进入终局锁定。

若存在未查看危险事件，必须调用危险确认：

```text
仍有危险征兆尚未查看。
若继续推动时间，可能错过干预机会。
```

## 6. 事件触发时机

时间系统不决定事件内容，但负责提供事件筛选上下文。

每次进入新时段时，事件系统应根据以下条件筛选事件：

| 条件 | 来源 |
|---|---|
| 当前天数 | TimeSystem |
| 当前时段 | TimeSystem |
| 当前周阶段 | TimeSystem |
| NPC 当前地点 | NPCBehaviorSystem |
| 地点状态 | LocationSystem |
| 资源状态 | ResourceSystem |
| 玩家历史干预 | InterventionHistory |
| 已触发事件记录 | EventHistorySystem |

## 7. 周结算规则

### 7.1 触发时机

周结算默认在以下时机触发：

```text
Day 7 / 14 / 21 / 28 / 35 / 42 / 49 / 56 的夜晚结算后
```

### 7.2 周结算内容

周结算至少显示：

- 本周香火变化
- 本周神力使用
- 本周阳德 / 阴德变化
- 本周关键事件
- 本周 NPC 命运变化摘要
- 下周主题预告

### 7.3 周结算跳转

周结算完成后：

| 条件 | 跳转 |
|---|---|
| 触发 Game Over | 游戏失败 / Game Over |
| Day 56 周结算后 | 进入终局阶段提示 |
| 普通周结算 | 返回主地图 HUD |

---

## 8. 终局触发规则

### 8.1 终局阶段

Day 57-60 为终局阶段。该阶段事件优先级高于普通日常事件。

终局阶段应减少随机日常事件，优先展示：

- 聚集地点决定
- 证据链最终确认
- 火灾 / 邪教 / 天灾风险
- 林潮音和陈远舟的最终选择
- Day 60 天灾事件

### 8.2 第 60 天结算顺序

```text
Day 60 早：最终聚集 / 庙会 / 广场事件
Day 60 午：地震 / 海啸 / 重大天灾
Day 60 晚：总评或 Game Over
```

### 8.3 终局跳转优先级

1. 若满足 Game Over 条件，进入 `ui-spec-game-over.md`。
2. 若未失败，进入 `ui-spec-final-summary.md`。
3. 总评后进入 `ui-spec-ending-screen.md`。
4. 可选保存到 `ui-spec-ending-gallery.md`。

## 11. 边界情况

| 情况 | 处理 |
|---|---|
| 推动时间时有危险事件未查看 | 弹出危险确认 |
| 当前事件未结算 | 禁用推动时间 |
| 推进失败 | 保持原时段，显示错误状态 |
| Day 60 晚后继续推进 | 禁止，进入终局结算 |
| 多个关键事件同时触发 | 事件系统按优先级排队 |
| 周结算和关键事件同时触发 | 先播关键事件，后周结算 |
| Game Over 与周结算同时触发 | Game Over 优先 |

## 13. 验收标准

- [ ] 游戏能从 Day 1 早推进到 Day 1 午。
- [ ] Day N 晚推进后进入 Day N+1 早。
- [ ] UI 顶栏正确显示当前天数和时段。
- [ ] 有危险事件未查看时，推动时间前出现确认。
- [ ] Day 7 晚结算后触发周总结。
- [ ] Day 60 终局事件后进入 Game Over 或 60 天总评。
- [ ] 推进失败时不会改变当前时间。
- [ ] 多个关键事件能按队列逐个播放。
