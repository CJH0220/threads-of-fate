# 缘线业线系统

> 文档状态：初版  
> 更新日期：2026-06-12  
> 用途：定义 NPC 关系、命运进度、关键节点、风险和角色结局映射规则

---

## 1. 系统目标

缘线业线系统负责把 NPC 之间的关系和 NPC 个人命运进度转化为可显示、可干预、可结算的数据。

本系统回答以下问题：

- 哪些 NPC 之间存在重要关系？
- 关系是什么类型、强度如何、是否被玩家看见？
- 每个关键 NPC 的命运线推进到哪里？
- 哪些事件会改变关系或命运节点？
- 玩家干预如何影响关系和命运走向？
- 角色结局如何从业线状态中判定？

MVP 阶段采用列表、进度值和节点状态表达，不实现复杂关系网图、不做自由 Agent 推理。

---

## 2. 核心定义

### 2.1 缘线

缘线表示两个 NPC 之间的关系。

```text
NpcA + NpcB + BondType + Strength + Visibility + RecentDelta
```

缘线不是单纯好感度，而是包含关系性质、强度和风险的结构化数据。

### 2.2 业线

业线表示一个 NPC 或一组 NPC 的命运进度。

```text
KarmaLineId + OwnerNpcId + Progress + CurrentNode + RiskState + EndingBias
```

业线用于表达角色是否接近好结局、中性结局、坏结局或特殊结局。

### 2.3 线与事件的关系

事件是缘线和业线变化的主要来源。

```text
事件发生 → 事件结果 → 缘线变化 / 业线节点推进 / 结局 flag 写入
```

玩家干预不直接写结局，只通过事件修正、状态标签和节点结果影响缘线与业线。

---

## 3. 设计原则

| 原则 | 说明 |
|---|---|
| 规则驱动 | 关系和命运变化由事件结果表、系统规则和条件判定决定 |
| 玩家可读 | 玩家应能理解“谁和谁关系变了”“谁的命运正在走向危险” |
| 不过度剧透 | UI 可显示趋势和风险，但不直接展示所有隐藏结局条件 |
| 低成本实现 | MVP 使用列表、进度条、节点状态，不做复杂动态关系网 |
| 可测试 | 每个事件对缘线、业线的影响应能被自动或手动验证 |
| AI 只写文本 | AI 可润色变化摘要，不能决定关系数值或命运节点 |

- 每个 NPC 多条完整支线树。
- 复杂心理模型和自然语言记忆检索。

## 5. 缘线规则

### 5.1 缘线类型

| BondType | 中文名 | 说明 | 颜色建议 |
|---|---|---|---|
| Love | 爱情 | 恋爱、暧昧、互相承诺 | 红 / 粉 |
| Family | 亲情 | 父母、子女、家庭责任 | 蓝 |
| Friendship | 友情 | 信任、陪伴、互助 | 青 / 蓝绿 |
| Interest | 利益 | 交易、合作、互相利用 | 金 / 黄 |
| Manipulation | 利用 | 一方诱导或操控另一方 | 紫 / 黑 |
| Hostility | 敌意 | 冲突、仇恨、对立 | 暗红 |
| Conspiracy | 共谋 | 共同隐瞒或推进危险计划 | 黑 / 紫 |
| Control | 控制 | 权力、精神或身份控制 | 灰黑 |
| Stranger | 陌生 | 未建立明确关系 | 灰 |

颜色必须配合文字或图标，不允许只靠颜色表达关系。

### 5.2 强度范围

缘线强度使用 0-100。

| 范围 | 等级 | 含义 |
|---:|---|---|
| 0-19 | Lv.1 | 关系微弱或刚建立 |
| 20-39 | Lv.2 | 有明确联系但不稳定 |
| 40-59 | Lv.3 | 稳定关系或明显影响 |
| 60-79 | Lv.4 | 强关系，能显著影响事件 |
| 80-100 | Lv.5 | 核心关系，可影响结局 |

### 5.3 关系变化来源

| 来源 | 示例 | 默认变化 |
|---|---|---:|
| 共同经历正向事件 | 一起躲过危险、互相支持 | +5 到 +15 |
| 误解或争吵 | 家庭冲突、约定破裂 | -5 到 -15 |
| 玩家正向干预 | 托梦化解误会、赐福促成沟通 | +3 到 +10 |
| 玩家负向干预 | 遮蔽真相、诱导错误选择 | -3 到 -12 |
| 邪教诱导 | 顾沉舟接触高压力 NPC | Manipulation +5 到 +20 |
| 关键节点选择 | 离岛、留下、揭露、隐瞒 | -20 到 +30 |

### 5.4 可见性

缘线有可见性状态。

| Visibility | 说明 | UI 表现 |
|---|---|---|
| Hidden | 完全隐藏 | 不显示 |
| Unknown | 知道有关系但不清楚性质 | 显示“？？？” |
| Partial | 显示关系类型但不显示强度 | “似乎关系紧张” |
| Visible | 显示类型、强度和近期变化 | 正常条目 |
| Revealed | 显示隐藏性质或危险关系 | 高亮或警告 |

玩家可通过观察、事件、强化观察、调查类事件提高可见性。

## 6. 业线规则

### 6.1 业线类型

| 类型 | 说明 | 示例 |
|---|---|---|
| Main | 主业线 | 林潮音是否属于归潮镇 |
| Side | 支线 | 母女和解、升学线、调查线 |
| Risk | 风险线 | 被邪教诱导、寺庙火灾风险 |
| Opportunity | 机会线 | 玩家干预后出现的替代命运 |

MVP 只要求每个核心 NPC 有一条主业线，支线可作为事件 flag 或摘要处理。

### 6.2 业线进度

业线进度使用 0-100%。

| 进度 | 含义 |
|---:|---|
| 0% | 命运线刚显形或尚未启动 |
| 25% | 第一关键节点出现 |
| 50% | 命运分叉明确 |
| 75% | 接近不可逆选择 |
| 100% | 业线完成，等待结局结算或已锁定结局 |

### 6.3 节点状态

| 状态 | 说明 |
|---|---|
| Locked | 未到时间或条件不足 |
| Hidden | 已存在但玩家尚未发现 |
| Active | 当前可推进或可干预 |
| Completed | 已完成，写入结果 |
| Failed | 已失败，可能写入坏结局 flag |
| Skipped | 被其他事件跳过，不再触发 |

### 6.4 结局倾向

业线会积累结局倾向，但不直接决定最终路线。

| EndingBias | 说明 |
|---|---|
| Good | 倾向角色好结局 |
| Neutral | 倾向中性结局 |
| Bad | 倾向坏结局 |
| Tragic | 接近强硬不幸结局 |
| Unknown | 玩家信息不足，结局未显形 |

## 7. 事件结算流程

缘线和业线在事件结算中按以下顺序处理：

```text
1. 读取事件基础结果
2. 应用玩家干预修正
3. 修改 NPC 状态标签
4. 修改缘线类型或强度
5. 推进或失败业线节点
6. 写入隐藏 flag
7. 更新玩家可见性
8. 生成 UI 结算摘要
9. 写入事件历史
```

### 7.1 缘线变化摘要

UI 结算中显示：

```text
林潮音 与 陈远舟 的关系更亲近了。缘线 +8
陈海生 对 顾沉舟 的信任上升了。危险缘线 +10
```

### 7.2 业线变化摘要

UI 结算中显示：

```text
林潮音的业线推进：灵力异常初显 → 被寺庙关注
陈海生的业线风险上升：他开始相信顾沉舟的话
```

## 8. 玩家干预影响

| 干预 | 对缘线影响 | 对业线影响 |
|---|---|---|
| 托梦 | 可减少误解、提高警觉、暴露隐藏关系 | 可激活警示节点或阻止风险节点 |
| 赐福 | 提高正向沟通、考试、行动成功率 | 提高 Good / Neutral 节点成功权重 |
| 牵引机缘 | 促成 NPC 相遇或错过相遇 | 可打开机会节点 |
| 遮蔽灾兆 | 隐藏危险信息，短期避免冲突 | 可能跳过警示节点并提高 Bad / Tragic 风险 |
| 显灵警示 | 对多人或地点发出强警告 | 可阻止高危节点或强制暴露风险 |
| 强化观察 | 不改数值，提高玩家可见性 | 显示更多当前节点和风险提示 |

## 9. MVP 示例

### 9.1 林潮音 - 陈远舟缘线

初始状态：

```text
BondId: bond_lin_chaoyin_chen_yuanzhou
BondType: Love
Strength: 60
Visibility: Partial
RecentDelta: 0
```

可能变化：

| 事件 | 结果 |
|---|---|
| 陈远舟提出离岛计划，林潮音愿意倾听 | Strength +8 |
| 林潮音被寺庙推为巫女，陈远舟没有及时支持 | Strength -10 |
| 玩家托梦让林潮音说出担忧 | Strength +5，Visibility +1 |
| 玩家遮蔽陈远舟获得真相 | Strength -8，写入隐藏误解 flag |

结局影响：

| 条件 | 可能结果 |
|---|---|
| Strength ≥ 75 且两人业线 Good / Neutral | 健康约定或共同离岛 |
| Strength 40-74 | 保持牵挂但未来不确定 |
| Strength < 40 且关键误解未解除 | 分离或错过 |

### 9.2 陈海生被顾沉舟拉拢业线

初始状态：

```text
KarmaLineId: karma_chen_haisheng_pressure
OwnerNpcId: chen_haisheng
Progress: 0
RiskState: Medium
EndingBias: Unknown
```

节点示例：

| 进度 | 节点 | 说明 |
|---:|---|---|
| 0% | 经济压力显现 | 陈海生抱怨寺庙和生活压力 |
| 25% | 与顾沉舟第一次接触 | 顾沉舟提供“新信仰”说法 |
| 50% | 家庭冲突升级 | 陈远舟和苏婉受影响 |
| 75% | 夜间聚会 | 陈海生可能参与邪教集会 |
| 100% | 选择救赎或投向邪教 | 写入角色结局 flag |

干预示例：

| 玩家行为 | 影响 |
|---|---|
| 对陈海生托梦 | 提高怀疑顾沉舟的概率 |
| 对陈海生赐福 | 降低家庭冲突升级概率 |
| 遮蔽许明川调查线索 | 顾沉舟拉拢进度更快 |
| 显灵警示酒屋或山林 | 阻止一次高危聚会 |

## 13. 边界情况

| 情况 | 处理规则 |
|---|---|
| 同一事件同时改变多条缘线 | 按事件结果表顺序写入，每条变化都进入结算摘要 |
| 缘线强度超过范围 | 裁剪到 0-100 |
| 关系类型冲突 | 使用更高优先级类型，或保留主类型并写入隐藏 flag |
| 业线节点条件未满足 | 节点保持 Locked 或 Hidden |
| 业线节点被跳过 | 标记 Skipped，不再触发对应事件 |
| 玩家未发现关系 | 后台仍结算，但 UI 显示 Unknown 或 Hidden |
| AI 文本与数值冲突 | 以数值和节点结果为准，重生成文本 |
| 角色死亡、失踪或强硬失败 | 相关业线直接 Failed 或 Completed，并写入结局 flag |

## 15. 验收标准

### 15.1 缘线验收

- [ ] 至少能读取并显示 5 条 MVP 缘线。
- [ ] 每条缘线包含 NPC A、NPC B、类型、强度和可见性。
- [ ] 事件结算能修改缘线强度。
- [ ] 缘线强度不会低于 0 或高于 100。
- [ ] 未解锁关系不会显示完整信息。
- [ ] 关系颜色必须配合文字或图标。

### 15.2 业线验收

- [ ] 每个 MVP 核心 NPC 至少有 1 条主业线。
- [ ] 每条业线包含进度、当前节点、风险和结局倾向。
- [ ] 事件结算能推进或失败业线节点。
- [ ] 业线节点状态能在 Locked / Hidden / Active / Completed / Failed 间切换。
- [ ] 业线完成后能给结局系统提供角色结局输入。

### 15.3 干预验收

- [ ] 托梦能影响至少 1 条缘线或业线风险。
- [ ] 赐福能提高至少 1 个正向节点成功权重。
- [ ] 遮蔽灾兆能制造至少 1 个负向隐藏 flag。
- [ ] 强化观察能提升关系或业线可见性。

### 15.4 MVP 竖切验收

MVP 竖切至少应能完成：

```text
查看林潮音与陈远舟缘线 → 触发青春线事件 → 使用托梦或赐福 → 推进时间 → 缘线强度变化 → 林潮音或陈远舟业线节点变化 → 结算摘要显示变化原因
```
