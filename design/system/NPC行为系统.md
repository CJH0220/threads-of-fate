# NPC 行为系统

> 文档状态：初版  
> 更新日期：2026-06-12  
> 用途：定义 NPC 在时间、地点、状态、事件和玩家干预之间的最小可实现行为规则

---

## 1. 系统目标

NPC 行为系统负责让归潮镇居民在每天早 / 午 / 晚的时间结构中拥有可预测、可触发、可结算的行为。

本系统要解决的问题是：

- NPC 当前在哪里？
- NPC 当前处于什么状态？
- NPC 为什么会参与某个事件？
- 玩家干预后 NPC 行为如何变化？
- NPC 行为如何影响事件、资源、缘线、业线和结局？

MVP 阶段不追求自由 Agent 模拟，而是使用规则、日程、状态标签和事件结果驱动 NPC 行为。

---

## 2. 设计原则

| 原则 | 说明 |
|---|---|
| 规则驱动优先 | NPC 行为由配置表和系统规则决定，不由 AI 自由决定 |
| 低成本可测试 | 每个 NPC 在每个时段的位置和状态应可被测试用例验证 |
| 事件优先 | NPC 行为服务于事件触发和结算，不做无意义复杂模拟 |
| 状态轻量化 | 使用状态标签和少量数值表达行为变化 |
| AI 只润色文本 | AI 可根据状态生成描述，但不能改变 NPC 行为结果 |
| MVP 聚焦少量角色 | 先实现少数关键 NPC 的完整闭环，再扩展到 14 人 |

### 3.1 MVP 完整行为 NPC

| NPC | 用途 | MVP 行为重点 |
|---|---|---|
| 林潮音 | 巫女线核心、青春线核心 | 学校、寺庙、咖啡店、沙滩之间移动；受梦境、灵力、寺庙关注影响 |
| 陈远舟 | 离岛梦想线、青春线核心 | 学校、书店、沙滩、海鲜店、港口之间移动；受学业、家庭和潮音关系影响 |
| 陈海生 | 家庭线、邪教易感角色 | 海鲜店、港口、酒屋之间移动；受经济压力、家庭矛盾和顾沉舟接触影响 |
| 顾沉舟 | 邪教传播线核心反派 | 港口、酒屋、广场、山林之间移动；主动接触高压力 NPC |
| 慧圆 | 寺庙腐败与巫女线推手 | 寺庙、酒屋、寺庙周边之间移动；推动香火和巫女线事件 |

### 3.2 MVP 背景行为 NPC

其他 NPC 在 MVP 中可以作为：

- 事件参与者
- 地点背景角色
- 结算文本中的提及对象
- 后续版本解锁的行为主体

它们不需要完整日程模拟，但需要有基础 ID、默认地点和可参与事件。

## 4. 玩家可见信息

玩家不会看到 NPC 行为系统的全部规则，但能通过 UI 看到足够做判断的信息。

| 信息 | 显示方式 | 示例 |
|---|---|---|
| 当前地点 | 人物面板、地图标记 | 林潮音：学校 |
| 当前状态 | 文本标签或图标 | 学业压力高、近日噩梦频繁 |
| 风险提示 | 模糊提示 | 似乎正被某人关注 |
| 可干预窗口 | 事件面板显示 | 可托梦、可赐福 |
| 近期变化 | 结算摘要 / 事件历史 | 陈海生对寺庙信任下降 |
| 业线进度 | 简化进度或阶段文本 | 离岛计划：初步萌芽 |

MVP 阶段不显示完整行为公式、精确权重和所有隐藏状态。

## 5. 核心概念

### 5.1 日程

日程决定 NPC 在没有特殊事件打断时，每天早 / 午 / 晚默认出现的位置。

```text
NpcId + DayRange + TimeSlot → LocationId
```

### 5.2 状态标签

状态标签表达 NPC 当前心理、身体、社会关系或剧情风险。

示例：

```text
normal
stressed
suspicious
influenced
blessed
dream_warned
injured
missing
high_risk
cult_contacted
```

### 5.3 行为倾向

行为倾向用于影响事件触发权重，而不是让 NPC 自由生成行为。

| 行为倾向 | 说明 | 可能影响 |
|---|---|---|
| Study | 倾向学习和升学 | 学校、书店事件权重上升 |
| Faith | 倾向寺庙和信仰 | 寺庙事件权重上升 |
| Escape | 倾向离岛 | 港口、沙滩、书店事件权重上升 |
| FamilyPressure | 受家庭压力影响 | 海鲜店、住所、争吵事件权重上升 |
| CultInfluenced | 受邪教诱导 | 酒屋、山林、广场事件权重上升 |
| Investigate | 倾向调查 | 港口、派出所、诊所事件权重上升 |

### 5.4 记忆摘要

记忆摘要是给事件文本、UI 摘要和后续结算使用的短文本状态，不承载复杂自然语言推理。

示例：

```text
林潮音最近连续梦见海水漫过土地庙。
陈远舟记得玩家曾在考试前赐福。
陈海生已经与顾沉舟有过一次夜间接触。
```

## 6. 每时段行为流程

每次时间推进后，NPC 行为按以下顺序处理：

```text
1. 读取当前 Day / TimeSlot
2. 根据日程规则计算默认地点
3. 应用强制事件位置覆盖
4. 应用状态标签对地点和事件权重的修正
5. 生成当前地点 NPC 列表
6. 计算可触发事件
7. 玩家观察或选择干预
8. 事件结算
9. 写入 NPC 状态、关系、业线和记忆摘要
10. 写入事件历史
```

---

## 7. 日程规则

### 7.1 默认日程优先级

| 优先级 | 来源 | 说明 |
|---|---|---|
| 1 | 强制事件 | 锚点事件、关键事件、Game Over 事件可强制 NPC 出现在指定地点 |
| 2 | 状态覆盖 | 受伤、失踪、高风险等状态可覆盖普通日程 |
| 3 | 玩家干预 | 牵引机缘、托梦、赐福等可临时提高某地点出现概率 |
| 4 | 普通日程 | NPC 默认每日时段地点 |
| 5 | 兜底地点 | 没有命中规则时使用初始地点第一项 |

### 7.2 MVP 日程示例

| NPC | 早 | 午 | 晚 |
|---|---|---|---|
| 林潮音 | 学校 | 咖啡店 / 寺庙 | 沙滩 / 寺庙 |
| 陈远舟 | 学校 | 书店 / 海鲜店 | 沙滩 / 港口 |
| 陈海生 | 港口 | 海鲜店 | 酒屋 / 海鲜店 |
| 顾沉舟 | 港口 | 广场 / 酒屋 | 酒屋 / 山林 |
| 慧圆 | 寺庙 | 寺庙 | 酒屋 / 寺庙周边 |

MVP 阶段允许使用固定日程加少量条件覆盖，不需要复杂路径寻路。

## 8. NPC 状态规则

### 8.1 通用状态

| 状态 ID | 中文名 | 说明 | 常见来源 |
|---|---|---|---|
| normal | 正常 | 默认状态 | 初始状态或风险解除 |
| stressed | 压力 | 学业、家庭、经济或灾兆造成压力 | 事件结算、周阶段推进 |
| suspicious | 怀疑 | NPC 对异常、邪教或他人行为产生怀疑 | 调查事件、托梦 |
| influenced | 受影响 | 被他人、邪教或强事件影响 | 顾沉舟接触、邪教事件 |
| blessed | 受赐福 | 获得短期正向修正 | 赐福 |
| dream_warned | 被托梦警示 | 获得模糊预警 | 托梦 |
| injured | 受伤 | 身体受损，行动受限 | 突发事件 |
| missing | 失踪 | 暂时不可见，可能触发搜寻事件 | 高风险事件 |
| high_risk | 高危 | 接近不幸结局或关键失败 | 多次负面事件累计 |

### 8.2 状态持续时间

状态可以有持续时间。

| 持续类型 | 说明 | 示例 |
|---|---|---|
| Instant | 只在结算时使用 | 本次事件资源变化 |
| TimeSlot | 持续一个时段 | 赐福在本时段有效 |
| Day | 持续到当天结束 | 学业焦虑 |
| Week | 持续到本周结算 | 对寺庙怀疑 |
| Permanent | 持续到被事件解除 | 被邪教接触过 |

---

## 9. 事件参与规则

NPC 参与事件分为四种角色。

| 参与类型 | 说明 | 示例 |
|---|---|---|
| 主参与者 | 事件的核心对象，状态必然变化 | 林潮音在巫女候选事件中 |
| 次参与者 | 受事件影响，但不是主要行动者 | 陈远舟在潮音被推为巫女时 |
| 旁观者 | 用于提供线索、见证或后续记忆 | 叶可可听到争吵 |
| 触发者 | 主动推动事件发生的人 | 顾沉舟接触陈海生 |

事件触发时，系统应检查：

- NPC 是否在该地点或可被强制移动到该地点
- NPC 当前状态是否满足事件条件
- NPC 是否被其他高优先级事件占用
- 玩家干预是否改变了事件触发权重
- 事件是否已经发生过或被阻断

---

## 10. 玩家干预对 NPC 行为的影响

| 干预 | 对 NPC 行为的影响 |
|---|---|
| 托梦 | 添加 `dream_warned` 状态，提高 NPC 避开风险或注意异常的概率 |
| 赐福 | 添加 `blessed` 状态，提高正向事件结果权重 |
| 牵引机缘 | 临时提高目标 NPC 到指定地点或遇见指定 NPC 的概率 |
| 遮蔽灾兆 | 降低 NPC 发现风险的概率，可能提高阴德或邪线倾向 |
| 显灵警示 | 对地点或多人添加警示效果，降低高危事件成功率 |
| 强化观察 | 不改变 NPC 行为，只提高玩家看到状态和风险提示的清晰度 |

干预不能直接替 NPC 做选择，只能改变状态、权重、可见信息或事件结果修正。

## 11. 记忆摘要规则

每个 NPC 至少保存以下摘要字段：

```text
RecentMemorySummary
ImportantFlags
LastInterventionReceived
KnownRelationshipChanges
RiskNotes
```

### 11.1 写入时机

| 时机 | 写入内容 |
|---|---|
| 事件结算后 | 事件结果、状态变化、关系变化 |
| 玩家干预后 | 干预类型、对象、是否产生明显效果 |
| 周结算后 | 本周关键变化摘要 |
| 结局结算前 | 角色最终业线摘要 |

### 11.2 可见性

记忆摘要分为玩家可见和系统隐藏两类。

| 类型 | 用途 | 示例 |
|---|---|---|
| Visible | 人物面板、事件历史、结算摘要 | “陈远舟最近更坚定地想离岛。” |
| Hidden | 触发条件、结局判定、AI 上下文 | `cult_contact_count >= 2` |

## 14. MVP 示例流程

### 14.1 Day 1 早

| NPC | 默认地点 | 状态 | 可触发内容 |
|---|---|---|---|
| 林潮音 | 学校 | `normal`, `dream_sensitive` | 噩梦后精神不佳的观察提示 |
| 陈远舟 | 学校 | `normal`, `study_pressure` | 备考和离岛目标提示 |
| 陈海生 | 港口 | `stressed`, `family_pressure` | 经济压力和港口抱怨 |
| 顾沉舟 | 港口 | `hidden_arrival` | 外来者上岛伏笔 |
| 慧圆 | 寺庙 | `temple_manager` | 香火不足和巫女线伏笔 |

### 14.2 玩家观察

玩家在主地图上看到：

- 学校有林潮音、陈远舟
- 港口有陈海生和陌生游客迹象
- 寺庙有慧圆和香火压力提示

### 14.3 玩家干预

如果玩家对林潮音使用托梦：

```text
林潮音添加 dream_warned 状态
林潮音相关高风险事件的警觉修正提高
人物面板显示：她似乎对昨夜的梦格外在意
```

如果玩家对陈远舟使用赐福：

```text
陈远舟添加 blessed 状态
升学 / 学习相关事件正向结果权重提高
结算摘要显示：陈远舟今天复习格外顺利
```

## 15. 边界情况

| 情况 | 处理规则 |
|---|---|
| NPC 同时被多个事件需要 | 高优先级事件先占用，低优先级事件延后或取消 |
| NPC 状态互相冲突 | 按状态优先级处理，`missing`、`injured`、`high_risk` 优先于普通日程 |
| 地点未解锁 | NPC 不进入该地点，使用兜底地点 |
| 日程没有匹配规则 | 使用 DefaultLocations 的第一个地点 |
| 玩家干预目标不可用 | UI 显示不可干预原因，不消耗神力 |
| NPC 失踪 | 地图不显示精确位置，只显示最后出现地点或线索 |
| 锚点事件需要 NPC 出场 | 允许强制移动，但必须写入历史或结算说明 |
| AI 文本与规则冲突 | 以规则结果为准，丢弃或重生成 AI 文本 |

## 16. 调参项

| 参数 | 说明 | MVP 建议 |
|---|---|---|
| `MaxEventsPerTimeSlot` | 每时段最多触发事件数 | 1-3 |
| `NpcScheduleRandomness` | 普通日程随机浮动比例 | 0-20% |
| `InterventionLocationBoost` | 牵引机缘对地点出现权重的提升 | +30% |
| `DreamWarningRiskReduction` | 托梦对风险事件的降低幅度 | -10% 到 -25% |
| `BlessingPositiveOutcomeBoost` | 赐福对正向结果的提升 | +10% 到 +25% |
| `HighRiskThreshold` | NPC 进入高危状态的风险值 | 70 |
| `CultInfluenceThreshold` | 被邪教明显影响的阈值 | 50 |

## 17. 验收标准

### 17.1 规则验收

- [ ] 每个 MVP NPC 在 Day 1 早 / 午 / 晚都有可计算地点。
- [ ] 每个 MVP NPC 至少有一个初始状态标签。
- [ ] NPC 状态能被事件结算修改。
- [ ] NPC 状态能影响事件触发或结果权重。
- [ ] 玩家干预能写入 NPC 状态或记忆摘要。
- [ ] 同一时段 NPC 不会同时参与两个互斥事件。
- [ ] 地点未解锁时 NPC 不会出现在该地点。

### 17.2 UI 验收

- [ ] 人物面板能显示 NPC 当前地点。
- [ ] 人物面板能显示至少 1 条玩家可见状态。
- [ ] 事件面板能显示参与 NPC。
- [ ] 时间推进结算能显示 NPC 状态变化摘要。
- [ ] 失踪、受伤、高危等特殊状态有明确 UI 表达。

### 17.3 数据验收

- [ ] `NPC基础表.csv` 中 P0 NPC 可映射到行为配置。
- [ ] `地点配置表.csv` 中 P0 地点可被 NPC 日程引用。
- [ ] 所有 `NpcId` 与内容文档、事件配置表一致。
- [ ] 所有 `LocationId` 与地点配置表一致。

### 17.4 MVP 竖切验收

MVP 竖切至少应能完成：

```text
Day 1 早 → 查看林潮音 / 陈远舟 / 陈海生位置 → 触发一个事件 → 对一个 NPC 使用托梦或赐福 → 推进到下一时段 → 显示 NPC 状态和资源变化
```
