# 事件系统

> 所属项目：《命运的织线》  
> 文档类型：系统策划案  
> 当前版本：v0.1 草案  
> 适用阶段：系统设计 / 研发拆解前置文档

---

## 1. 系统目标

事件系统负责把「时间、地点、NPC 状态、缘线、业线、玩家干预」转化为可播放、可结算、可记录的游戏内容。

它要解决三个问题：

1. **让小镇动起来**：NPC 不只是静态属性，而是在每天早/午/晚根据状态触发事件。
2. **让玩家干预有反馈**：玩家的赐福、托梦、遮蔽信息、引导机缘等行为，会改变事件触发概率、事件分支和结算结果。
3. **让涌现叙事可控**：AI 可以参与文本变化，但事件结构、触发条件、结算结果必须由系统规则约束。

---

## 2. 玩家能看到什么

玩家不会直接看到完整事件公式，但能看到足够做判断的信息。

### 2.1 地图事件标记

主地图上显示事件标记：

| 标记类型 | 含义 | 示例 |
|---|---|---|
| 日常事件 | 普通生活片段，风险低 | 买早餐、闲聊、散步 |
| 关键事件 | 会推进业线、缘线或主线 | 告白、考试、调查、争吵 |
| 突发事件 | 高风险、高影响力事件 | 火灾、失踪、事故、邪教集会 |
| 命运节点 | 影响角色长期结局的事件 | 是否离岛、是否成为巫女 |
| 危险征兆 | 未来高风险事件的前兆 | 寺庙电线老化、港口夜船记录 |

### 2.2 事件预览信息

点击事件标记后，玩家可查看：

| 信息 | 显示方式 |
|---|---|
| 事件名称 | 明文显示 |
| 参与 NPC | 明文显示 |
| 地点 | 明文显示 |
| 当前时段 | 明文显示 |
| 风险等级 | 低 / 中 / 高 / 致命 |
| 可能影响 | 模糊提示，如“可能影响香火”“可能改变两人关系” |
| 可干预方式 | 显示当前可用神力技能 |
| 成功概率 | 不显示精确数值，显示“容易 / 普通 / 困难 / 几乎不可能” |

### 2.3 事件推进面板

事件发生时进入事件推进面板：

- 标题：事件名称
- 参与角色：NPC 列表
- 正文：事件描述与对话
- 玩家干预点：可选的神力技能或观察选项
- 结算反馈：资源变化、关系变化、业线变化、后续线索

## 3. 事件类型

### 3.1 日常事件

日常事件用于表现小镇生活和积累轻量状态变化。

| 属性 | 说明 |
|---|---|
| 频率 | 高 |
| 影响力 | 低 |
| 是否可被 AI 润色 | 是 |
| 是否推动主线 | 通常否 |
| 是否进入历史记录 | 只记录摘要 |

示例：

- 小美在咖啡厅复习
- 老王在港口修船
- 叶可可在店里和客人聊天

### 3.2 关键事件

关键事件用于推进 NPC 业线、缘线或三条核心主线。

| 属性 | 说明 |
|---|---|
| 频率 | 中 |
| 影响力 | 高 |
| 是否可被 AI 润色 | 是，但结果不可自由生成 |
| 是否推动主线 | 是 |
| 是否进入历史记录 | 必须记录 |

示例：

- 陈远舟向林潮音提出离岛计划
- 周行知发现夜船记录
- 许明川获得诊所异常线索
- 陈海生第一次接触顾沉舟

### 3.3 突发事件

突发事件用于制造强风险和不可逆后果。

| 属性 | 说明 |
|---|---|
| 频率 | 低 |
| 影响力 | 极高 |
| 是否可被 AI 润色 | 仅限描述，不能改变结果 |
| 是否推动主线 | 通常是 |
| 是否进入历史记录 | 必须记录 |
| 是否可导致游戏结束 | 可以 |

示例：

- 寺庙火灾
- 岛民集体叛教
- 异教徒公开集会
- 港口失踪事件

### 3.4 锚点事件

锚点事件是保证主线框架成立的固定事件。

| 属性 | 说明 |
|---|---|
| 触发方式 | 固定天数或固定周阶段 |
| 是否可跳过 | 不可跳过 |
| 是否可改变结果 | 部分可改变 |
| 作用 | 保证 60 天主线结构不散 |

示例：

- Day 1：天界绩效通知
- 第 2 周：顾沉舟上岛
- 第 5 周：土地公梦见地震/海啸
- Day 60：天灾结算

### 3.5 机会事件

机会事件是由玩家干预或 NPC 状态临时生成的事件。

| 属性 | 说明 |
|---|---|
| 来源 | 玩家干预、缘线变化、业线状态、地点状态 |
| 影响力 | 低到高 |
| 是否可重复 | 通常不可重复 |
| 作用 | 制造涌现感和“我改变了命运”的反馈 |

示例：

- 玩家赐福后，小美在书店遇到陈远舟
- 玩家遮蔽信息后，许明川错过关键证据
- 玩家引导机缘后，苏婉提前发现陈海生异常

## 4. 事件数据结构

每个事件至少需要以下字段。

```text
EventId
EventName
EventType
WeekRange
DayRange
TimeSlot
LocationId
ParticipantNpcIds
RequiredConditions
BlockedConditions
TriggerWeight
RiskLevel
InterventionWindows
ResolutionRules
OutcomeEffects
FollowUpEventIds
AiTextPolicy
HistoryRecordPolicy
```

### 4.1 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| EventId | string | 唯一 ID，例如 `event_cult_001` |
| EventName | string | 显示名称 |
| EventType | enum | Daily / Key / Emergency / Anchor / Opportunity |
| WeekRange | int range | 可出现周数 |
| DayRange | int range | 可出现天数 |
| TimeSlot | enum | Morning / Afternoon / Night / Any |
| LocationId | string | 发生地点 |
| ParticipantNpcIds | string[] | 参与 NPC |
| RequiredConditions | condition[] | 必须满足的触发条件 |
| BlockedConditions | condition[] | 满足则禁止触发 |
| TriggerWeight | int | 触发权重 |
| RiskLevel | enum | Low / Medium / High / Fatal |
| InterventionWindows | intervention[] | 可干预窗口 |
| ResolutionRules | rule[] | 结算规则 |
| OutcomeEffects | effect[] | 结果影响 |
| FollowUpEventIds | string[] | 后续事件 |
| AiTextPolicy | enum | None / FlavorOnly / DialogueAllowed / SummaryAllowed |
| HistoryRecordPolicy | enum | None / Summary / Full |

## 5. 事件触发条件

事件触发由四类条件共同决定。

### 5.1 时间条件

| 条件 | 示例 |
|---|---|
| 天数 | Day 8-14 |
| 周阶段 | 第 2 周“外来者入潮” |
| 时段 | 夜晚 |
| 是否周结算前 | 第 7 天夜晚 |

### 5.2 地点条件

| 条件 | 示例 |
|---|---|
| NPC 当前地点 | 林潮音在寺庙 |
| 地点状态 | 寺庙安全隐患 > 50 |
| 地点标签 | 港口、学校、诊所、广场 |
| 地点是否被破坏 | 寺庙未烧毁 |

### 5.3 NPC 条件

| 条件 | 示例 |
|---|---|
| 属性阈值 | 林潮音信仰 ≥ 6 |
| 情绪状态 | 陈海生怨气 ≥ 70 |
| 业线进度 | 陈远舟离岛线 ≥ 40% |
| 记忆标签 | 许明川已获得“夜船线索” |
| 阵营倾向 | 顾沉舟信任度 ≥ 50 |

### 5.4 关系与世界条件

| 条件 | 示例 |
|---|---|
| 缘线强度 | 林潮音 → 陈远舟爱情线 ≥ 60 |
| 邪教传播值 | 邪教传播 ≥ 40 |
| 香火 | 香火 < 80 |
| 阴德 / 阳德 | 阴德 ≥ 50 |
| 前置事件 | `event_port_003` 已完成 |
| 阻断事件 | 顾沉舟已被逮捕则不触发邪教集会 |

## 6. 事件选择规则

每个时段系统按以下顺序选择事件：

1. 检查是否有必须触发的锚点事件。
2. 检查是否有强制触发的突发事件。
3. 为每个地点生成候选事件池。
4. 根据 NPC 状态、地点状态、业线进度计算触发权重。
5. 排除被 BlockedConditions 阻断的事件。
6. 从候选池中抽取事件。
7. 标记参与 NPC，避免同一 NPC 在同一时段参与多个冲突事件。
8. 生成地图事件标记，等待玩家观察或推进时间。

### 6.1 同时事件冲突规则

如果多个事件需要同一 NPC：

1. 锚点事件优先级最高。
2. 强硬不幸结局相关事件优先。
3. 主线关键事件优先。
4. 业线进度更接近终点的事件优先。
5. 触发权重更高的事件优先。
6. 低优先级事件延后到下一个可用时段或取消。

## 7. 玩家干预点

玩家不能直接选择事件结果，只能在事件前或事件中影响条件。

### 7.1 事件前干预

| 干预 | 效果 |
|---|---|
| 托梦 | 给 NPC 添加短期记忆或情绪倾向 |
| 赐福 | 提高某次判定的成功率 |
| 引导机缘 | 提高两个 NPC 同地点相遇概率 |
| 遮蔽信息 | 降低某线索被发现概率 |
| 显灵警示 | 提高 NPC 对危险事件的回避概率 |

### 7.2 事件中干预

| 干预 | 效果 |
|---|---|
| 增加命运硬币 | 提高正向结算概率 |
| 减少命运硬币 | 提高负向或失败概率 |
| 强化情绪 | 放大当前 NPC 情绪 |
| 平息情绪 | 降低冲突升级概率 |
| 改变小物件 | 改变现场细节，触发不同分支 |

### 7.3 干预限制

- 每次干预消耗神力。
- 干预失败不默认退款。
- 高风险事件可能需要多次前置干预才能改变结果。
- 强硬不幸结局一旦触发，通常不可挽回。
- 玩家不能直接让 NPC 做违背自身动机的事。

## 8. 事件结算规则

事件结算分为三层。

### 8.1 固定结果

用于锚点事件和部分主线事件。

示例：

- Day 1 必定收到天界绩效通知。
- Day 60 必定发生天灾结算。
- 顾沉舟第 2 周必定上岛。

### 8.2 条件分支结果

根据状态阈值进入不同分支。

示例：

```text
如果 林潮音信仰 >= 70 且 陈远舟缘线 >= 60：
    林潮音愿意与陈远舟认真讨论离岛
否则如果 林潮音信仰 >= 70 且 邪教影响 >= 50：
    林潮音更倾向接受巫女身份
否则：
    林潮音保持犹豫
```

### 8.3 命运硬币结果

用于不确定事件。

基础流程：

1. 事件定义需求值，例如“说服难度 6”。
2. NPC 根据相关属性获得基础硬币数。
3. 玩家干预可以增加或减少硬币。
4. 抛硬币得到成功数。
5. 成功数达到需求值则进入成功分支，否则失败。
6. 大成功或大失败可触发额外后果。

示例：

```text
事件：陈远舟说服林潮音离岛
需求：说服成功数 >= 4
基础硬币：
- 陈远舟心智 +1
- 陈远舟魅力 +1
- 二人爱情缘线强度 >= 60：+1
- 林潮音巫女压力 >= 70：-1
玩家赐福：+1
结算：
- 成功数 >= 5：林潮音主动考虑离岛，业线推进 +20%
- 成功数 4：林潮音动摇，业线推进 +10%
- 成功数 2-3：无明显变化
- 成功数 <= 1：林潮音更抗拒离岛，信仰倾向 +10
```

## 9. 事件结果影响

事件结果可影响以下对象。

| 对象 | 可变内容 |
|---|---|
| NPC | 情绪、属性、记忆、标签、业线进度 |
| 缘线 | 性质、强度、活跃值 |
| 业线 | 节点完成、进度增减、结局锁定 |
| 地点 | 安全隐患、损坏状态、可用性 |
| 世界 | 邪教传播值、香火、阴德、阳德 |
| 玩家 | 神力、技能解锁、称号倾向 |
| 后续事件 | 解锁、阻断、提前、延后 |

### 9.1 历史记录

关键事件、突发事件、命运节点必须写入事件历史。

记录内容：

```text
Day
TimeSlot
EventId
EventName
Participants
Location
PlayerInterventions
OutcomeSummary
StateChanges
UnlockedFollowUps
```

## 10. AI 生成边界

AI 可以增强表达，但不能决定核心规则。

### 10.1 允许 AI 生成

| 内容 | 说明 |
|---|---|
| 对话润色 | 在角色性格和事件结果约束下生成 |
| 情绪描写 | 描述 NPC 的表情、语气、动作 |
| 后日谈文本 | 根据最终状态生成个性化总结 |
| 日常事件变体 | 不影响核心系统的小文本变化 |

### 10.2 禁止 AI 自由生成

| 内容 | 原因 |
|---|---|
| 事件是否触发 | 必须由规则系统决定 |
| 事件结果 | 必须由条件和命运硬币决定 |
| 数值变化 | 必须由配置表决定 |
| 新增关键设定 | 避免破坏世界观 |
| 敏感内容细节 | 需要合规边界 |
| 强硬不幸结局 | 必须由预设规则触发 |

### 10.3 敏感内容处理

涉及暴力、死亡、邪教、未成年、性侵害等内容时：

- 使用线索、传闻、状态变化表达。
- 不写露骨过程。
- 不把伤害描写作为爽点。
- 结算重点放在后果、创伤、救援、调查和小镇反应上。

## 11. 示例事件

### 11.1 日常事件：咖啡厅复习

| 字段 | 内容 |
|---|---|
| EventId | `event_daily_cafe_study_001` |
| 类型 | 日常事件 |
| 时间 | Day 1-14，下午 |
| 地点 | 咖啡厅 |
| 参与 NPC | 小美、陈远舟 |
| 条件 | 两人未发生严重冲突 |
| 结果 | 小幅增加熟悉度，有概率解锁学习相关对话 |

结算：

- 两人缘线强度 +1
- 小美学习压力 +1
- 若玩家引导机缘：额外触发一次对话

### 11.2 关键事件：夜船记录

| 字段 | 内容 |
|---|---|
| EventId | `event_port_nightboat_001` |
| 类型 | 关键事件 |
| 时间 | 第 3 周夜晚 |
| 地点 | 港口 |
| 参与 NPC | 周行知 |
| 条件 | 顾沉舟已上岛 |
| 风险 | 中 |
| 结果 | 解锁邪教调查线索 |

分支：

- 若周行知警觉 ≥ 6：获得“夜船记录”记忆标签。
- 若港口被遮蔽信息：线索延后。
- 若邪教传播 ≥ 50：周行知被警告，后续调查难度提高。

### 11.3 突发事件：寺庙火灾

| 字段 | 内容 |
|---|---|
| EventId | `event_temple_fire_001` |
| 类型 | 突发事件 |
| 时间 | 第 6-8 周 |
| 地点 | 寺庙 |
| 参与 NPC | 陈海生、慧圆、苏婉 |
| 条件 | 寺庙安全隐患高，陈海生怨气高，邪教影响高 |
| 风险 | 致命 |
| 结果 | 可能触发强硬不幸结局 |

分支：

- 若寺庙安全检查已完成：火灾概率大幅降低。
- 若陈海生已救赎：他会主动预警或救人。
- 若玩家忽视多次危险征兆：火灾触发后不可挽回。
- 若火灾发生且寺庙彻底损毁：进入强硬不幸结局。

## 12. 边界情况

| 情况 | 处理 |
|---|---|
| 事件需要的 NPC 已死亡或离岛 | 事件禁用或替换为后续传闻 |
| 两个主线事件争用同一 NPC | 按优先级选择一个，另一个延后 |
| 玩家连续阻断主线事件 | 锚点事件仍会推进，但结果变差 |
| AI 生成内容与规则冲突 | 规则优先，重新生成文本 |
| 事件结果导致数值越界 | 按系统上下限截断 |
| 强硬不幸结局触发 | 立即进入结局流程，不再生成普通事件 |
| 玩家没有神力 | 可观察事件，但不能干预 |
| 关键线索被错过 | 后续可通过高成本神力查询或替代线索补救，除非标记为不可逆 |

## 13. 验收标准

事件系统完成后，应满足以下条件：

1. 系统能按天数、时段、地点和 NPC 状态生成候选事件。
2. 锚点事件能在指定时间稳定触发。
3. 事件能正确检查 RequiredConditions 和 BlockedConditions。
4. 同一 NPC 不会在同一时段参与多个冲突事件。
5. 玩家干预能改变事件权重、分支或命运硬币。
6. 事件结算能修改 NPC、缘线、业线、地点和世界资源。
7. 关键事件和突发事件会写入历史记录。
8. AI 只负责文本表达，不决定触发、数值和结局。
9. 敏感事件不会生成露骨描写。
10. 至少能配置并跑通以下 3 个测试事件：
    - 1 个日常事件
    - 1 个关键事件
    - 1 个突发事件
