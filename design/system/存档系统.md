# 存档系统

> 文档状态：初版  
> 更新日期：2026-06-12  
> 用途：定义保存、读取、自动保存、存档摘要、结局记录和版本兼容规则

---

## 1. 系统目标

存档系统负责保存玩家在《命运的织线》中的进度、系统状态、事件历史和结局记录，确保玩家能安全离开并继续游戏。

本系统要解决的问题是：

- 什么时候可以保存？
- 保存哪些数据？
- 存档列表显示哪些摘要？
- 读档失败或存档损坏时如何处理？
- 设置和结局记录是否跟随单个存档？
- 后续版本字段变化时如何兼容？

P0 / MVP 阶段只需要可靠的本地保存和读取，不需要云存档、多设备同步或复杂回滚。

---

## 2. 设计原则

| 原则 | 说明 |
|---|---|
| 本地优先 | MVP 使用本地文件保存，格式可为 JSON 或 Godot Resource |
| 状态完整 | 读档后必须能恢复时间、资源、NPC、事件、缘线业线和 UI 所需摘要 |
| 安全保存 | 事件结算和时间推进中不允许写入半状态 |
| 摘要轻量 | 存档列表只读取元数据，不完整加载大存档 |
| 可恢复 | 存档损坏、版本不匹配或字段缺失时必须给出可理解反馈 |
| 不剧透 | 存档摘要不能显示玩家尚未解锁的隐藏信息 |

---

## 3. MVP 范围

### 3.1 P0 必须支持

| 功能 | 说明 |
|---|---|
| 手动保存 | 从暂停菜单保存当前安全状态 |
| 自动保存 | 在关键检查点保存到自动槽 |
| 读档列表 | 展示已有存档与摘要 |
| 加载存档 | 从主菜单或暂停菜单进入存档进度 |
| 删除存档 | 二次确认后删除指定手动存档 |
| 损坏提示 | 存档不可读时显示错误并禁用加载 |
| 未保存提示 | 从暂停菜单加载其他存档或回主菜单前提示 |

### 3.2 P0 不做

- 云存档。
- 多用户账户。
- 存档分支树。
- 任意历史时段回滚。
- 存档截图缩略图自动生成。
- 存档加密和反作弊。

## 4. 存档槽规则

### 4.1 槽位数量

| 槽位 | 数量 | 用途 |
|---|---:|---|
| 手动槽 | 3 | 玩家主动保存 |
| 自动槽 | 1 | 系统自动覆盖 |
| 结局记录 | 独立全局记录 | 保存已解锁结局和回顾 |
| 设置文件 | 独立全局设置 | 保存音量、显示、辅助功能等 |

P1 可扩展为更多手动槽。

### 4.2 槽位命名

建议内部命名：

```text
manual_slot_01
manual_slot_02
manual_slot_03
autosave_slot
settings_global
ending_gallery_global
```

UI 显示：

```text
存档 01
存档 02
存档 03
自动保存
```

## 5. 保存时机

### 5.1 自动保存时机

| 时机 | 说明 |
|---|---|
| 新游戏完成后 | 第 0 夜结束，进入 Day 1 早前 |
| 第一个可操作时段开始 | 玩家第一次进入主地图 HUD |
| 每次时间推进结算完成后 | 已完成事件、资源、NPC 和 UI 摘要结算 |
| 周总结完成后 | 周阶段数据稳定后 |
| 终局结算完成后 | 写入结局记录前后均可保存 |

自动保存只能在系统处于安全状态时发生。

### 5.2 手动保存时机

玩家可在暂停菜单点击“保存游戏”。

允许保存条件：

- 当前在主地图 HUD 或安全暂停状态。
- 不在事件播放中。
- 不在时间推进结算中。
- 不在确认弹窗、结局演出或 Game Over 演出中。
- 当前存档槽可写。

禁止保存时必须灰化按钮并显示原因。

---

## 6. 不可保存状态

| 状态 | 原因 | UI 反馈 |
|---|---|---|
| 事件结算中 | 资源、NPC、缘线业线尚未稳定 | 保存按钮灰化：事件结算中 |
| 时间推进中 | 当前时段正在切换 | 保存按钮灰化：时间推进中 |
| 终局判定中 | 结局状态尚未写入 | 保存按钮灰化：终局结算中 |
| Game Over 演出中 | 失败结果已锁定 | 只允许返回主菜单或查看总结 |
| 删除确认打开 | 避免并发操作 | 等待确认结束 |
| 存档写入中 | 防止重复写入 | 显示保存中 |

## 7. 存档内容

存档由元数据和完整载荷两部分组成。

### 7.1 SaveMetadata

存档列表只读取元数据。

```text

- SaveSlotId
- SaveName
- SaveVersion
- GameVersion
- LastSavedAt
- Day
- TimeSlot
- WeekStage
- Incense
- DivinePower
- YangDe
- YinDe
- RouteHint
- MainLocationHint
- IsAutosave
- IsCorrupted
```

```text

- NpcStates
- EventHistory
- TriggeredEventFlags

- KarmaLineStates
- TownState
- EndingState
- TutorialState
- UiResumeContext
```

设置和结局图鉴默认独立于单个存档。

## 10. 结局记录

结局记录独立于单个游戏存档，用于结局回顾 / 图鉴。

```text
EndingGalleryState
- UnlockedFinalEndingIds
- UnlockedCharacterEndingIds
- UnlockedMajorEventIds
- NewlyUnlockedIds
- FirstUnlockedAt
- LastUnlockedAt
```

写入时机：

- Game Over 结算完成后。
- Day 60 最终路线结算完成后。
- 角色结局展示完成后。

未解锁结局不得在存档摘要中剧透。

## 11. 保存流程

### 11.1 手动保存流程

```text
玩家点击保存游戏
→ 检查是否可保存
→ 生成 SaveMetadata
→ 生成 SavePayload
→ 写入临时文件
→ 校验临时文件可读
→ 替换目标槽位
→ 标记未保存状态为 false
→ 显示 Toast：命运已记录
```

### 11.2 自动保存流程

```text
系统进入自动保存检查点
→ 检查当前状态稳定
→ 生成 autosave_slot
→ 写入临时文件
→ 校验成功后覆盖自动槽
→ 若失败，记录错误但不中断游戏
```

自动保存失败不应让游戏流程中断，但必须可在日志或调试信息中追踪。

---

## 12. 读取流程

```text
玩家打开加载游戏
→ 读取所有 SaveMetadata
→ 标记空槽 / 正常 / 损坏 / 版本风险
→ 玩家选择存档
→ 点击加载
→ 如来自暂停菜单，提示当前未保存进度可能丢失
→ 读取 SavePayload
→ 校验版本与必要字段
→ 恢复 Time / Resource / NPC / Event / Bond / Karma / Ending 状态
→ 进入主地图 HUD 或存档记录的恢复界面
```

读取失败时：

- 不改变当前游戏状态。
- 停留在加载界面。
- 显示错误提示。
- 禁用该槽位加载按钮。

---

## 13. 删除流程

删除存档属于危险操作，必须二次确认。

```text
玩家点击删除
→ 弹出确认框
→ 默认焦点在取消
→ 玩家确认删除
→ 删除目标槽位文件
→ 刷新存档列表
```

自动槽是否允许删除可作为 P1 设置；P0 可不提供删除自动槽入口。

---

## 14. 版本兼容

### 14.1 版本字段

每个存档必须包含：

```text
SaveVersion
GameVersion
CreatedAt
LastSavedAt
```

### 14.2 兼容策略

| 情况 | 处理 |
|---|---|
| SaveVersion 完全匹配 | 正常读取 |
| 小版本字段新增 | 使用默认值补齐 |
| 关键字段缺失 | 标记为版本风险或损坏 |
| 大版本不兼容 | 禁用加载，提示版本不兼容 |
| JSON / Resource 解析失败 | 标记损坏 |

MVP 可以只做同版本读取和损坏提示；字段迁移工具可后置。

| 系统 | 依赖关系 |
|---|---|
| 时间系统 | 保存 Day、TimeSlot、WeekStage、终局锁定状态 |

## 17. 边界情况

| 情况 | 处理规则 |
|---|---|
| 保存中退出游戏 | 禁用退出按钮或等待写入完成 |
| 写入失败 | 不覆盖旧存档，显示保存失败 |
| 临时文件写入成功但替换失败 | 保留旧存档，提示失败 |
| 读取存档时当前进度未保存 | 二次确认后再加载 |
| 存档损坏 | 卡片标红，加载禁用，可删除 |
| 设置损坏 | 使用默认设置并提示已恢复默认 |
| 结局记录损坏 | 不影响主存档读取，可重建空记录 |
| 自动保存失败 | 不阻断游戏，只记录错误和提示可选 |
| 终局后保存 | 保存最终总结和结局记录，不允许回到终局前状态，除非另有设计 |

## 18. 调参项

| 参数 | 默认 | 说明 |
|---|---:|---|
| `ManualSaveSlotCount` | 3 | 手动存档槽数量 |
| `AutoSaveSlotCount` | 1 | 自动存档槽数量 |
| `EnableAutosaveAfterTimeAdvance` | true | 时间推进后自动保存 |
| `EnableAutosaveAfterWeekSummary` | true | 周总结后自动保存 |
| `EnableAutosaveAfterEnding` | true | 结局后自动保存 |
| `AllowDeleteAutosave` | false | 是否允许删除自动槽 |
| `SaveVersion` | 1 | 当前存档版本 |

## 19. 验收标准

### 19.1 保存验收

- [ ] 玩家能从暂停菜单手动保存。
- [ ] 保存成功后显示 Toast。
- [ ] 保存失败不覆盖旧存档。
- [ ] 时间推进或事件结算中保存按钮不可用。
- [ ] 自动保存能在时间推进完成后写入自动槽。

### 19.2 读取验收

- [ ] 主菜单和暂停菜单都能打开加载游戏界面。
- [ ] 存档列表能显示天数、时段、香火、神力、路线倾向和最近保存时间。
- [ ] 空槽不可加载。
- [ ] 损坏存档显示错误状态且不可加载。
- [ ] 加载成功后恢复到主地图 HUD。
- [ ] 从暂停菜单加载其他存档前会提示未保存进度可能丢失。

### 19.3 数据验收

- [ ] 读档后时间状态一致。
- [ ] 读档后资源值一致。
- [ ] 读档后 NPC 位置和状态一致。
- [ ] 读档后事件历史不重复触发已完成事件。
- [ ] 读档后缘线和业线状态一致。
- [ ] 读档后结局记录和设置不被错误覆盖。

### 19.4 边界验收

- [ ] 删除存档必须二次确认，默认焦点在取消。
- [ ] 设置损坏时能恢复默认设置。
- [ ] 结局记录损坏不影响游戏存档读取。
- [ ] 版本不兼容存档有明确提示。
