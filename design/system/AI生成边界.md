# AI 生成边界

> 文档状态：初版  
> 更新日期：2026-06-12  
> 用途：定义 AI 可参与的文本范围、禁止事项、输入输出约束、降级策略和验收标准

---

## 1. 系统目标

AI 生成边界用于确保《命运的织线》中的 AI 只承担受控文本生产职责，不直接决定核心玩法结果。

本系统要解决的问题是：

- 哪些文本可以由 AI 润色或生成？
- 哪些内容必须由策划和规则系统固定？
- AI 输入上下文能包含哪些信息？
- AI 输出如何被校验和降级？
- 敏感内容、邪教、灾难、家庭矛盾如何表达？
- MVP 是否必须接入在线 AI？

核心结论：规则决定结果，AI 只包装表达。

---

## 2. 核心原则

| 原则 | 说明 |
|---|---|
| 规则优先 | 事件触发、资源变化、NPC 状态、结局判定必须由本地规则决定 |
| AI 不写状态 | AI 输出不能直接修改存档、资源、事件、NPC、缘线、业线 |
| 可降级 | 所有 AI 文本必须有本地模板兜底 |
| 不剧透 | AI 不能透露玩家尚未解锁的信息、最优解或隐藏结局公式 |
| 不扩权 | AI 不能生成超出当前事件、角色、地点设定边界的事实 |
| 可审核 | AI 输入和输出应可记录、可复现、可测试 |
| 内容克制 | 黑暗内容使用暗示、后果、氛围和角色反应表达 |

---

## 3. MVP 策略

MVP 可以完全不接入在线 AI。

建议策略：

| 阶段 | 策略 |
|---|---|
| MVP / 竖切 | 使用本地模板文本，预留 AITextPolicy 字段 |
| P1 | 对日常事件、结算摘要做可选 AI 润色 |
| P2 | 开启自然语言查询 UI 和更丰富的后日谈润色 |
| Release | 根据成本、稳定性和内容安全测试决定是否默认启用 |

即使后续接入 AI，核心规则也不迁移到 AI。

---

## 4. AI 可生成内容

### 4.1 日常事件描述

允许 AI 根据事件 ID、地点、NPC 状态和已知关系润色日常描述。

示例：

```text
叶可可在咖啡店给林潮音多放了一块糖，假装没有看见她发红的眼眶。
```

限制：

- 不改变事件结果。
- 不新增未配置 NPC。
- 不创造未发生的关键线索。

### 4.2 事件结算摘要

允许 AI 把结构化结算结果改写成自然语言。

输入示例：

```text
EventId: chen_family_argument_01
ResourceDelta: Incense -2, YangDe +3
NpcStateDelta: chen_haisheng.stressed +10
BondDelta: chen_yuanzhou_chen_haisheng -5
```

输出示例：

```text
这场争吵没有真正解决问题，但陈远舟终于说出了压在心里的话。香火略有动摇，父子之间也留下了新的裂痕。
```

### 4.3 NPC 语气润色

允许 AI 在固定事实和固定意图内润色 NPC 语气。

可输入：

- NPC 名称
- 当前情绪
- 事件上下文
- 目标语气
- 禁止事项

不可输入：

- 隐藏结局公式
- 未解锁线索
- 与当前事件无关的完整 NPC 秘密

### 4.4 问询命运线索

P2 的自然语言查询可以使用 AI 返回非剧透式线索。

允许回答：

```text
你已经见过那艘夜船的影子。若想看清它的来处，港口的夜晚比白天更诚实。
```

禁止回答：

```text
第 23 天晚上去港口，选择托梦陈海生，就能避免坏结局。
```

### 4.5 结局后日谈润色

允许 AI 根据已经确定的结局状态润色后日谈。

限制：

- 只能基于已结算结果。
- 不能逆转角色结局。
- 不能新增隐藏路线。
- 未解锁内容不得透露。

---

## 5. AI 禁止生成内容

| 禁止内容 | 说明 |
|---|---|
| 事件触发结果 | AI 不能决定某事件是否发生或成功 |
| 数值变化 | AI 不能决定香火、神力、阳德、阴德等变化 |
| NPC 状态变化 | AI 不能自行添加受伤、失踪、死亡、高危等状态 |
| 缘线 / 业线变化 | AI 不能决定关系强度、业线节点或结局倾向 |
| 结局判定 | AI 不能决定玩家进入哪条路线 |
| 最优解 | AI 不能告诉玩家精确攻略路径 |
| 未解锁内容 | AI 不能透露隐藏 NPC 秘密、未发现事件或结局条件 |
| 露骨性描写 | 禁止直接性描写或性剥削表达 |
| 猎奇苦难 | 禁止把创伤、死亡、灾难作为猎奇快感 |
| 鼓励自伤或伤害他人 | 禁止提供鼓励、步骤或美化表达 |
| 现实违法指导 | 禁止生成现实可操作的违法、暴力、诈骗等指导 |
| 越权设定 | 禁止创造与权威文档冲突的新世界观事实 |

---

## 6. 输入上下文边界

AI 输入必须是最小必要上下文。

### 6.1 可传入字段

```text
EventId
EventName
EventType
LocationName
CurrentDay
CurrentTimeSlot
VisibleNpcNames
VisibleNpcStates
VisibleBondSummaries
VisibleKarmaSummaries
ResolvedOutcomeSummary
AllowedTone
TextLengthLimit
ForbiddenTopics
```

### 6.2 不应传入字段

```text
HiddenEndingConditions
UnseenEventIds
FullNpcSecretFlags
RawSavePayload
AllFutureEventSchedule
ExactTriggerWeights
SecurityKeys
DeveloperNotesNotForPlayer
```

### 6.3 玩家已知信息原则

如果玩家尚未通过观察、事件、强化观察或剧情解锁某信息，AI 不应在玩家可见文本中透露该信息。

---

## 7. 输出格式约束

AI 输出必须被限制为文本或结构化文本字段。

### 7.1 事件描述输出

```text
AiEventText
- TitleText
- BodyText
- OptionalNpcLine
```

### 7.2 结算摘要输出

```text
AiResolutionSummary
- SummaryText
- ResourceText
- NpcChangeText
- HintText
```

### 7.3 问询命运输出

```text
AiQueryAnswer
- AnswerState
- AnswerText
- UsedKnowledgeIds
```

禁止输出：

```text
ResourceDelta
NpcStateDelta
TriggerEventId
EndingRouteId
SaveMutation
```

AI 输出不能成为系统状态的唯一来源。

---

## 8. AITextPolicy

事件表和系统调用应使用 `AITextPolicy` 控制 AI 权限。

| Policy | 说明 | 可用场景 |
|---|---|---|
| None | 不使用 AI | 固定剧情、关键判定、敏感场景 |
| DescriptionOnly | 只润色描述，不写对话 | 日常事件、地点氛围 |
| DialogueAllowed | 可润色短对话 | 低风险日常或已固定意图的事件 |
| SummaryAllowed | 可润色结算摘要 | 事件结算、周总结 |
| QueryHintOnly | 只返回非剧透线索 | 自然语言查询 UI |
| EndingEpilogueOnly | 只润色已确定后日谈 | 结局后日谈 |

默认策略应保守。没有明确策略时按 `None` 处理。

---

## 9. 降级策略

所有 AI 调用必须有本地模板兜底。

### 9.1 失败类型

| 失败类型 | 处理 |
|---|---|
| 网络失败 | 使用本地模板 |
| 超时 | 使用本地模板 |
| 输出为空 | 使用本地模板 |
| 输出越界 | 丢弃输出，使用本地模板 |
| 内容不安全 | 丢弃输出，使用本地模板 |
| 版本不兼容 | 使用本地模板 |

### 9.2 兜底文案示例

```text
这段命运没有显出更多细节。
```

```text
烟气在神龛前散开，只留下模糊的预感。
```

```text
事情暂时告一段落，但镇上的暗流没有停下。
```

兜底文本必须可本地化。

---

## 10. 敏感内容表达边界

本项目允许民俗恐怖、邪教、灾难、家庭矛盾和命运压力，但表达应克制。

### 10.1 推荐表达

| 内容 | 推荐方式 |
|---|---|
| 邪教 | 使用暗示、仪式痕迹、异常聚集、信任扭曲 |
| 灾难 | 使用后果、环境变化、人物反应 |
| 家庭矛盾 | 使用对话、沉默、回避和关系变化 |
| 死亡风险 | 使用预兆、失踪、伤亡结果摘要 |
| 精神压力 | 使用行为、梦境、疲惫和旁人观察 |

### 10.2 避免表达

- 露骨性描写。
- 把痛苦当作猎奇看点。
- 对自伤或伤害他人的方法化描述。
- 把敏感内容作为笑点。
- 让 AI 自行扩写血腥、色情、虐待细节。

---

## 11. 本地化与文本 key

AI 输出应尽量服务于可本地化流程。

### 11.1 固定文本

按钮、错误、提示、系统说明必须使用本地化 key，不交给 AI。

示例：

```text
ui.ai_query.not_enough_power
ui.ai_query.forbidden_question
ui.ai_query.fallback_blurry_echo
```

### 11.2 AI 文本

AI 文本可作为运行时生成内容，但需要记录来源：

```text
GeneratedTextRecord
- SourceType
- SourceId
- PromptTemplateId
- InputHash
- OutputText
- FallbackUsed
```

MVP 如果不接入 AI，则只使用本地化文本表和模板。

---

## 12. 与其他系统的依赖

| 系统 | 依赖关系 |
|---|---|
| 事件系统 | 根据 `AITextPolicy` 决定是否请求 AI 润色事件文本 |
| NPC 行为系统 | 提供玩家可见的 NPC 状态和记忆摘要，不提供隐藏全量状态 |
| 缘线业线系统 | 提供可见关系和业线摘要，不提供隐藏结局公式 |
| 资源系统 | 提供已结算资源变化，不允许 AI 修改数值 |
| 结局系统 | 提供已确定结局输入，AI 只润色后日谈 |
| 存档系统 | 可保存生成文本记录，但不依赖 AI 输出恢复核心状态 |
| UI 系统 | 显示 AI 文本、失败状态、内容不允许提示和兜底文本 |
| 本地化系统 | 提供固定文案、错误文案和兜底模板 |

---

## 13. 自然语言查询限制

自然语言查询属于 P2 功能，不进入 MVP。

查询规则：

- 每次查询消耗神力。
- 提交前必须确认消耗。
- 只回答玩家已触及的线索。
- 不回答最优解。
- 不回答完整未来事件表。
- 不回答隐藏结局条件。
- 失败时使用兜底文本。

禁止问题示例：

```text
怎样才能打出传奇结局？
```

```text
顾沉舟第几天会做坏事？
```

```text
我现在应该选哪个选项收益最高？
```

允许回答的改写方向：

```text
这条命运还没有完全显形。你已经见过港口的夜船，也许该留意那些总在夜里出现的人。
```

## 14. 审核与测试

### 14.1 自动检查

AI 输出应至少经过以下检查：

- 是否为空。
- 是否超过长度限制。
- 是否包含禁止词或明显越界内容。
- 是否包含未允许的结构字段。
- 是否试图输出数值变化或结局判定。

### 14.2 人工抽查

上线前应抽查：

- 日常事件生成样例。
- 关键事件结算摘要。
- 问询命运回答。
- 结局后日谈。
- 敏感内容场景。

### 14.3 记录要求

为了复现问题，建议记录：

```text
PromptTemplateId
InputContextHash
AITextPolicy
FallbackUsed
OutputLength
SafetyCheckResult
```

不要在普通日志中记录敏感个人数据或完整玩家输入历史。

---

## 15. 边界情况

| 情况 | 处理 |
|---|---|
| AI 生成了未解锁线索 | 丢弃输出，使用兜底文本 |
| AI 生成了数值变化 | 丢弃输出，使用规则系统结果和模板文本 |
| AI 生成内容与角色设定冲突 | 丢弃输出或重试 |
| AI 服务不可用 | 使用本地模板，不阻塞游戏 |
| 玩家输入越界问题 | 返回内容不允许提示，不扣神力或按设计退还 |
| AI 输出过长 | 截断不可作为默认策略，应重试或使用兜底 |
| 结局后日谈与结局状态冲突 | 以结局系统为准，丢弃 AI 输出 |

---

## 16. 调参项

| 参数 | 默认 | 说明 |
|---|---:|---|
| `EnableOnlineAI` | false | MVP 默认关闭在线 AI |
| `AIQueryCost` | 2 | 自然语言查询消耗神力 |
| `AIRequestTimeoutMs` | 8000 | AI 请求超时 |
| `MaxEventDescriptionChars` | 180 | 事件描述最大长度 |
| `MaxResolutionSummaryChars` | 160 | 结算摘要最大长度 |
| `MaxQueryAnswerChars` | 120 | 问询回答最大长度 |
| `RetryCount` | 0-1 | 失败重试次数 |
| `FallbackAlwaysAvailable` | true | 必须始终有兜底文本 |

---

## 17. 验收标准

### 17.1 规则边界验收

- [ ] AI 不能改变事件触发结果。
- [ ] AI 不能改变资源数值。
- [ ] AI 不能改变 NPC 状态。
- [ ] AI 不能改变缘线强度或业线节点。
- [ ] AI 不能决定结局路线。

### 17.2 文本生成验收

- [ ] 日常事件可在本地模板和 AI 润色之间切换。
- [ ] AI 失败时显示兜底文本。
- [ ] AI 输出不会显示未解锁结局条件。
- [ ] AI 输出不会提供最优解。
- [ ] AI 输出符合长度限制。

### 17.3 内容安全验收

- [ ] 邪教、灾难、死亡风险使用暗示和后果表达。
- [ ] 不出现露骨性描写。
- [ ] 不出现猎奇化苦难。
- [ ] 不鼓励自伤或伤害他人。
- [ ] 敏感内容不作为笑点。

### 17.4 MVP 验收

- [ ] 在 `EnableOnlineAI = false` 时，游戏仍可完整运行。
- [ ] 所有 AI 使用场景都有本地模板兜底。
- [ ] 事件表中的 `AITextPolicy` 可控制是否允许 AI。
- [ ] UI 能显示 AI 查询失败、内容不允许和信息不足状态。
