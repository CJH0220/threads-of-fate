# 神力干预系统

> 文档状态：初版  
> 更新日期：2026-06-12  
> 所属阶段：P0  
> 用途：定义玩家如何用神力影响事件、NPC 和命运走向

---

## 1. 系统目标

神力干预系统是玩家与小镇命运互动的核心。玩家不是直接控制 NPC，而是以土地公身份在关键节点进行间接影响。

系统目标：

1. 让玩家能做出有代价的选择。
2. 让干预结果影响事件、NPC 状态、资源和结局。
3. 保持“不能直接操控人类意志”的世界观限制。
4. 支持正神、邪神、中性等不同路线。

---

## 2. 干预原则

### 2.1 玩家不能做什么

玩家不能：

- 直接命令 NPC 行动。
- 直接修改已发生事件。
- 直接杀死或复活 NPC。
- 直接改变 NPC 核心人格。
- 无代价地获得完整真相。

### 2.2 玩家可以做什么

玩家可以：

- 让 NPC 做梦。
- 给某次行动带来好运。
- 让两个人更容易相遇。
- 暂时遮蔽某条信息。
- 用显灵制造警示。
- 强化自己对命运线的观察。

---

## 3. 干预类型

| 干预 | ResourceId | 默认消耗 | 路线倾向 | MVP |
|---|---|---:|---|---|
| 托梦 | `dream_hint` | 2 | 阳德 / 中性 | 是 |
| 赐福 | `blessing` | 3 | 阳德 | 是 |
| 牵引机缘 | `fate_pull` | 4 | 中性 / 阳德 | P1 可做，P0 可配置 |
| 遮蔽灾兆 | `omen_hide` | 3 | 阴德 | P1 可做，P0 可配置 |
| 显灵警示 | `spirit_warning` | 5 | 阳德 | P1 可做，P0 可配置 |
| 强化观察 | `insight` | 1 | 中性 | P0 可选 |

---

## 4. 玩家可见信息

干预按钮应显示：

- 干预名称
- 神力消耗
- 目标对象或事件
- 大致结果倾向
- 是否有风险
- 是否当前可用

示例：

```text
托梦（消耗 2 神力）
让林潮音梦见危险征兆，可能提高她对寺庙异常的警觉。
```

成功率不显示精确数值，只显示：

```text
容易 / 普通 / 困难 / 几乎不可能
```

## 5. 干预详规

### 5.1 托梦

| 项目 | 规则 |
|---|---|
| 目标 | NPC |
| 消耗 | 2 神力 |
| 作用 | 改变 NPC 对风险、关系或事件的认知倾向 |
| 典型效果 | 增加警觉、降低冲动、触发特殊对话 |
| 资源倾向 | 阳德或中性 |
| 失败表现 | 梦境模糊，只产生轻微情绪变化 |

适用场景：

- 提醒林潮音注意异常。
- 让陈远舟重新思考离岛计划。
- 让苏婉察觉陈海生危险状态。

### 5.2 赐福

| 项目 | 规则 |
|---|---|
| 目标 | NPC / 事件判定 |
| 消耗 | 3 神力 |
| 作用 | 提高一次正向行动或判定成功率 |
| 典型效果 | 成功率提升、命运硬币增加、负面结果缓和 |
| 资源倾向 | 阳德 |
| 失败表现 | 只获得轻微正向修正 |

适用场景：

- 赐福陈远舟的勇气。
- 赐福许明川找到证据。
- 赐福林潮音在压力下保持清醒。

### 5.3 牵引机缘

| 项目 | 规则 |
|---|---|
| 目标 | 两名 NPC / NPC 与地点 |
| 消耗 | 4 神力 |
| 作用 | 提高相遇、发现、对话或事件触发概率 |
| 典型效果 | 让两人同一时段到同一地点 |
| 资源倾向 | 中性或阳德 |
| 风险 | 也可能让错误的人相遇 |

适用场景：

- 让许明川遇到周行知。
- 让林潮音避开顾沉舟。
- 让苏婉提前发现家庭风险。

### 5.4 遮蔽灾兆

| 项目 | 规则 |
|---|---|
| 目标 | 信息 / 线索 / 风险提示 |
| 消耗 | 3 神力 |
| 作用 | 暂时降低 NPC 或系统对某风险的察觉 |
| 典型效果 | 延迟调查、隐藏证据、避免短期冲突 |
| 资源倾向 | 阴德 |
| 风险 | 长期可能导致更大事故 |

适用场景：

- 暂时阻止许明川发现某条证据。
- 避免寺庙腐败立刻暴露。
- 让邪教线继续推进以换取其他路线收益。

### 5.5 显灵警示

| 项目 | 规则 |
|---|---|
| 目标 | 地点 / 群体 / 关键事件 |
| 消耗 | 5 神力 |
| 作用 | 强制制造明显警告，影响多个 NPC |
| 典型效果 | 降低高风险事件发生概率，提升香火或恐惧 |
| 资源倾向 | 阳德，但可能造成恐慌 |
| 风险 | 过度使用可能让居民畏惧而非信任 |

适用场景：

- 寺庙火灾前警示。
- 港口夜船前警示。
- 庙会聚集前警示。

### 5.6 强化观察

| 项目 | 规则 |
|---|---|
| 目标 | NPC / 事件 / 地点 |
| 消耗 | 1 神力 |
| 作用 | 显示更多风险或可能影响 |
| 典型效果 | 解锁一条模糊提示 |
| 资源倾向 | 中性 |
| 风险 | 不直接改变事件结果 |

## 6. 干预窗口

不是所有时候都能干预。事件可配置干预窗口。

| 窗口 | 说明 | 示例 |
|---|---|---|
| 事件前 | 事件还未发生，只能影响触发概率或参与者状态 | 让陈远舟更有勇气 |
| 事件中 | 事件正在推进，可改变分支 | 争吵中托梦提醒 |
| 事件后 | 事件已发生，只能影响后续补救 | 事故后引导和解 |

P0 阶段建议优先支持：

```text
事件前干预
事件中一次选择干预
```

---

## 7. 成功判定

### 7.1 MVP 简化判定

P0 可使用固定修正：

```text
最终判定值 = 事件基础值 + NPC 状态修正 + 干预修正
```

| 结果 | 判定 |
|---|---|
| 成功 | 最终判定值 ≥ 目标难度 |
| 部分成功 | 最终判定值接近目标难度 |
| 失败 | 最终判定值 < 目标难度 |

### 7.2 命运硬币表现

UI 可把判定包装成命运硬币：

```text
硬币数 = 基础硬币 + 角色状态硬币 + 干预奖励硬币
需求值 = 事件难度
```

玩家看到的是硬币和结果，不需要看到底层公式。

---

## 8. 结果影响

干预可能影响：

| 目标 | 示例 |
|---|---|
| 资源 | 神力 -3、阳德 +5、阴德 +3、香火 +2 |
| NPC 状态 | 警觉 +1、压力 -1、信任 +1 |
| 事件结果 | 成功 / 部分成功 / 失败 |
| 事件触发 | 后续事件解锁或阻断 |
| 关系 | 缘线强度改变 |
| 业线 | 业线节点推进或偏移 |
| 结局 | 路线倾向变化 |

---

## 9. 与资源系统的关系

- 所有干预都必须先检查神力。
- 干预消耗立即扣除，除非设计明确允许失败不扣。
- 阳德 / 阴德变化在事件结算后统一处理。
- 不允许 UI 直接修改资源，必须通过干预或事件结算。

---

## 10. 与事件系统的关系

事件配置中应声明：

```text
InterventionWindows
AllowedInterventionIds
ResolutionRules
OutcomeEffects
```

若事件未声明某干预可用，则 UI 不显示该干预。

## 13. 边界情况

| 情况 | 处理 |
|---|---|
| 神力不足 | 禁用按钮并提示 |
| 事件无干预窗口 | 不显示干预按钮 |
| 目标 NPC 不在事件中 | 只允许事件前牵引类干预 |
| 同一事件多次干预 | P0 默认只允许一次 |
| 干预失败 | 仍记录历史，并显示轻微反馈 |
| 阴德干预导致短期好结果 | 允许，但记录阴德和后续风险 |

## 15. 验收标准

- [ ] 事件面板能显示当前可用干预。
- [ ] 神力不足时干预按钮禁用。
- [ ] 执行干预会扣除神力。
- [ ] 托梦能改变至少一个事件结果或 NPC 状态。
- [ ] 赐福能提高至少一次判定成功率。
- [ ] 干预结果能写入历史记录。
- [ ] 阴德类干预会增加阴德或记录阴德倾向。
- [ ] 每个事件 P0 阶段最多只允许一次干预。
