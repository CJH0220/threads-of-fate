## 9. 研发交付检查

研发开始拆任务前，应满足：

- [ ] 当前 MVP 范围明确。
- [ ] P0 UI 界面明确。
- [ ] P0 系统规则明确。
- [ ] P0 数据表字段明确。
- [ ] 首个竖切目标明确。
- [ ] 不做事项明确。
- [ ] 所有核心名词统一。
- [ ] 可以从文档拆出任务列表。

建议首个研发竖切：

```text
主菜单 → 新游戏 → 主地图 HUD → 查看 3 名 NPC → 触发 1 个事件 → 使用 1 次干预 → 推动时间 → 显示结算
```

- [ ] 每个 P0 系统有验收标准。

- [ ] 事件流程有至少 1 条 golden path。
- [ ] 神力不足、无事件、无存档、读档失败等边界状态有说明。
- [ ] Game Over 或临时终局有触发条件。
