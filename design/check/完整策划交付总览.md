## 1. 项目基准

| 项目 | 当前定稿 |
|---|---|
| 游戏名 | 《命运的织线》 |
| 引擎方向 | Godot |
| 美术方向 | 简约 2D 像素风 |
| 类型 | 低成本叙事模拟 / 命运干预 |
| 主舞台 | 归潮镇 |
| 玩家身份 | 土地公，岛民称“岛爷” |
| 游戏时长 | 60 天，早 / 午 / 晚 |
| 核心资源 | 香火、神力、阳德、阴德 |
| 核心循环 | 观察命运 → 选择干预 → 时间推进 → 事件结算 → 命运变化 |
| MVP 原则 | 规则驱动优先，AI 文本可选且可降级 |

## 3. 首个研发竖切建议

建议首个竖切按以下顺序实现：

```text
主菜单
→ 新游戏
→ 开局剧情
→ 主地图 HUD
→ 查看林潮音 / 陈远舟 / 陈海生
→ 触发咖啡店复习或陈海生发脾气
→ 使用托梦或赐福
→ 事件结算
→ 推进时辰
→ 显示本时段摘要
```

对应文件：

- `content/MVP竖切剧情脚本.md`
- `UI/ui-spec-main-menu.md`
- `UI/ui-spec-new-game.md`
- `UI/ui-spec-hud.md`
- `UI/ui-spec-character-panel.md`
- `UI/ui-spec-event-panel.md`
- `UI/ui-spec-intervention-panel.md`
- `UI/ui-spec-time-advance.md`
- `system/事件系统.md`
- `system/事件结果扩展规则.md`
- `data/事件配置表.csv`
- `data/事件结果表.csv`

## 4. 研发拆任务入口

建议按以下任务包拆解：

1. Godot 项目初始化与数据加载。
2. 本地 CSV / JSON 读取和 ID 校验。
3. 主菜单、新游戏、加载、设置基础 UI。
4. 主地图 HUD 和地点点击。
5. NPC 数据展示。
6. 事件触发与事件面板。
7. 神力干预与资源结算。
8. 时间推进与摘要。
9. 存档和读档。
10. 结局与 Game Over 基础框架。

---

## 5. 测试入口

测试优先验证：

- 新游戏能进入主地图。
- P0 数据表能被读取。
- 事件 ID、NPC ID、地点 ID 无缺失引用。
- 神力不足时不能干预。
- 干预后资源变化正确。
- 时间推进后事件不重复结算。
- 保存读档后时间、资源、NPC 状态一致。
- Game Over 和 60 天终局有可触发路径。
