# MVP 范围检查

> 文档状态：初版  
> 更新日期：2026-06-12  
> 用途：确认《命运的织线》首版 MVP 是否收敛、可实现、可验收

---

## 1. MVP 目标

首版 MVP 的目标不是做完整 60 天全部内容，而是验证核心玩法闭环是否成立：

```text
观察命运 → 选择干预 → 推动时间 → 事件结算 → 资源变化 → 角色命运变化 → 阶段总结
```

MVP 应优先完成一个可玩的竖切版本，证明玩家能理解并享受“有限神力干预小镇命运”的核心体验。

---

## 2. P0 必须包含

### 2.1 时间与流程

- [ ] 支持天数推进。
- [ ] 支持早 / 午 / 晚三个时段或 MVP 简化时段。
- [ ] 支持推动时间。
- [ ] 支持事件结算后返回主地图 HUD。
- [ ] 支持阶段总结或简版周总结。
- [ ] 支持终局或临时结算界面。

### 2.2 地图与 UI

- [ ] 主菜单可进入新游戏。
- [ ] 主地图 HUD 可显示时间、香火、神力、阳德、阴德。
- [ ] 主地图可进入人物、地点、事件至少一种核心面板。
- [ ] 推动时间按钮可用。
- [ ] 时间推进后有变化摘要。
- [ ] 错误、空状态、加载状态不显示空白界面。

### 2.3 核心资源

- [ ] 香火有初始值和变化来源。
- [ ] 神力有初始值和消耗方式。
- [ ] 阳德和阴德至少能记录变化。
- [ ] 资源变化会在 UI 中反馈。
- [ ] 资源阈值能影响结算或路线倾向。

### 2.4 NPC 与内容

- [ ] 至少 3 名核心 NPC 可用。
- [ ] 每名 MVP NPC 有基础信息、当前状态、至少一条业线。
- [ ] 至少一组关键关系可见。
- [ ] 至少一条角色结局或阶段结果可展示。

建议 MVP NPC：

| NPC | 用途 |
|---|---|
| 林潮音 | 巫女线、灵力异常、青春线 |
| 陈远舟 | 离岛梦想、升学压力、爱情线 |
| 陈海生 | 家庭压力、邪教易感、风险事件 |

### 2.5 事件系统

- [ ] 至少 7 天可玩事件池。
- [ ] 至少 1 个锚点事件。
- [ ] 至少 3 个日常事件。
- [ ] 至少 2 个关键事件。
- [ ] 事件有触发条件。
- [ ] 事件有参与 NPC。
- [ ] 事件有结算结果。
- [ ] 事件能写入历史记录或摘要。

### 2.6 干预系统

- [ ] 至少实现 2 种神力干预。
- [ ] 干预会消耗神力。
- [ ] 干预会改变事件结果、资源或 NPC 状态。
- [ ] 神力不足时不能执行干预，并有明确提示。

建议 MVP 干预：

| 干预 | 用途 |
|---|---|
| 托梦 | 提醒 NPC、改变风险认知 |
| 赐福 | 提高某次判定或事件成功率 |

### 2.7 结算与反馈

- [ ] 时间推进后显示变化摘要。
- [ ] 关键事件结算显示资源变化。
- [ ] 关键事件结算显示 NPC 状态变化。
- [ ] 阶段结束显示简版总结。
- [ ] 失败或终局至少有一个临时结果界面。

---

## 3. MVP 暂不包含

- [ ] 暂不包含完整 60 天全部事件。
- [ ] 暂不包含 14 名 NPC 的完整 Agent 推演。
- [ ] 暂不包含复杂关系网动画。
- [ ] 暂不包含自然语言自由查询。
- [ ] 暂不包含多周目继承。
- [ ] 暂不包含完整结局回顾画廊。
- [ ] 暂不包含高成本过场动画。
- [ ] 暂不包含复杂 AI 记忆数据库。

---

## 4. MVP 数据要求

- [ ] 有事件配置表。
- [ ] 有 NPC 基础数据。
- [ ] 有地点基础数据。
- [ ] 有资源初始值。
- [ ] 有干预技能配置。
- [ ] 所有 ID 命名符合 `data/README.md`。

---

## 5. MVP 验收标准

- [ ] 玩家可以从主菜单开始新游戏。
- [ ] 玩家可以在主地图查看当前时间和资源。
- [ ] 玩家可以查看至少 3 名 NPC 的基础信息。
- [ ] 玩家可以触发至少 1 个关键事件。
- [ ] 玩家可以执行至少 1 次神力干预。
- [ ] 玩家可以推动时间并看到结算结果。
- [ ] 事件结果能改变至少一种资源或 NPC 状态。
- [ ] MVP 流程能完成一个小型阶段结算。
- [ ] 游戏不会因为无事件、无数据、加载中或错误状态显示空白界面。

---

## 6. MVP 风险项

| 风险 | 说明 | 建议 |
|---|---|---|
| AI 范围过大 | 如果让 AI 决定事件结果，研发和测试不可控 | 规则结算优先，AI 只润色文本 |
| NPC 数量过多 | 14 人全做会拖慢首版 | MVP 先做 3-6 人 |
| UI 范围过大 | 过多界面会分散实现 | 先完成主地图、人物、事件、时间推进 |
| 结局过复杂 | 多结局全做成本高 | MVP 先做阶段总结或单个临时结局 |
| 数据表缺失 | 没有数据表会导致硬编码 | 先补 NPC、地点、资源、技能表 |
