# UI 规格：游戏失败 / Game Over

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

Game Over 界面用于在香火失败、强硬不幸结局或关键失败事件触发时，清楚告诉玩家失败原因，并提供读档、返回主菜单或重新开始的选择。

---

## 2. 触发条件

| 类型 | 示例 |
|---|---|
| 香火失败 | 60 天后香火 < 100 |
| 强硬不幸结局 | 寺庙被烧毁、异教徒入侵、集体叛教 |
| 特殊终局 | 某个关键命运事件导致不可逆失败 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 香火熄灭                              │
├──────────────────────────────────────┤
│ 土地庙前最后一缕香火散去。             │
│ 归潮镇不再记得你的名字。               │
│                                      │
│ 失败原因：香火低于 100                 │
│ 触发时间：第 60 天 夜晚                │
│ 关键诱因：连续错过寺庙事件              │
│                                      │
│ [读取存档] [重新开始] [返回主菜单]      │
└──────────────────────────────────────┘
```

---

## 4. 展示内容

| 模块 | 内容 |
|---|---|
| 失败标题 | 如“香火熄灭”“神位崩塌”“归潮镇沦陷” |
| 失败描述 | 1–2 段叙事文本 |
| 失败原因 | 具体规则原因 |
| 触发时间 | 第 X 天 · 时段 |
| 关键诱因 | 可选，列出 1–3 个重要原因 |
| 操作 | 读取存档、重新开始、返回主菜单 |

---

## 5. 视觉规则

- 色调：暗灰、深蓝、暗红。
- 像素背景：破败土地庙、熄灭香炉、空荡小镇。
- 标题可使用红色或灰白色高对比字体。
- 不使用跳脸或恐怖刺激表现，保持克制和叙事感。

---

## 6. 交互规则

| 操作 | 结果 |
|---|---|
| 读取存档 | 打开加载游戏界面 |
| 重新开始 | 进入新游戏界面，需确认 |
| 返回主菜单 | 返回主菜单 |
| Esc | 不直接退出，聚焦按钮或无效 |

规则：

- 重新开始必须二次确认。
- 若没有可读存档，读取存档按钮灰化。
- 可以保存失败记录到结局回顾，P2。

---

## 7. Godot 实现建议

```text
GameOverScreen (Control)
├── Background
├── PanelContainer
│   ├── FailureTitle
│   ├── FailureDescription
│   ├── FailureReasonBlock
│   └── ButtonRow
│       ├── LoadButton
│       ├── RestartButton
│       └── MainMenuButton
└── ConfirmDialogLayer
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 失败类型 | EndingSystem | 标题和视觉风格 |
| 失败原因 | FailureReason | 规则说明 |
| 触发时间 | TimeSystem | 显示时间 |
| 关键诱因 | EventHistorySystem | 回顾说明 |
| 是否有存档 | SaveSystem | 控制读取按钮 |

---

## 9. 验收标准

- [ ] 香火失败或强硬不幸结局能进入 Game Over 界面。
- [ ] 界面显示明确失败标题和失败原因。
- [ ] 显示触发时间和关键诱因。
- [ ] 有存档时可进入读档界面。
- [ ] 无存档时读取存档按钮灰化。
- [ ] 重新开始前必须二次确认。
- [ ] 返回主菜单按钮可正常返回。
