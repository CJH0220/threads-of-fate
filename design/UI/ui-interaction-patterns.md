# UI 规范：交互模式库

> 文档状态：初稿  
> 更新日期：2026-06-12  
> 所属阶段：P0 全局规范  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

本文定义《命运的织线》UI 的通用交互模式，供所有界面、组件、程序实现和 QA 验收复用。

目标是避免每个界面重复定义按钮、Tab、弹窗、列表、返回行为等基础交互。

---

## 2. 输入方式

| 输入 | 支持阶段 | 用途 |
|---|---|---|
| 鼠标点击 | P0 | 所有 UI 主操作 |
| 鼠标 Hover | P0 | 说明、状态预览、非关键信息 |
| 键盘导航 | P0 | 菜单、弹窗、设置、存档、主地图基础焦点 |
| 快捷键 | P1 | 主地图 Tab、常用面板 |
| 手柄 | P2 | 若后续移植或增强辅助功能 |

规则：

- 不能存在只能 hover 才能理解的关键信息。
- 不能存在必须精确拖拽才能完成的 P0 操作。
- Slider 必须支持键盘左右键调整。

---

## 3. 按钮模式

### 3.1 状态

| 状态 | 视觉 | 行为 |
|---|---|---|
| 默认 | 木牌 / 纸符底色，深色描边 | 可点击 |
| Hover | 亮边框或轻微提亮 | 鼠标悬停反馈 |
| Focus | 明显外框或高亮描边 | 键盘当前目标 |
| Pressed | 下压 1–2 像素或变暗 | 触发中 |
| Disabled | 降低饱和度，保留可读文字 | 不可点击，不进焦点 |

### 3.2 文案

按钮文案必须说明动作：

| 不推荐 | 推荐 |
|---|---|
| 确定 | 删除存档 / 继续推动 / 应用设置 |
| 返回 | 返回主菜单 / 返回地图 |
| 是 | 确认消耗神力 |

危险按钮必须使用具体后果动词。

---

## 4. Tab 模式

用于主地图底栏、设置分类、图鉴分类等。

### 4.1 状态

| 状态 | 表现 |
|---|---|
| 未选中 | 常规按钮样式 |
| 当前选中 | 更深底色或顶部标记 |
| Hover / Focus | 高亮边框 |
| Disabled | 灰化，显示原因 |

### 4.2 交互

| 操作 | 结果 |
|---|---|
| 鼠标点击 Tab | 切换内容 |
| ← / → | 在同组 Tab 间切换焦点 |
| Enter / Space | 激活焦点 Tab |
| 数字键 1–4 | 主地图底栏 Tab 快捷切换，P1 |

Tab 切换不应重置玩家当前选中对象，除非明确说明。

---

## 5. 列表与卡片模式

用于存档列表、人物列表、事件列表、缘线/业线列表。

### 5.1 列表状态

| 状态 | 表现 |
|---|---|
| 有内容 | 显示卡片列表 |
| 空状态 | 使用 `StateMessage` |
| 加载中 | 显示加载文案 |
| 错误 | 显示错误说明与重试 / 返回 |

### 5.2 卡片规则

每张卡片必须有：

- 标题。
- 一行摘要。
- 状态标记。
- 是否可点击的清晰反馈。

如果卡片可进入详情，整张卡片可点击；如果卡片内有危险操作，危险按钮必须独立确认。

---

## 6. 弹窗模式

所有弹窗默认阻塞背景操作。

| 类型 | 使用场景 | 关闭规则 |
|---|---|---|
| 普通弹窗 | 设置提示、说明 | Esc / 关闭按钮 |
| 确认弹窗 | 应用设置、覆盖存档 | 确认 / 取消 |
| 危险确认 | 删除、退出、推动危险时间 | 必须显式选择 |
| 信息弹窗 | 教程、关于 | Esc / 关闭按钮 |

规则：

- 弹窗打开时焦点进入弹窗内。
- 弹窗关闭时焦点回到打开它的控件。
- 危险确认点击遮罩不关闭。
- 普通信息弹窗可配置点击遮罩关闭。

---

## 7. Toast 模式

Toast 用于非阻塞反馈。

| 类型 | 示例 | 时长 |
|---|---|---|
| 成功 | 命运已记录 | 2 秒 |
| 普通 | 设置已应用 | 2 秒 |
| 线索 | 获得线索：港口夜谈 | 3 秒 |
| 警告 | 神力不足 | 3 秒 |
| 错误 | 保存失败，请稍后再试 | 4 秒 |

规则：

- Toast 不抢焦点。
- Toast 不阻塞点击。
- 多条 Toast 堆叠显示。
- 重复资源变化应合并，如“香火 +1 ×3”。

---

## 8. Tooltip / 悬停说明

Hover 可用于解释资源、状态、地点人数等补充信息，但不能承载唯一关键信息。

| 场景 | Tooltip 内容 |
|---|---|
| 香火 | 当前香火含义、每日消耗 |
| 神力 | 当前可用神力、恢复规则 |
| 地点 | 地点名、当前人数、风险标记 |
| 禁用按钮 | 禁用原因 |

键盘焦点停留时也应显示同等说明，避免键盘玩家看不到 Hover 信息。

---

## 9. 返回与导航模式

### 9.1 通用返回

| 场景 | Esc 行为 |
|---|---|
| 主菜单 | 无弹窗时不退出游戏，可聚焦退出按钮 |
| 主地图 HUD | 打开暂停菜单 |
| 普通面板 | 返回来源界面 |
| 弹窗 | 取消 / 关闭 |
| 最终结局界面 | 不直接退出，避免误跳过 |

### 9.2 来源记录

所有面板关闭时使用 `NavigationContext` 返回来源：

- 游戏内默认回主地图 HUD。
- 游戏外默认回主菜单。
- 设置界面从暂停菜单进入时返回暂停菜单。
- 设置界面从主菜单进入时返回主菜单。

---

## 10. 主地图交互模式

主地图是 P0 主要操作界面。

### 10.1 鼠标

| 操作 | 结果 |
|---|---|
| 点击地点 | 打开地点面板 |
| 点击人物标记 | 打开人物面板 |
| 点击事件标记 | 打开事件面板 |
| 点击危险提示 | 定位到相关事件 / 地点 |
| 点击推动时间 | 进入时间推进流程 |

### 10.2 键盘

键盘焦点顺序：

```text
设置 → 左侧提示区 → 地图地点 / 事件标记 → 底栏 Tab → 上下文列表 → 推动时间
```

地图内焦点排序：

1. 危险事件。
2. 主线事件。
3. 普通事件。
4. 地点。
5. 人物标记。

---

## 11. 危险操作模式

危险操作必须符合：

- 触发前有明确视觉标记。
- 执行前二次确认。
- 默认焦点在安全选择上。
- 文案说明后果。
- 执行后有 Toast 或结果反馈。

危险操作包括：

- 删除存档。
- 覆盖存档。
- 退出游戏。
- 返回主菜单且有未保存内容。
- 消耗神力干预。
- 未查看危险事件时推动时间。

---

## 12. 加载、空、错误模式

所有列表和异步流程必须使用 `ui-component-empty-loading-error.md` 定义的状态组件。

| 状态 | 要求 |
|---|---|
| 空 | 说明当前没有内容，并给出下一步 |
| 加载 | 显示正在处理的文案 |
| 错误 | 说明失败结果和可做操作 |
| 禁用 | 说明暂不可用原因 |

不允许出现空白面板。

---

## 13. 本地化交互限制

- 按钮宽度预留 40% 文本扩展。
- 长标题允许换行或滚动。
- 不把按钮文案烘焙到图片中。
- 避免只靠中文短词区分危险操作。

---

## 14. 验收标准

- [ ] 所有 P0 按钮具备默认、Hover、Focus、Pressed、Disabled 状态。
- [ ] 弹窗打开时背景不可操作，关闭后焦点回到来源控件。
- [ ] 主地图地点、事件和推动时间都支持键盘焦点。
- [ ] Tooltip 信息在键盘焦点下也能查看。
- [ ] 危险操作必须经过确认弹窗。
- [ ] 列表为空、加载、错误时不出现空白界面。
- [ ] Toast 不抢焦点、不阻塞操作。
- [ ] Esc 行为符合导航模式表。
