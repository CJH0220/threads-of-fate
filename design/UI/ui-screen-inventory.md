# 《命运的织线》UI 面板总表

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 引擎方向：Godot  
> 美术方向：简约像素风  
> 用途：列出游戏可能出现的 UI 面板、导航关系、优先级与后续单屏规格拆分计划。

---

## 1. 总体 UI 分层

游戏 UI 分为四层：

```text
启动与外层菜单
├── 启动画面
├── 主菜单
├── 新游戏 / 创建存档
├── 加载游戏 / 读档
├── 设置
├── 关于我们
└── 退出确认

游戏内主循环界面
├── 主地图 HUD
├── 人物面板
├── 地点面板
├── 事件推进面板
├── 缘线视图
├── 业线视图
├── 干预选择面板
├── 时间推进 / 结算面板
└── 暂停菜单

系统反馈与弹窗
├── 通用确认弹窗
├── 提示 Toast
├── 教程提示
├── 获得 / 失去资源提示
├── 神力不足提示
├── 危险事件警告
├── 存档成功 / 失败提示
└── 加载 / 错误 / 空状态

阶段与结局界面
├── 周结算界面
├── 角色结局界面
├── 60 天总评界面
├── 游戏失败界面
├── 正神 / 邪神 / 中性 / 传奇结局界面
└── 结局回顾 / 图鉴入口
```

## 3. 启动与外层菜单

### 3.1 启动画面

| 项 | 设计 |
|---|---|
| 用途 | 展示游戏 Logo、制作组标识、初始化加载 |
| 核心内容 | 游戏名、像素风背景、加载提示 |
| 操作 | 任意键 / 点击进入主菜单；初始化未完成时不可跳过 |
| 优先级 | P1 |
| Godot 建议 | `Control` + `TextureRect` + `Label` + 简单淡入淡出 |

低成本表现：小镇清晨、土地庙、香火烟雾、游戏标题像素字。

---

### 3.2 主菜单

| 项 | 设计 |
|---|---|
| 用途 | 游戏入口与外层导航中枢 |
| 核心按钮 | 继续游戏、新游戏、加载游戏、设置、关于我们、退出 |
| 背景 | 像素风归潮镇远景或土地庙静态图 |
| 操作 | 鼠标点击；键盘上下选择 + Enter 确认；遵循 `ui-accessibility-guidelines.md` |
| 优先级 | P0 |
| Godot 建议 | `VBoxContainer` 按钮组 + `TextureRect` 背景 |

按钮顺序：

```text
继续游戏
新游戏
加载游戏
设置
关于我们
退出
```

规则：

- 没有存档时，“继续游戏”灰化或隐藏。
- “退出”必须弹出二次确认。
- 主菜单应支持返回键从子界面回到主菜单。

---

### 3.3 新游戏 / 创建存档

| 项 | 设计 |
|---|---|
| 用途 | 开始新一轮 60 天游戏 |
| 核心内容 | 存档槽选择、难度/模式、开局确认 |
| 操作 | 选择空存档槽 → 确认新游戏 → 进入开局剧情 |
| 优先级 | P0 |
| Godot 建议 | `PanelContainer` + `ItemList` / 自定义存档卡片 |

MVP 字段：

| 字段 | 说明 |
|---|---|
| 存档槽 | 选择写入哪个槽位 |
| 开局模式 | 默认标准模式 |
| 提示 | “一局为 60 天，每天 3 个时段” |
| 确认按钮 | 开始编织命运 |

后置字段：

- 难度选择
- 初始神力修正
- 开局事件包选择
- 自定义小镇名字

---

### 3.4 加载游戏 / 读档

| 项 | 设计 |
|---|---|
| 用途 | 选择已有存档并进入游戏 |
| 核心内容 | 存档卡片列表、当前天数、时段、路线倾向、保存时间 |
| 操作 | 点击存档 → 查看详情 → 加载 |
| 优先级 | P0 |
| Godot 建议 | `ScrollContainer` + 存档卡片组件 |

存档卡片字段：

| 字段 | 示例 |
|---|---|
| 存档名 | 存档 01 |
| 游戏进度 | 第 12 天 · 夜晚 |
| 资源概况 | 香火 63 / 神力 4 |
| 路线倾向 | 阳德偏高 / 阴德偏高 / 中立 |
| 最近保存 | 2026-06-10 21:30 |

空状态：

```text
尚无存档。
请从“新游戏”开始第一段命运。
```

---

### 3.5 设置界面

| 项 | 设计 |
|---|---|
| 用途 | 调整音频、显示、文本速度、辅助功能 |
| 入口 | 主菜单、暂停菜单 |
| 优先级 | P0 |
| Godot 建议 | `TabContainer`：音频 / 显示 / 操作 / 辅助功能 |

设置分类：

| 分类 | 选项 |
|---|---|
| 音频 | 主音量、音乐、音效、环境音 |
| 显示 | 窗口模式、分辨率、像素缩放、屏幕震动开关 |
| 文本 | 打字速度、Auto 速度、是否跳过已读文本 |
| 操作 | 键位提示、快捷键查看 |
| 辅助功能 | 字体大小、减少动画、颜色辅助、确认提示强度 |

MVP 必做：音量、窗口模式、文本速度、减少动画。

---

### 3.6 关于我们

| 项 | 设计 |
|---|---|
| 用途 | 展示制作人员、鸣谢、版本信息 |
| 核心内容 | 制作组、成员、素材/工具鸣谢、版本号 |
| 优先级 | P2 |
| Godot 建议 | `ScrollContainer` + `RichTextLabel` |

---

### 3.7 退出确认

| 项 | 设计 |
|---|---|
| 用途 | 防止误退出 |
| 文案 | “确定要离开归潮镇吗？” |
| 操作 | 确定退出 / 取消 |
| 优先级 | P0 |
| Godot 建议 | 通用确认弹窗组件 |

### 4.7 干预选择面板

| 项 | 设计 |
|---|---|
| 用途 | 玩家选择消耗神力的干预方式 |
| 入口 | 人物、地点、事件、业线节点 |
| 优先级 | P1 |
| 单屏规格 | `ui-spec-intervention-panel.md` |

核心字段：

| 字段 | 说明 |
|---|---|
| 干预名称 | 如托梦、赐福、遮蔽、牵引机缘 |
| 神力消耗 | 明确显示消耗 |
| 可能影响 | 模糊提示，不剧透结果 |
| 风险提示 | 可能带来副作用 |
| 确认按钮 | 二次确认消耗 |

---

### 4.8 时间推进 / 结算面板

| 项 | 设计 |
|---|---|
| 用途 | 推进早/午/晚时段，并展示本时段变化 |
| 入口 | 主地图“推动时间”按钮 |
| 优先级 | P0 |
| 单屏规格 | `ui-spec-time-advance.md` |

流程：

```text
点击推动时间
→ 若有未查看危险事件，弹出确认
→ 播放短暂“命运流转中”过渡
→ 展示事件变化 / 资源变化
→ 返回主地图
```

---

### 4.9 暂停菜单

| 项 | 设计 |
|---|---|
| 用途 | 游戏内暂停、保存、设置、返回主菜单 |
| 入口 | Esc |
| 优先级 | P0 |
| 单屏规格 | `ui-spec-pause-menu.md` |

按钮：

```text
继续游戏
保存游戏
加载游戏
设置
返回主菜单
退出游戏
```

规则：

- 返回主菜单和退出游戏需二次确认。
- 保存成功后显示 Toast。

## 9. 全局 UI 约束

全局交互与可访问性以以下文件为准：

| 规范 | 文件 | 用途 |
|---|---|---|
| 可访问性基线 | `ui-accessibility-guidelines.md` | 键盘导航、焦点顺序、字号、颜色辅助、减少动画、危险确认 |
| 交互模式库 | `ui-interaction-patterns.md` | 按钮、Tab、列表、弹窗、Toast、Tooltip、返回行为、危险操作 |
| 本地化与文案 | `ui-localization-guidelines.md` | 文本 key、文案长度、排版扩展、动态数值格式、错误文案 |

通用约束：

- 所有界面默认支持鼠标操作。
- P0 菜单、弹窗、设置、存档和主地图基础操作必须支持键盘导航。
- Esc 返回上一级、关闭当前弹窗，或在主地图打开暂停菜单。
- 所有弹窗 / 面板关闭时返回 `NavigationContext` 记录的来源界面；若来源缺失，游戏内默认返回主地图 HUD，游戏外默认返回主菜单。
- 所有危险操作必须二次确认，且默认焦点在安全按钮上。
- 所有可点击项必须有 default / hover / focus / pressed / disabled 状态。
- 所有列表必须有空状态。
- 所有加载都必须有明确提示，不能假死。
- 所有错误状态必须说明结果和下一步。
- 所有关键风险、关系、资源变化不能只靠颜色表达，必须配合文字、图标或形状。
- 设置界面必须提供字体大小、减少动画、高对比 UI、颜色辅助标记。
- 所有 UI 文案必须能被后续本地化替换，避免写死在脚本中。
- MVP 阶段不依赖复杂动画、复杂关系网、实时 AI 输入或 Live2D。
- Godot 实现优先使用 `Control` 系列节点与可复用组件。
