# UI 规格：事件回顾 / 历史记录

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P2  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

事件回顾用于查看玩家已经经历过的重要事件、角色命运节点和本局关键选择。它服务于复盘和结局回顾，不进入 MVP。

---

## 2. 入口

| 来源 | 触发 |
|---|---|
| 事件推进面板 | 已结束事件点击“查看回顾” |
| 周结算界面 | 点击重大事件条目 |
| 60 天总评 | 点击关键事件回顾 |
| 结局回顾 / 图鉴 | 点击重大事件分类 |
| 人物面板 | 点击人物相关历史事件，P2 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 事件回顾                       [返回] │
├──────────────────────────────────────┤
│ 筛选：[全部] [人物] [地点] [周次]       │
│                                      │
│ 第 12 天 夜晚 · 港口                   │
│ 港口夜谈                              │
│ 林潮音与陈远舟的关系发生变化。          │
│                                      │
│ 第 14 天 下午 · 寺庙                   │
│ 寺庙事故被阻止                         │
│ 香火 +5，阳德 +10                      │
└──────────────────────────────────────┘
```

---

## 4. 展示字段

| 字段 | 内容 |
|---|---|
| 时间 | 第 X 天 · 时段 |
| 地点 | 事件发生地点 |
| 事件标题 | 事件名 |
| 参与角色 | 头像 / 名字列表 |
| 摘要 | 1–2 句事件结果 |
| 变化 | 资源、缘线、业线变化 |
| 标签 | 主线、支线、危险、已干预等 |

---

## 5. 交互规则

- 点击事件条目展开详情。
- 点击参与角色进入人物面板。
- 筛选可按人物、地点、周次、事件类型过滤。
- 未经历事件不显示，避免剧透。
- 回顾模式不重复触发奖励、结算或剧情状态变化。

---

## 6. 状态与变体

| 状态 | 表现 |
|---|---|
| 无历史事件 | “命运尚未留下痕迹” |
| 普通事件 | 标准列表条目 |
| 危险事件 | 红色标签 |
| 已干预事件 | 显示神力图标或“已干预”标签 |
| 结局关键事件 | 金色或特殊标签 |

---

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 已发生事件 | EventHistorySystem | 列表展示 |
| 事件摘要 | EventNarrativeArchive | 回顾文本 |
| 参与角色 | NPCDatabase | 头像和跳转 |
| 资源变化 | EventResolutionSystem | 显示结果 |
| 标签 | EventMetadata | 筛选与视觉 |

---

## 9. 验收标准

- [ ] 周结算、总评或结局回顾可进入事件回顾。
- [ ] 事件列表只显示已经历事件。
- [ ] 条目显示时间、地点、标题和摘要。
- [ ] 点击事件条目可查看详情。
- [ ] 点击参与角色能进入人物面板。
- [ ] 回顾模式不会重复触发奖励或剧情状态变化。
