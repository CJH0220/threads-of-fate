# UI 组件规格：Toast 提示

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P1  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

Toast 用于展示短暂、非阻塞的系统反馈，如保存成功、资源变化、线索获得、操作失败等。

---

## 2. 使用场景

| 场景 | 示例文案 |
|---|---|
| 保存成功 | 命运已记录 |
| 保存失败 | 保存失败，请稍后再试 |
| 获得线索 | 获得线索：港口夜谈 |
| 资源变化 | 香火 +3 |
| 神力不足 | 神力不足 |
| 设置保存 | 设置已应用 |

---

## 3. 布局规格

```text
┌──────────────────────┐
│ 图标  命运已记录       │
└──────────────────────┘
```

位置建议：

- 默认：屏幕右上角
- 游戏内 HUD：右上或底部中央，避免遮挡主地图事件
- 多条 Toast：垂直堆叠，最新在上

---

## 4. 类型

| 类型 | 颜色 | 用途 |
|---|---|---|
| 普通 | 米白 / 深棕 | 一般提示 |
| 成功 | 绿色 / 金色 | 保存、获得 |
| 警告 | 橙色 | 资源不足、可能错过 |
| 错误 | 红色 | 失败、异常 |
| 线索 | 蓝紫色 | 新线索、新命运信息 |

必须有文字，不只靠颜色。

---

## 5. 交互规则

- 默认显示 2–3 秒后自动消失。
- 鼠标悬停时可延长显示，P2。
- 错误类 Toast 可显示 4 秒。
- 同类型短时间重复 Toast 应合并，如“香火 +1 ×3”。
- Toast 不阻塞玩家操作。

---

## 6. Godot 实现建议

```text
ToastLayer (CanvasLayer)
└── ToastStack (VBoxContainer)
    └── ToastItem (PanelContainer)
        ├── Icon (TextureRect)
        └── MessageLabel
```

动效：淡入 + 轻微上移；减少动画开启时直接显示 / 隐藏。

---

## 7. 数据接口

建议提供统一调用：

```text
ToastService.show(message, type, duration)
```

参数：

| 参数 | 说明 |
|---|---|
| message | 显示文案 |
| type | 普通 / 成功 / 警告 / 错误 / 线索 |
| duration | 显示秒数，可选 |

---

## 8. 验收标准

- [ ] 保存成功时显示成功 Toast。
- [ ] 保存失败时显示错误 Toast。
- [ ] Toast 显示后能自动消失。
- [ ] 多条 Toast 能正确堆叠，不互相遮挡。
- [ ] Toast 不阻塞鼠标点击其他 UI。
- [ ] 减少动画开启时不播放位移动画。
