# UI 规格：时间推进 / 结算面板

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

时间推进界面用于结束当前时段，播放短暂过渡，并展示本次时间流动造成的主要变化。它连接“玩家观察与干预”和“世界模拟结果”。

---

## 2. 流程

```text
点击主地图“推动时间”
→ 检查危险事件 / 未查看关键事件
→ 必要时弹出确认
→ 显示“命运流转中……”
→ 推进时段
→ 展示变化摘要
→ 若触发关键事件，进入事件推进面板
→ 返回主地图 HUD
```

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 命运流转中……                          │
├──────────────────────────────────────┤
│ 时间：第12天 下午 → 第12天 夜晚         │
│                                      │
│ 本时段变化：                          │
│ - 香火 -1                             │
│ - 林潮音前往港口                       │
│ - 学校出现新事件：课堂争执              │
│                                      │
│ [继续]                                │
└──────────────────────────────────────┘
```

---

## 4. 组件清单

| 组件 | 内容 | 交互 |
|---|---|---|
| 遮罩 | 暂时遮挡主地图 | 无 |
| 标题 | 命运流转中 / 时段结算 | 无 |
| 时间变化 | 旧时段 → 新时段 | 无 |
| 变化列表 | 资源、人物、事件变化 | 可点击相关项，P1 |
| 继续按钮 | 关闭摘要或进入事件 | 点击确认 |

---

## 5. 危险确认

若存在未查看危险事件，点击推动时间后先显示：

```text
仍有危险征兆尚未查看。
若继续推动时间，可能错过干预机会。

[继续推动] [返回查看]
```

危险确认必须使用红色边框和警告图标。

---

## 6. 变化摘要分类

| 类型 | 示例 |
|---|---|
| 时间变化 | 第 12 天 下午 → 夜晚 |
| 资源变化 | 香火 -1、神力 +1 |
| 人物移动 | 林潮音 前往 港口 |
| 新事件 | 港口出现“夜色中的争执” |
| 业线变化 | 陈远舟主业线推进 |
| 危险提示 | 寺庙出现异常香火 |

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 正常推进 | 显示过渡和摘要 |
| 有危险未处理 | 先弹危险确认 |
| 触发关键事件 | 摘要后进入事件推进面板 |
| 跨天 | 显示日结资源变化 |
| 跨周 | 摘要后进入周结算界面 |
| 推进失败 | 显示错误并保持原时段 |

---

## 8. Godot 实现建议

```text
TimeAdvanceOverlay (CanvasLayer)
├── DimBackground
├── AdvancePanel (PanelContainer)
│   ├── TitleLabel
│   ├── TimeChangeLabel
│   ├── ChangeList (VBoxContainer)
│   └── ContinueButton
└── DangerConfirmDialog
```

---

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 当前时段 | TimeSystem | 显示推进前后 |
| 危险事件状态 | EventRiskSystem | 判断是否确认 |
| 推进结果 | SimulationSystem | 生成变化摘要 |
| 资源变化 | PlayerResourceSystem | 显示增减 |
| 触发事件 | EventScheduler | 决定后续跳转 |

---

## 10. 验收标准

- [ ] 点击推动时间能进入时间推进流程。
- [ ] 有危险事件未查看时，先出现二次确认。
- [ ] 确认推进后，当前时段正确变化。
- [ ] 时间推进后显示至少时间与资源变化摘要。
- [ ] 触发关键事件时能进入事件推进面板。
- [ ] 跨周时能进入周结算界面或预留入口。
- [ ] 推进失败时不会改变当前时段，并显示错误提示。
