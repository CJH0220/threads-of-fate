## 0. 全局规范引用

本界面必须遵循：

---

## 1. 用途

新游戏界面用于创建一轮新的 60 天游戏，选择存档槽，并让玩家理解本局基础规则。

---

## 2. 入口与出口

| 类型 | 说明 |
|---|---|
| 入口 | 主菜单点击“新游戏” |
| 成功出口 | 进入开局剧情 / 主地图 HUD |
| 取消出口 | 返回主菜单 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 新游戏                                │
├──────────────────────────────────────┤
│ 选择存档槽：                          │
│ [空槽 01] [空槽 02] [已有存档 03]      │
│                                      │
│ 模式：标准模式                         │
│ 说明：60 天，每天 早 / 午 / 晚 3 时段   │
│                                      │
│ [开始编织命运]        [返回]           │
└──────────────────────────────────────┘
```

---

## 4. 组件清单

| 组件 | 内容 | 交互 |
|---|---|---|
| 标题 | 新游戏 | 无 |
| 存档槽列表 | 空槽 / 已有存档 | 点击选择 |
| 模式说明 | 标准模式 | MVP 固定不可改 |
| 规则说明 | 60 天、3 时段、有限神力 | 无 |
| 开始按钮 | 开始编织命运 | 创建存档并进入游戏 |
| 返回按钮 | 返回主菜单 | 返回 |

---

## 5. 交互规则

- 玩家必须选择一个存档槽才能开始。
- 选择空槽时直接开始。
- 选择已有存档时必须弹出覆盖确认。
- MVP 只提供标准模式，不做难度选择。
- 创建失败时保留在当前界面并显示错误提示。

---

## 6. 状态与变体

| 状态 | 表现 |
|---|---|
| 无选中存档槽 | 开始按钮灰化 |
| 空槽选中 | 开始按钮可用 |
| 已有存档选中 | 开始按钮可用，但点击后弹覆盖确认 |
| 创建中 | 按钮禁用，显示“命运织线生成中……” |
| 创建失败 | Toast 或错误条提示 |

---

```text
NewGameScreen (Control)
├── PanelContainer
│   ├── TitleLabel
│   ├── SaveSlotList (GridContainer / VBoxContainer)
│   ├── ModeInfoPanel
│   └── ButtonRow
│       ├── StartButton
│       └── BackButton
└── ConfirmDialogLayer
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 存档槽列表 | SaveSystem | 展示可选槽位 |
| 存档摘要 | SaveSystem | 判断是否覆盖 |
| 新游戏默认配置 | GameConfig | 初始化游戏 |
| 当前版本 | ProjectSettings | 写入存档元数据 |

---

## 9. 验收标准

- [ ] 从主菜单点击新游戏能进入该界面。
- [ ] 空槽和已有存档槽显示状态不同。
- [ ] 未选择存档槽时无法开始游戏。
- [ ] 覆盖已有存档前必须二次确认。
- [ ] 创建成功后进入开局剧情或主地图 HUD。
- [ ] 创建失败时显示明确错误，不崩溃。
