# UI 规格：缘线视图

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P1  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

缘线视图用于展示 NPC 之间的人际关系，包括关系对象、关系性质、强度和近期变化。MVP 采用列表式表达，避免一开始实现复杂关系网络图。

---

## 2. 入口

| 来源 | 触发 |
|---|---|
| 主地图底栏 | 点击“缘线”Tab |
| 人物面板 | 点击缘线模块 |
| 事件结算 | 点击关系变化项 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 缘线                                 │
├──────────────────────────────────────┤
│ 筛选：[全部人物] [林潮音] [陈远舟]     │
│                                      │
│ 林潮音                               │
│  - 陈远舟  友情 Lv.3  +5              │
│  - 顾沉舟  利益 Lv.2  -               │
│                                      │
│ 陈远舟                               │
│  - 林潮音  友情 Lv.3  +5              │
└──────────────────────────────────────┘
```

---

## 4. 展示字段

| 字段 | 说明 |
|---|---|
| 主体人物 | 当前关系所属人物 |
| 关系对象 | 另一名 NPC |
| 关系性质 | 爱情、友情、利益、陌生、仇恨 |
| 强度 | Lv.1–Lv.5 或 0–100 |
| 近期变化 | +5 / -3，可选 |
| 活跃状态 | 是否近期被事件影响 |

---

## 5. 颜色规则

| 关系 | 颜色 |
|---|---|
| 爱情 | 红 / 粉 |
| 友情 | 蓝 |
| 利益 | 金 |
| 陌生 | 灰 |
| 仇恨 | 黑 / 暗红 |

同时必须使用图标或文字标记，不能只靠颜色表达关系。

---

## 6. 交互规则

| 操作 | 结果 |
|---|---|
| 点击人物名 | 打开人物面板 |
| 点击关系条目 | 展开关系详情 |
| 筛选人物 | 只显示该人物相关缘线 |
| Hover 关系 | 显示关系说明和最近事件来源 |

关系详情可显示：

- 关系来源事件
- 最近变化
- 当前影响
- 可用干预入口，P1

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 无关系 | “暂无明确缘线” |
| 未解锁关系 | 显示“？？？”或模糊关系 |
| 近期变化 | 条目右侧显示 + / - |
| 危险关系 | 仇恨或高风险关系标红 |
| 筛选无结果 | “该人物暂无已知缘线” |

---

## 8. Godot 实现建议

```text
BondView (PanelContainer)
├── Header
├── FilterBar (HBoxContainer)
├── BondScroll (ScrollContainer)
│   └── BondList (VBoxContainer)
└── DetailPanel 可选
```

---

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| NPC 列表 | NPCDatabase | 筛选和分组 |
| 缘线数据 | BondSystem | 展示关系 |
| 解锁状态 | KnowledgeSystem | 控制可见性 |
| 最近变化 | EventHistorySystem | 显示 + / - 来源 |

---

## 10. 验收标准

- [ ] 主地图底栏点击缘线 Tab 能打开缘线视图。
- [ ] 缘线条目显示人物、关系对象、关系性质和强度。
- [ ] 关系颜色和文字/图标同时表达关系类型。
- [ ] 点击人物名能打开人物面板。
- [ ] 筛选某个 NPC 后只显示相关缘线。
- [ ] 无关系和未解锁关系有明确状态。
