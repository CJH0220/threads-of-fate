# UI 规格：教程提示 Overlay

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P1  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

教程提示 Overlay 用于在玩家第一次接触关键系统时，提供简短、低打扰的操作说明。它不应打断叙事节奏，也不应一次性塞给玩家过多信息。

---

## 2. 触发场景

| 教程点 | 触发时机 | 说明 |
|---|---|---|
| 主地图教程 | 第一次进入主地图 HUD | 说明地点、事件、推动时间 |
| 人物教程 | 第一次打开人物面板 | 说明行程、缘线、业线 |
| 地点教程 | 第一次打开地点面板 | 说明今日人物和事件 |
| 事件教程 | 第一次进入事件推进面板 | 说明文本推进和结算 |
| 干预教程 | 第一次出现干预选项 | 说明神力消耗和不确定结果 |
| 推动时间教程 | 第一次点击推动时间 | 说明时段推进与错过风险 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 半透明遮罩                            │
│                                      │
│      ┌──────────────────────┐        │
│      │ 这里是当前时段和资源  │        │
│      │ 香火、神力会影响选择  │        │
│      │ [知道了] [不再提示]   │        │
│      └──────────────────────┘        │
│         ↑ 高亮目标 UI 元素             │
└──────────────────────────────────────┘
```

---

## 4. 组件清单

| 组件 | 内容 | 交互 |
|---|---|---|
| 半透明遮罩 | 降低背景干扰 | 点击空白可选关闭 |
| 高亮框 | 指向目标 UI 元素 | 无 |
| 提示面板 | 标题 + 说明文本 | 无 |
| 知道了按钮 | 关闭当前提示 | 点击关闭 |
| 不再提示 | 关闭并记录偏好 | 点击后不再显示该类教程 |
| 下一步按钮 | 多步教程使用 | 进入下一提示 |

---

## 5. 文案原则

- 每条教程不超过 2 句。
- 使用玩家语言，不使用程序术语。
- 强调“你可以做什么”和“为什么重要”。
- 不剧透系统最优解。

示例：

```text
这里显示当前时段。
每次推动时间，居民的位置和事件都会变化。
```

---

## 6. 交互规则

- 教程只在第一次触发时自动显示。
- 玩家可在设置中重置教程。
- 关键教程可强制显示一次，如第一次推动时间。
- 教程不应覆盖确认弹窗或危险警告。
- 减少动画开启时，高亮框不闪烁，只静态显示。

---

## 7. Godot 实现建议

```text
TutorialOverlay (CanvasLayer)
├── DimBackground
├── HighlightFrame
├── TooltipPanel (PanelContainer)
│   ├── TitleLabel
│   ├── BodyText
│   └── ButtonRow
│       ├── NextOrOkButton
│       └── DontShowAgainButton
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 教程触发状态 | TutorialState | 判断是否显示 |
| 目标元素位置 | UI Registry | 定位高亮框 |
| 玩家偏好 | SettingsService | 不再提示 / 重置教程 |
| 当前场景 | NavigationContext | 选择教程内容 |

---

## 9. 验收标准

- [ ] 第一次进入主地图时能显示主地图教程。
- [ ] 教程能高亮目标 UI 元素。
- [ ] 点击“知道了”能关闭教程。
- [ ] 点击“不再提示”后同类教程不再自动出现。
- [ ] 设置中重置教程后可再次显示。
- [ ] 减少动画开启时教程不使用闪烁动效。
