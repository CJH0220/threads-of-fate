> 更新日期：2026-06-10

> 引擎方向：Godot

---

## 0. 全局规范引用

本界面必须遵循：

- `ui-accessibility-guidelines.md`
- `ui-interaction-patterns.md`
- `ui-localization-guidelines.md`

---

## 1. 用途

60 天总评界面用于在一局游戏结束时总结玩家的最终路线、资源结果、NPC 命运和关键事件。它是整局体验的收束界面。

---

## 2. 入口与出口

| 类型 | 说明 |
|---|---|
| 入口 | 第 60 天结束且未触发 Game Over 后进入 |
| 出口 | 进入结局界面 / 返回主菜单 / 保存结局记录 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 60 天总评                             │
├──────────────────────────────────────┤
│ 最终路线：天界善神                    │
│ 香火：126    阳德：132    阴德：24     │
│                                      │
│ 居民命运：                            │
│ 林潮音：好结局                         │
│ 陈远舟：中性结局                       │
│ 顾沉舟：坏结局                         │
│                                      │
│ 关键事件回顾：                         │
│ - 阻止寺庙事故                         │
│ - 港口夜谈改变林潮音命运                │
│                                      │
│ [查看最终结局] [返回主菜单]             │
└──────────────────────────────────────┘
```

---

## 4. 展示内容

| 模块 | 内容 |
|---|---|
| 最终路线 | 正神 / 邪神 / 中性 / 传奇 / 失败 |
| 资源结果 | 香火、阳德、阴德、神力最终值可选 |
| NPC 结局 | 每个关键 NPC 的结局类型 |
| 关键事件 | 本局影响最大的事件列表 |
| 统计 | 干预次数、成功次数、错过事件数可选 |
| 操作 | 查看最终结局、返回主菜单 |

---

## 5. 视觉规则

| 路线 | 表现 |
|---|---|
| 正神 | 金色、暖光、居民供奉 |
| 邪神 | 紫黑、暗红、压抑小镇 |
| 中性 | 暖灰、平淡延续 |
| 传奇 | 金紫混合、传说化光效 |
| 失败 | 暗灰、破败、香火熄灭 |

---

## 6. 交互规则

- 点击 NPC 结局条目可进入角色结局界面，P1。
- 点击关键事件可查看事件回顾，P2。
- 点击查看最终结局进入结局演出界面。
- 返回主菜单前提示是否保存结局记录。

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 最终路线 | EndingSystem | 显示总评类型 |
| 最终资源 | PlayerResourceSystem | 显示结果 |
| NPC 结局 | CharacterEndingSystem | 角色命运列表 |
| 关键事件 | EventHistorySystem | 回顾列表 |
| 结局记录 | GallerySystem | 保存回顾 |

---

## 9. 验收标准

- [ ] 第 60 天结束且未触发 Game Over 时能进入总评界面。
- [ ] 总评界面正确显示最终路线和核心资源。
- [ ] 至少显示关键 NPC 的结局列表。
- [ ] 显示本局关键事件回顾。
- [ ] 点击查看最终结局能进入结局界面或预留入口。
- [ ] 返回主菜单前能保存或提示保存结局记录。
