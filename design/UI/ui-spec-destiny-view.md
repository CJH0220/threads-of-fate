# UI 规格：业线视图

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P1  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

业线视图用于展示 NPC 的命运进度、关键节点、风险和结局倾向。MVP 采用进度条 + 节点列表，后续再升级为命运树。

---

## 2. 入口

| 来源 | 触发 |
|---|---|
| 主地图底栏 | 点击“业线”Tab |
| 人物面板 | 点击业线摘要 |
| 事件结算 | 点击业线变化项 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 业线                                 │
├──────────────────────────────────────┤
│ 筛选：[全部] [林潮音] [主线] [风险]    │
│                                      │
│ 林潮音：寻找自我                      │
│ [■■■■□□] 65%                         │
│ 已完成：课堂争执                       │
│ 当前：港口夜谈                         │
│ 风险：夜晚独自前往港口                  │
│                                      │
│ 陈远舟：离开小镇                       │
│ [■■■□□□] 45%                         │
└──────────────────────────────────────┘
```

---

## 4. 展示字段

| 字段 | 说明 |
|---|---|
| NPC | 业线所属角色 |
| 业线名称 | 主业线 / 支线名称 |
| 类型 | 主线、支线、机会业线 |
| 进度 | 0%–100% |
| 已完成节点 | 已触发的关键事件 |
| 当前节点 | 正在逼近或可干预节点 |
| 风险提示 | 可能导致坏结局的征兆 |
| 结局倾向 | 模糊提示，不直接剧透 |

---

## 5. 视觉规则

| 类型 | 颜色 |
|---|---|
| 主业线 | 金色 |
| 支线 | 蓝 / 绿 / 紫 |
| 机会业线 | 白 / 浅黄 |
| 危险节点 | 红色 |
| 已完成节点 | 灰化或打勾 |

---

## 6. 交互规则

| 操作 | 结果 |
|---|---|
| 点击 NPC 名 | 打开人物面板 |
| 点击业线条目 | 展开节点详情 |
| 点击风险节点 | 查看风险说明，可进入干预面板 |
| 筛选风险 | 只显示存在危险节点的业线 |

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 无业线 | “命运尚未显形” |
| 未解锁业线 | 显示“未知命运” |
| 业线完成 | 显示完成标签，可进入角色结局 |
| 业线失败 | 显示失败标签和原因摘要 |
| 危险逼近 | 红色提示和警告图标 |

---


---

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| NPC 列表 | NPCDatabase | 分组展示 |
| 业线数据 | DestinySystem | 进度和节点 |
| 风险状态 | RiskSystem | 风险提示 |
| 解锁状态 | KnowledgeSystem | 控制未知信息 |
| 结局状态 | EndingSystem | 完成 / 失败显示 |

---

## 10. 验收标准

- [ ] 主地图底栏点击业线 Tab 能打开业线视图。
- [ ] 每条业线显示 NPC、名称、类型和进度。
- [ ] 当前节点和风险节点能显示清楚。
- [ ] 点击 NPC 能打开人物面板。
- [ ] 点击风险节点能进入详情或干预入口。
- [ ] 无业线、未解锁、完成、失败状态均有明确表现。
