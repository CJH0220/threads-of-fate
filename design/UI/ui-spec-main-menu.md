# UI 规格：主菜单

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

主菜单是玩家进入游戏后的外层导航中枢，用于开始新游戏、继续游戏、加载存档、调整设置、查看关于信息和退出游戏。

主菜单应建立游戏第一印象：温暖的小镇、土地庙、轻微神秘感，以及“命运即将被编织”的氛围。

---

## 2. 玩家到达场景

玩家在以下情况进入主菜单：

- 启动游戏后进入。
- 从游戏内暂停菜单返回主菜单。
- 游戏结束后返回主菜单。
- 读取失败或异常后返回主菜单。

玩家此时的目标是快速进入游戏或管理基础设置。

---

```text
┌──────────────────────────────────────┐
│              像素风小镇背景           │
│                                      │

│                                      │
```

## 4. 组件清单

| 组件 | 类型 | 内容 | 交互 |
|---|---|---|---|
| 背景图 | `TextureRect` | 土地庙 / 归潮镇远景 | 无 |
| 游戏 Logo | `TextureRect` / `Label` | 《命运的织线》 | 无 |
| 继续游戏 | `Button` | 读取最近存档 | 无存档时灰化或隐藏 |
| 新游戏 | `Button` | 进入创建游戏界面 | 点击跳转 |
| 加载游戏 | `Button` | 进入读档列表 | 点击跳转 |
| 设置 | `Button` | 打开设置界面 | 点击跳转 |
| 关于我们 | `Button` | 打开制作信息 | 点击跳转 |
| 退出游戏 | `Button` | 弹出退出确认 | 二次确认 |
| 版本号 | `Label` | 如 v0.1.0 | 无 |

---

## 5. 视觉规范

- 背景建议使用静态像素图：土地庙、小镇入口、清晨或黄昏。
- 按钮使用木牌、布条或纸符样式。
- 选中按钮时边框发亮或按钮轻微上浮。
- 鼠标按下时按钮下压 1–2 像素。
- Logo 可使用像素字体，配合细微香火烟雾动画。

---

## 6. 交互规则

| 操作 | 结果 |
|---|---|
| 点击继续游戏 | 加载最近存档并进入主地图 HUD |
| 点击新游戏 | 进入新游戏 / 创建游戏界面 |
| 点击加载游戏 | 进入读档列表 |
| 点击设置 | 打开设置界面 |
| 点击关于我们 | 打开关于我们界面 |
| 点击退出游戏 | 弹出退出确认 |
| Esc | 若无弹窗，不做操作或聚焦退出按钮 |

键盘支持：

- 上 / 下：切换按钮焦点
- Enter / Space：确认当前按钮
- Esc：关闭当前子弹窗

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 首次启动无存档 | “继续游戏”隐藏或灰化 |
| 有存档 | 显示“继续游戏” |
| 正在加载 | 按钮禁用，显示“命运织线展开中……” |
| 读档失败 | 显示错误提示，停留主菜单 |
| 退出确认 | 显示通用确认弹窗 |

---

## 8. Godot 实现建议

```text
MainMenu (Control)
├── Background (TextureRect)
├── SmokeEffect (AnimatedSprite2D / GPUParticles2D 可选)
├── MenuRoot (VBoxContainer)
│   ├── Logo (TextureRect / Label)
│   ├── ContinueButton (Button)
│   ├── NewGameButton (Button)
│   ├── LoadGameButton (Button)
│   ├── SettingsButton (Button)
│   ├── CreditsButton (Button)
│   └── QuitButton (Button)
├── VersionLabel (Label)
└── ConfirmDialogLayer (CanvasLayer)
```

---

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 是否存在存档 | SaveSystem | 控制继续游戏显示状态 |
| 最近存档路径 | SaveSystem | 继续游戏 |
| 游戏版本号 | ProjectSettings / 常量 | 显示版本 |
| 设置数据 | SettingsService | 从设置界面返回后保持状态 |

---

## 10. 验收标准

- [ ] 无存档时，“继续游戏”不会进入错误状态。
- [ ] 有存档时，“继续游戏”可进入最近存档。
- [ ] 新游戏、加载游戏、设置、关于我们按钮均能跳转到正确界面。
- [ ] 退出游戏必须出现二次确认。
- [ ] 鼠标 hover、点击、键盘焦点均有清晰反馈。
- [ ] 主菜单在 16:9 分辨率下布局不遮挡、不溢出。
