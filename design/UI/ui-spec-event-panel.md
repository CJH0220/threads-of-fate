- `ui-accessibility-guidelines.md`

## 1. 用途

事件推进面板用于呈现关键事件的叙事文本、参与角色、玩家干预、事件结果和结算反馈。它是“观察 → 干预 → 结果反馈”核心循环中最重要的反馈界面。

---

## 2. 入口与出口

### 入口

| 来源 | 触发 |
|---|---|
| 主地图 | 点击事件标记 |
| 地点面板 | 点击地点事件 |
| 推动时间 | 时间推进后自动触发关键事件 |
| 人物面板 | 点击人物相关事件 |

### 出口

| 操作 | 结果 |
|---|---|
| 事件未结束时点击继续 | 推进下一段文本 |
| 结算阶段点击确定 | 按后续跳转规则进入下一界面 |
| Esc | 非关键事件可退出；关键事件需确认 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────────────┐
│ 事件名：港口的夜色                  [Auto]   │
├───────────────┬──────────────────────────────┤
│ 参与角色       │ 事件文本区                    │
│ 林潮音 头像     │ “她站在港口边，像是在等待……” │
│ 陈远舟 头像     │                              │
├───────────────┴──────────────────────────────┤
│ 干预选项 / 结果变化 / 结算摘要                │
│ [继续] [干预] [确定]                          │
└──────────────────────────────────────────────┘
```

---

## 4. 事件阶段

```text
进入事件
→ 显示事件标题与参与角色
→ 分段播放事件文本
→ 可选：玩家干预
→ 展示角色反应
→ 展示资源 / 缘线 / 业线变化
→ 点击确定返回主地图
```

---

## 5. 组件清单

| 组件 | 类型 | 内容 | 交互 |
|---|---|---|---|
| 事件标题 | `Label` | 事件名 | 无 |
| 参与角色 | 头像列表 | 角色头像、姓名 | 点击打开人物面板，关键演出中可禁用 |
| 事件文本 | `RichTextLabel` | 分段文本 | 点击推进 / Space 推进 |
| 演出区 | `TextureRect` / 图标 | 简单背景、表情、图标 | 无 |
| 干预区 | `Button` 列表 | 可用干预选项 | 点击进入干预确认 |
| 结算区 | 列表 | 资源、关系、业线变化 | 无 |
| Auto | `ToggleButton` | 自动播放 | 切换 Auto |
| 继续 | `Button` | 下一段文本 | 点击推进 |
| 确定 | `Button` | 结束事件 | 结算阶段可用 |

---

## 6. 文本推进规则

| 操作 | 结果 |
|---|---|
| 点击文本区 | 若正在逐字显示，则立即显示完整句；否则进入下一段 |
| 点击继续 | 同上 |
| Space | 同上 |
| Auto 开启 | 按设置的 Auto 速度自动推进 |
| Esc | 非关键事件退出；关键事件弹确认 |

---

## 7. 干预规则

干预选项应明确显示：

| 字段 | 说明 |
|---|---|
| 干预名称 | 如托梦、赐福、遮蔽信息、牵引机缘 |
| 神力消耗 | 如消耗 2 神力 |
| 模糊影响 | 如“可能让她鼓起勇气” |
| 风险提示 | 如“也可能影响另一人的判断” |

规则：

- 神力不足时选项灰化。
- 消耗神力前必须二次确认。
- 不直接剧透最终结果。
- MVP 可只提供预设干预，不接入自然语言输入。

---

## 8. 结算展示

| 类型 | 表现 |
|---|---|
| 香火变化 | 香火 +3 / -2 |
| 神力变化 | 神力 -2 |
| 阳德 / 阴德 | 阳德 +5 / 阴德 +3 |
| 缘线变化 | 林潮音 ↔ 陈远舟 友情 +5 |
| 业线变化 | 林潮音主业线推进至 65% |
| 新线索 | 解锁“港口夜谈” |

结算以列表方式展示，不使用复杂动画。可配合小图标和数字跳动。

---

## 9. 后续跳转规则

事件结算完成后，点击“确定”按以下规则跳转：

1. 若事件结果直接触发 Game Over，进入 `ui-spec-game-over.md`。
2. 若当前由时间推进队列触发，且仍有待播放关键事件，进入下一条关键事件。
3. 若当前由时间推进队列触发，且所有关键事件已完成，返回 `ui-spec-time-advance.md` 的后续跳转判断。
4. 若玩家从主地图、地点面板或人物面板主动打开事件，返回来源界面；来源缺失时返回主地图 HUD。
5. 若事件解锁新线索，显示 Toast 或在结算列表中标记，不中断返回流程。

---

## 10. 状态与变体

| 状态 | 表现 |
|---|---|
| 普通事件 | 标准文本 + 结果 |
| 关键事件 | 退出需确认，标题有特殊标记 |
| 危险事件 | 红色边框 / 警告图标 |
| 已读事件 | 可快速跳过已读文本，若设置允许 |
| AI 文本失败 | 使用预设模板文本兜底 |
| 无可用干预 | 显示“此刻只能旁观” |
| 神力不足 | 干预按钮灰化 |

```text
EventPanel (PanelContainer)
├── Header (HBoxContainer)
│   ├── EventTitleLabel
│   └── AutoToggle
├── Content (HBoxContainer)

│   └── TextAndStage (VBoxContainer)

│       └── EventText (RichTextLabel)
├── ResultOrChoiceArea (VBoxContainer)
└── FooterButtons (HBoxContainer)
    ├── ContinueButton
    ├── InterveneButton
    └── ConfirmButton
```

## 12. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 事件基础信息 | EventDatabase | 标题、地点、参与角色 |
| 事件文本 | EventNarrativeSystem | 分段文本 |
| 参与角色 | NPCDatabase | 头像、姓名 |
| 干预选项 | InterventionSystem | 可用干预列表 |
| 神力值 | PlayerResourceSystem | 控制干预可用性 |
| 结算结果 | EventResolutionSystem | 资源、关系、业线变化 |
| 已读状态 | EventHistorySystem | 控制跳过已读 |

---

## 13. 验收标准

- [ ] 点击事件标记后能打开事件推进面板。
- [ ] 事件标题、参与角色和第一段文本显示正确。
- [ ] 点击文本区、继续按钮或 Space 能推进文本。
- [ ] Auto 开关能自动推进文本，并受设置中的 Auto 速度影响。
- [ ] 神力不足时干预按钮灰化并显示原因。
- [ ] 执行干预前必须出现消耗确认。
- [ ] 事件结束后展示资源、缘线或业线变化。
- [ ] 点击确定后返回主地图并刷新相关状态。
- [ ] AI 文本失败时能显示预设兜底文本。
