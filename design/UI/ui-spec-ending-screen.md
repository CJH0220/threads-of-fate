# UI 规格：最终结局界面

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P0 文本版 / P1 演出版  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

最终结局界面用于展示整局游戏的最终路线结果，如正神、邪神、中性、传奇或失败结局。它承接 60 天总评，是整局叙事的最终落点。

---

## 2. 入口与出口

| 类型 | 说明 |
|---|---|
| 入口 | 60 天总评点击“查看最终结局” |
| 失败入口 | Game Over 后可直接进入失败结局演出，P1 |
| 出口 | 返回主菜单 / 查看结局回顾 / 重新开始 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 天界善神结局                          │
├──────────────────────────────────────┤

│                                      │
│ [结局回顾] [重新开始] [返回主菜单]       │
└──────────────────────────────────────┘
```

---

## 4. 结局类型

| 结局 | 条件方向 | 视觉方向 |
|---|---|---|
| 失败结局 | 香火不足或强硬不幸结局 | 暗灰、香火熄灭 |
| 邪神结局 | 阴德高、阳德低 | 紫黑、暗红、小镇压抑 |
| 天界善神结局 | 阳德高、阴德低 | 金色、暖光、居民供奉 |
| 中性结局 | 香火达标但阴阳德不足 | 暖灰、平淡继续 |
| 传奇结局 | 香火、阳德、阴德均高 | 金紫混合、传说化 |

---

## 5. 展示模块

| 模块 | 内容 |
|---|---|
| 结局标题 | 最终结局名 |
| 结局插图 | 一张像素插图或静态背景 |
| 结局文本 | 2–4 段叙事收束 |
| 最终评价 | 如守护者、旁观者、腐化者、传说 |
| 操作按钮 | 结局回顾、重新开始、返回主菜单 |

---

## 6. 交互规则

- 首次进入最终结局时，保存结局记录。
- 点击结局回顾进入图鉴 / 本局回顾。
- 点击重新开始进入新游戏界面，需确认。
- 点击返回主菜单直接回到主菜单。
- Esc 不直接退出结局界面，防止误跳过。

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 首次解锁 | 显示“新结局已记录” |
| 已解锁回看 | 不重复记录，只展示 |
| 无结局插图 | 使用通用像素背景 |
| 失败结局 | 可显示读取存档入口 |

---

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 最终结局类型 | EndingSystem | 选择文本与视觉 |
| 结局文本 | NarrativeDatabase | 展示结局 |
| 最终评价 | EndingSystem | 显示评价 |
| 是否首次解锁 | GallerySystem | 记录提示 |
| 结局插图 | AssetDatabase | 显示背景 |

---

## 10. 验收标准

- [ ] 60 天总评后能进入最终结局界面。
- [ ] 不同结局类型显示不同标题、文本和视觉风格。
- [ ] 首次解锁结局时能记录到结局回顾。
- [ ] 重新开始前必须二次确认。
- [ ] 返回主菜单按钮可正常返回。
- [ ] Esc 不会误跳过结局界面。
