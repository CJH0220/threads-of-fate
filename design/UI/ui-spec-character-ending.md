# UI 规格：角色结局界面

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P1  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

角色结局界面用于展示单个 NPC 的业线最终结果，包括结局类型、原因摘要、关键事件和对玩家路线资源的影响。它让玩家理解某个角色为什么走向了这个结局。

---

## 2. 触发时机

| 触发 | 说明 |
|---|---|
| 某 NPC 主业线达到 100% | 触发角色结局 |
| 某 NPC 主业线失败 | 触发失败结局 |
| 60 天总评中点击角色条目 | 回顾该角色结局 |
| 结局回顾图鉴中点击角色 | 回顾已解锁结局 |

---

```text
┌──────────────────────────────────────┐

├──────────────────────────────────────┤

│                                      │

│                                      │

```

---

## 4. 展示模块

| 模块 | 内容 |
|---|---|
| 角色名 | NPC 姓名 |
| 结局类型 | 好结局 / 坏结局 / 中性结局 / 特殊结局 |
| 结局插图 | 像素头像或简单结局插画 |
| 结局文本 | 1–3 段叙事说明 |
| 原因摘要 | 关键事件和状态条件 |
| 资源影响 | 阳德、阴德、香火变化 |
| 后续入口 | 查看相关事件 / 继续 |

---

## 5. 结局类型视觉规则

| 类型 | 表现 |
|---|---|
| 好结局 | 暖色、金色边框、柔和光效 |
| 坏结局 | 暗色、红紫边框、低饱和 |
| 中性结局 | 暖灰、平静、无强烈特效 |
| 特殊结局 | 根据剧情自定义，但保持像素风 |

---

## 6. 交互规则

- 点击继续返回上一流程：事件结算、周结算或 60 天总评。
- 点击相关事件进入事件回顾，P2。
- 若多个角色同时触发结局，按重要性排队展示。
- 首次解锁角色结局时，可显示“已记录到结局回顾”。

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 首次触发 | 正常展示并记录图鉴 |
| 回顾模式 | 不重复发放资源，只展示历史结果 |
| 多角色连续结局 | 显示“下一个”按钮 |
| 结局数据缺失 | 使用兜底文案和通用插图 |

---

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| NPC 信息 | NPCDatabase | 姓名、头像 |
| 结局类型 | CharacterEndingSystem | 视觉和标题 |
| 结局文本 | NarrativeDatabase / 模板 | 展示文本 |
| 关键原因 | EventHistorySystem / DestinySystem | 原因列表 |
| 资源影响 | RewardSystem | 显示变化 |
| 是否首次解锁 | GallerySystem | 控制记录提示 |

---

## 10. 验收标准

- [ ] NPC 主业线完成或失败时能触发角色结局界面。
- [ ] 角色结局界面显示角色名、结局类型和结局文本。
- [ ] 原因摘要至少列出 1 个关键原因。
- [ ] 资源影响显示正确。
- [ ] 回顾模式不会重复发放资源。
- [ ] 多角色连续结局能按队列展示。
