# UI 规格：启动画面

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P2  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

启动画面用于展示游戏 Logo、初始化资源，并在进入主菜单前建立小镇与土地庙的第一印象。

---

## 2. 流程

```text
启动游戏
→ 显示制作组 / 引擎初始化
→ 显示游戏 Logo 与像素背景
→ 初始化完成后显示“点击任意处继续”
→ 进入主菜单
```

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│                                      │
│          像素风土地庙 / 香炉          │
│                                      │
│            《命运的织线》              │
│                                      │
│          点击任意处继续               │
│                                      │
└──────────────────────────────────────┘
```

---

## 4. 视觉建议

- 背景：土地庙、归潮镇远景、晨雾、香火。
- 动效：Logo 淡入、香火轻微飘动。
- 减少动画开启时：静态背景 + 静态 Logo。
- 不建议加载过多高成本动画。

---

## 5. 交互规则

- 初始化未完成前，不显示“点击任意处继续”。
- 初始化完成后，鼠标点击、Enter、Space 可进入主菜单。
- 若加载失败，显示错误提示和重试按钮。
- 首次启动可停留稍久，重复启动应尽快进入主菜单。

---

## 6. Godot 实现建议

```text
SplashScreen (Control)
├── Background (TextureRect)
├── Logo (TextureRect / Label)
├── LoadingLabel
├── ContinuePrompt
└── ErrorPanel 可选
```

---

## 7. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 初始化状态 | BootService | 控制提示显示 |
| 版本号 | ProjectSettings | 可选显示 |
| 错误信息 | BootService | 显示失败原因 |

---

## 8. 验收标准

- [ ] 启动游戏后先进入启动画面。
- [ ] 初始化完成前不能进入主菜单。
- [ ] 初始化完成后点击或按键可进入主菜单。
- [ ] 加载失败时显示错误和重试入口。
- [ ] 减少动画开启时启动画面无动态烟雾或闪烁。
