# UI 组件规格：空 / 加载 / 错误状态

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

空 / 加载 / 错误状态组件用于所有列表、面板、异步流程和数据缺失场景，保证玩家不会看到空白界面或误以为游戏卡死。

---

## 2. 状态类型

| 类型 | 用途 | 示例 |
|---|---|---|
| 空状态 | 数据为空但系统正常 | 当前没有事件 |
| 加载状态 | 系统正在处理 | 命运织线生成中…… |
| 错误状态 | 操作失败或数据异常 | 读取存档失败 |
| 禁用状态 | 功能暂不可用 | 查询能力尚未解锁 |

---

## 3. 布局规格

```text
┌──────────────────────────────┐
│          [像素图标]            │
│                               │
│        此刻风平浪静             │
│    当前时段没有可查看事件。      │
│                               │
│        [可选操作按钮]           │
└──────────────────────────────┘
```

---

## 4. 文案示例

| 场景 | 标题 | 说明 |
|---|---|---|
| 无存档 | 尚无存档 | 请从新游戏开始第一段命运。 |
| 无事件 | 此刻风平浪静 | 当前时段没有可查看事件。 |
| 地点无人 | 无人停留 | 这个时段没有居民来到这里。 |
| 未选中人物 | 尚未选择居民 | 点击地图上的居民查看命运。 |
| 加载存档 | 命运织线展开中 | 正在读取存档。 |
| 时间推进 | 命运流转中 | 居民正在行动，事件正在变化。 |
| AI 失败 | 命运雾气散乱 | 暂时无法生成文本，已使用默认描述。 |
| 读档失败 | 读取失败 | 存档可能损坏或版本不兼容。 |

---

## 5. 交互规则

- 空状态可提供下一步按钮，如“开始新游戏”“返回主地图”。
- 加载状态必须有明确文案，不允许纯空白。
- 长时间加载超过设定阈值时，应显示“仍在处理”或重试入口，P1。
- 错误状态必须说明结果和下一步，而不是只显示错误码。
- 禁用状态必须说明解锁条件或暂不可用原因。

---

## 7. 数据接口

建议统一调用：

```text
StateMessage.set_state(type, title, description, action_text, action_callback)
```

---

## 8. 验收标准

- [ ] 无存档、无事件、地点无人、未选中人物均有空状态。
- [ ] 读档、时间推进、AI 查询等流程均有加载状态。
- [ ] 读档失败、生成失败等错误均有明确提示。
- [ ] 空状态和错误状态可提供下一步操作。
- [ ] 所有状态文案可被本地化替换。
