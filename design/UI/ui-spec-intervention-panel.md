# UI 规格：干预选择面板

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P1  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

干预选择面板用于让玩家消耗神力，对人物、地点、事件或业线节点进行有限干预。它必须强调：玩家只能轻推命运，不能直接控制人物。

---

## 2. 入口

| 来源 | 示例 |
|---|---|
| 人物面板 | 对某人托梦 / 赐福 |
| 地点面板 | 对某地点制造机缘 |
| 事件面板 | 在事件中途轻微干预 |
| 业线视图 | 推动或遮蔽某个节点 |
| 主地图 | 对地点或事件标记使用干预 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 干预命运：林潮音                      │
├──────────────────────────────────────┤
│ 当前神力：4                           │
│                                      │
│ [托梦]      消耗 2  让她隐约感到不安   │
│ [赐福]      消耗 1  提高行动顺利概率   │
│ [牵引机缘]  消耗 3  增加一次偶遇机会   │
│ [遮蔽信息]  消耗 2  让某条信息暂时错过 │
│                                      │
│ [确认干预]                [取消]      │
└──────────────────────────────────────┘
```

---

## 4. 干预卡片字段

| 字段 | 说明 |
|---|---|
| 干预名称 | 托梦、赐福、牵引机缘、遮蔽信息等 |
| 神力消耗 | 明确数字 |
| 简短说明 | 模糊描述可能影响 |
| 风险提示 | 若有副作用，给出非剧透提示 |
| 可用状态 | 神力不足、条件不满足时灰化 |

---

## 5. 交互规则

- 玩家选择一个干预项后，“确认干预”可用。
- 神力不足的干预项灰化，Hover 显示原因。
- 确认干预前必须二次确认消耗。
- 干预结果不直接剧透，只在后续事件或结算中反馈。
- MVP 不接入自然语言输入，只使用预设干预项。

---

## 6. 状态与变体

| 状态 | 表现 |
|---|---|
| 有可用干预 | 显示干预列表 |
| 无可用干预 | 显示“此刻只能旁观” |
| 神力不足 | 干预项灰化 |
| 已选择干预 | 卡片高亮 |
| 干预确认 | 弹出确认消耗 |
| 干预成功 | 返回来源界面并显示反馈 |
| 干预失败 | 显示原因，不扣资源 |

---

## 7. Godot 实现建议

```text
InterventionPanel (PanelContainer)
├── Header
├── DivinePowerLabel
├── InterventionList (VBoxContainer)
│   └── InterventionCard
├── WarningText
└── ButtonRow
    ├── ConfirmButton
    └── CancelButton
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 当前神力 | PlayerResourceSystem | 判断可用性 |
| 目标对象 | NavigationContext | 显示干预目标 |
| 可用干预列表 | InterventionSystem | 生成卡片 |
| 干预成本 | InterventionConfig | 显示消耗 |
| 干预结果 | InterventionResolver | 执行并反馈 |

---

## 9. 验收标准

- [ ] 从人物、地点或事件入口能打开干预面板。
- [ ] 干预项显示名称、消耗和说明。
- [ ] 神力不足时对应干预项不可选。
- [ ] 未选择干预时确认按钮不可用。
- [ ] 确认干预前必须出现消耗确认。
- [ ] 干预成功后扣除神力并返回来源界面。
- [ ] 无可用干预时显示明确空状态。
