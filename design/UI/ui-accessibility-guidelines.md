# UI 规范：可访问性基线

> 文档状态：初稿  
> 更新日期：2026-06-12  
> 所属阶段：P0 全局规范  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

本文定义《命运的织线》所有 UI 的最低可访问性要求。所有 P0 界面、通用组件和后续新增界面都必须遵守本规范。

目标不是做复杂辅助系统，而是保证玩家至少可以清楚阅读、键盘操作、理解风险，并且不会因为颜色、动画或鼠标精度限制而无法游玩。

---

## 2. 支持基线

| 项 | 要求 |
|---|---|
| 目标平台 | PC |
| 主要输入 | 鼠标 + 键盘 |
| 必须支持 | 鼠标点击、键盘导航、Esc 返回 |
| 后续可选 | 手柄焦点导航 |
| 不支持 | 触屏专用交互 |

P0 阶段默认达到“基础可访问性”：

- 所有核心流程可用鼠标完成。
- 所有菜单、弹窗、设置、存档界面可用键盘完成。
- 关键风险信息不只靠颜色表达。
- 文本可读，字号可调。
- 动画可减少或关闭。

---

## 3. 键盘与焦点

### 3.1 通用按键

| 按键 | 行为 |
|---|---|
| Tab | 移动到下一个可交互元素 |
| Shift + Tab | 移动到上一个可交互元素 |
| ↑ / ↓ / ← / → | 在同组控件内移动焦点 |
| Enter | 确认当前焦点控件 |
| Space | 触发按钮 / Toggle |
| Esc | 返回上一级、关闭弹窗或打开暂停菜单 |
| 1 / 2 / 3 / 4 | 主地图底栏 Tab 快捷键，P1 |

### 3.2 焦点显示

所有可交互元素必须有清晰焦点态：

- 边框加亮或外框描边。
- 不只依赖轻微颜色变化。
- 焦点态应比 hover 更明显。
- 禁用控件不可获得焦点。

### 3.3 焦点顺序

焦点顺序必须符合视觉阅读顺序：

1. 从上到下。
2. 从左到右。
3. 主要行动按钮排在次要行动前。
4. 危险确认弹窗默认聚焦安全按钮。

---

## 4. P0 界面焦点顺序

### 4.1 主菜单

```text
继续游戏 → 新游戏 → 加载游戏 → 设置 → 关于我们 → 退出游戏
```

无存档时，如果“继续游戏”灰化，则跳过该按钮。

### 4.2 新游戏 / 创建存档

```text
存档槽列表 → 模式选择 → 开始编织命运 → 返回
```

### 4.3 加载游戏 / 读档

```text
存档槽列表 → 加载 → 删除 / 覆盖相关操作 → 返回
```

删除和覆盖操作必须进入确认弹窗，不能单次按键直接执行。

### 4.4 设置界面

```text
分类列表 → 当前分类设置项 → 恢复默认 → 应用 → 返回 / 关闭
```

Slider 获得焦点后：

- ← / → 调整数值。
- Shift + ← / → 大步调整。
- Esc 离开设置界面前检查未保存更改。

### 4.5 主地图 HUD

```text
设置 → 左侧提示区 → 地图地点 / 事件标记 → 底栏 Tab → 上下文列表 → 推动时间
```

地图区域中的地点焦点顺序应按屏幕位置从左上到右下排列。危险事件标记必须排在普通事件标记之前。

### 4.6 暂停菜单

```text
继续游戏 → 保存游戏 → 加载游戏 → 设置 → 返回主菜单 → 退出游戏
```

Esc 等同“继续游戏”。

### 4.7 通用确认弹窗

普通确认：

```text
确认 → 取消
```

危险确认：

```text
取消 / 返回查看 → 危险确认按钮
```

危险确认中 Enter 默认触发安全按钮，不触发危险按钮。

## 5. 文本可读性

| 项 | 要求 |
|---|---|
| 正文字号 | 不低于 18px 等效尺寸 |
| 主要按钮 | 不低于 20px 等效尺寸 |
| 标题 | 不低于 24px 等效尺寸 |
| 小字说明 | 不低于 15px 等效尺寸，避免承载关键信息 |
| 行距 | 至少为字号的 1.2 倍 |

设置界面必须提供：

- 标准字号。
- 大字号。
- 特大字号，P1。

字号变大时优先允许滚动，不压缩文字到不可读。

## 6. 颜色与对比

### 6.1 颜色不作为唯一信息来源

以下信息必须同时使用文字、图标、形状或排序辅助表达：

- 危险等级。
- 关系性质。
- 业线风险。
- 资源增减。
- 按钮状态。
- 错误 / 成功提示。

示例：

```text
错误：红色 + “!” 图标 + “读取失败”文字
危险事件：红色边框 + 感叹号 + 危险文案
关系类型：颜色条 + 关系名称 + 数值
```

### 6.2 对比要求

| 元素 | 要求 |
|---|---|
| 正文 | 深色文字配浅色底，或浅色文字配深色底 |
| 危险按钮 | 暗红底 + 浅色文字 + 深色描边 |
| 禁用按钮 | 降低饱和度，但仍能读清文字 |
| 焦点框 | 与背景明显区分 |

## 7. 动画与动效

设置界面必须提供“减少动画”。开启后：

- 禁用循环闪烁。
- 禁用大幅位移和缩放。
- Toast 改为淡入淡出或直接显示 / 隐藏。
- 危险提示由脉冲改为静态高亮。
- 时间推进保留必要反馈，但不播放强烈屏幕震动。

默认动效应短、轻、可跳过：

| 动效 | 建议时长 |
|---|---|
| 按钮按下反馈 | 0.05–0.12 秒 |
| 面板打开 / 关闭 | 0.12–0.25 秒 |
| Toast 淡入淡出 | 0.15–0.3 秒 |
| 时间推进过渡 | 0.8–1.5 秒，可跳过 |

---

## 8. 危险操作保护

以下操作必须二次确认：

- 删除存档。
- 覆盖存档。
- 退出游戏。
- 返回主菜单且存在未保存进度。
- 恢复默认设置。
- 消耗神力进行关键干预。
- 未查看危险事件时推动时间。

危险确认必须满足：

- 标题明确说明操作。
- 正文说明后果。
- 确认按钮使用具体动词，如“删除存档”“继续推动”。
- 默认焦点在取消或返回查看。
- Esc 等同取消。

---

## 9. 音频与提示

所有关键反馈不能只依赖音效：

| 反馈 | 必须有视觉 / 文本表现 |
|---|---|
| 保存成功 | Toast 文案 |
| 保存失败 | 错误 Toast / 错误状态 |
| 神力不足 | 禁用状态 + 提示文案 |
| 危险事件 | HUD 图标 + 文案 |
| 新线索 | Toast + 事件列表标记 |

---

## 10. 本地化与文本扩展

为后续本地化预留空间：

- 按钮文本预留约 40% 扩展宽度。
- 文本密集面板必须支持滚动。
- 不把关键文案写进图片。
- UI 文案使用文本 key 管理，不写死在脚本中。
- 中文省略号统一使用“……”。

---

## 11. 验收标准

- [ ] P0 菜单和弹窗可只用键盘完成操作。
- [ ] 所有可交互元素都有焦点态。
- [ ] 危险确认默认焦点不在危险按钮上。
- [ ] 关键风险信息不只依赖颜色。
- [ ] 设置界面包含字号、减少动画、高对比 UI、颜色辅助标记。
- [ ] Toast、危险提示、错误状态均有文字说明。
- [ ] 减少动画开启后，不播放循环闪烁或大幅位移动画。
- [ ] 字号变大时，核心文本不溢出屏幕。
