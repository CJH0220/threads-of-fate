# UI 规格：暂停菜单

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

暂停菜单用于游戏内暂停、保存、加载、设置、返回主菜单和退出游戏。

---

## 2. 入口与出口

| 类型 | 说明 |
|---|---|
| 入口 | 游戏内按 Esc 或点击设置按钮 |
| 出口 | 继续游戏 / 返回主菜单 / 退出游戏 |

---

## 3. 布局规格

```text
┌──────────────────────────┐
│ 暂停                     │
├──────────────────────────┤
│ [继续游戏]               │
│ [保存游戏]               │
│ [加载游戏]               │
│ [设置]                   │
│ [返回主菜单]             │
│ [退出游戏]               │
└──────────────────────────┘
```

---

## 4. 交互规则

| 操作 | 结果 |
|---|---|
| 继续游戏 | 关闭暂停菜单 |
| 保存游戏 | 保存当前进度并显示 Toast |
| 加载游戏 | 打开读档列表 |
| 设置 | 打开设置界面 |
| 返回主菜单 | 二次确认后返回主菜单 |
| 退出游戏 | 二次确认后退出 |
| Esc | 关闭暂停菜单 |

规则：

- 时间推进、事件结算中不可打开暂停菜单，除非进入安全暂停状态。
- 保存失败必须显示错误提示。
- 返回主菜单和退出游戏都属于危险操作，必须二次确认。

---

## 5. 状态与变体

| 状态 | 表现 |
|---|---|
| 正常可保存 | 保存按钮可用 |
| 当前不可保存 | 保存按钮灰化，Hover 显示原因 |
| 保存中 | 按钮禁用，显示“保存中……” |
| 保存成功 | Toast：“命运已记录” |
| 保存失败 | Toast / 错误条提示 |

---

## 6. Godot 实现建议

```text
PauseMenu (CanvasLayer)
├── DimBackground
├── PanelContainer
│   └── ButtonList (VBoxContainer)
│       ├── ResumeButton
│       ├── SaveButton
│       ├── LoadButton
│       ├── SettingsButton
│       ├── MainMenuButton
│       └── QuitButton
└── ConfirmDialogLayer
```

---

## 7. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 当前是否可保存 | GameStateService | 控制保存按钮 |
| 当前存档槽 | SaveSystem | 保存位置 |
| 未保存状态 | SaveSystem | 返回主菜单提示 |

---

## 8. 验收标准

- [ ] 游戏内按 Esc 能打开和关闭暂停菜单。
- [ ] 继续游戏按钮能恢复游戏。
- [ ] 保存游戏成功后显示 Toast。
- [ ] 不可保存状态下保存按钮灰化并说明原因。
- [ ] 加载游戏按钮能打开读档列表。
- [ ] 设置按钮能打开设置界面并可返回暂停菜单。
- [ ] 返回主菜单和退出游戏前必须二次确认。
