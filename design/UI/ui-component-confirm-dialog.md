# UI 组件规格：通用确认弹窗

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

通用确认弹窗用于所有需要玩家二次确认的操作，尤其是会导致资源消耗、进度丢失、存档覆盖或退出游戏的操作。

---

## 2. 使用场景

| 场景 | 示例 |
|---|---|
| 退出游戏 | 确定要离开归潮镇吗？ |
| 返回主菜单 | 未保存的进度可能丢失 |
| 覆盖存档 | 该存档已有记录，确定覆盖？ |
| 删除存档 | 删除后无法恢复 |
| 消耗神力 | 将消耗 2 点神力进行干预 |
| 推动时间 | 仍有危险事件未查看，确定继续？ |
| 恢复默认设置 | 当前设置将被重置 |

---

## 3. 布局规格

```text
┌──────────────────────────────┐
│ 标题：确定推动时间？           │
├──────────────────────────────┤
│ 仍有危险征兆尚未查看。         │
│ 如果继续，可能错过干预机会。   │
│                              │
│ [确认继续]        [取消]      │
└──────────────────────────────┘
```

---

## 4. 组件清单

| 组件 | 内容 |
|---|---|
| 遮罩 | 半透明背景，阻止误点其他 UI |
| 标题 | 清楚说明操作 |
| 正文 | 说明后果 |
| 确认按钮 | 执行操作 |
| 取消按钮 | 关闭弹窗 |
| 危险图标 | 危险操作时显示 |

---

## 5. 类型

| 类型 | 用途 | 视觉 |
|---|---|---|
| 普通确认 | 设置应用、普通跳转 | 常规边框 |
| 警告确认 | 推动时间、放弃更改 | 橙色边框 |
| 危险确认 | 删除存档、退出、不可逆干预 | 红色边框 + 图标 |

---

## 6. 交互规则

- Esc 等同取消。
- Enter 默认触发安全按钮：通常是取消，不是确认危险操作。
- 危险操作的确认按钮应使用明确文案，如“删除存档”，避免只写“确定”。
- 点击弹窗外不关闭危险确认，普通确认可配置。
- 弹窗打开时，背景 UI 不可操作。

---

## 7. Godot 实现建议

```text
ConfirmDialog (CanvasLayer)
├── DimBackground
└── DialogPanel (PanelContainer)
    ├── TitleLabel
    ├── MessageLabel
    └── ButtonRow
        ├── ConfirmButton
        └── CancelButton
```

---

## 8. 数据接口

建议统一调用：

```text
ConfirmDialog.show(title, message, confirm_text, cancel_text, type, on_confirm)
```

---

## 9. 验收标准

- [ ] 退出游戏、删除存档、覆盖存档、消耗神力均使用确认弹窗。
- [ ] 危险操作有红色或明显警告样式。
- [ ] Esc 能取消弹窗。
- [ ] 背景 UI 在弹窗打开时不可点击。
- [ ] 危险操作的确认按钮文案明确说明后果。
- [ ] 点击取消不会执行操作。
