## 1. 设计目标与原则

### 1.1 玩家需求

玩家扮演土地公，需要在 **有限神力** 下：

- 俯瞰归潮镇，掌握 **时间、地点、人物、事件**
- 观察 **缘线（关系）** 与 **业线（命运）**
- 在早/午/晚时段之间做干预决策，并 **推动时间** 让世界演化

UI 的核心任务是：**把涌现叙事压缩成可读的「命运仪表盘」**，而不是传统 RPG 的角色扮演界面。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **观察优先于操作** | 默认态是「看」；干预是二次确认后的消耗行为 |
| **时间感始终可见** | 第 X 天、时段、本周剩余天数常驻顶栏 |
| **道德资源对称展示** | 善意 / 恶意并列，避免玩家忽视路线后果 |
| **地图为中枢** | 所有面板从地图进入，结束回到地图 |
| **文本可推进、可 Auto** | 事件面板支持点击推进与自动播放 |
| **AI 能力渐进解锁** | 自然语言查询、事件内 AI 输入为后期技能，MVP 可占位 |

### 1.3 视角定稿

- **俯视策略地图**，不跟随 NPC 移动（见 `work.md` Q8）
- 地点为可点击节点；事件以地图标记动态刷新

## 4. Screen A：游戏主界面（HUD + 主地图）

**草图**：`幻灯片1.PNG`

### 4.1 布局分区

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏：时间 │ 善意 恶意 神力 香火 │ 总地图 │ 选项设置        │
├──────┬──────────────────────────────────────────┬───────────┤
│ 左栏 │              中央主地图                   │ （可留空） │
│ 情境 │   地点节点 + 事件标记 + 缩放平移           │           │
│ 槽   │                                          │           │
├──────┴──────────────────────────────────────────┴───────────┤
│ 底栏：[人物][事件][缘线][业线] │ 上下文区 │ [推动时间]      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 顶栏（Top HUD）

| 元素 | 位置 | 内容 | 交互 | 数据来源 |
|------|------|------|------|----------|
| 日期时段 | 左 | `第 {day} 天` `{早/午/晚}` | 仅展示；可点击展开日历（后期） | `GameTimeService` |
| 周倒计时 | 左下 | `本周结束还有 {n} 天` | 仅展示 | `day % 7` |
| 善意 | 中 | `善意 {value}` | Hover _tooltip：本周增减来源 | `KarmaResource.Good` |
| 恶意 | 中 | `恶意 {value}` | 同上 | `KarmaResource.Evil` |
| 神力 | 中 | `神力 {value}` | Hover：今日已消耗 / 干预上限 | `DivinePower` |
| 香火 | 中 | `香火 {value}` | Hover：每日 -1 规则、距离 100 差值 | `Incense` |
| 总地图 | 右 | 按钮 | 打开全览 overlay，支持快速跳转 | — |
| 选项设置 | 右 | 按钮 | 打开设置菜单 | — |

**规则对齐**（来自策划案）：

- 一局 **60 游戏日**，每日 **3 时段**
- 阴德 / 阳德只增不减；香火每日 -1，初始 50，60 天后 ≥100 维持神位
- 草图写「恶意/善意」，与策划 **阴德/阳德** 命名需统一（见 §13）

### 4.3 左栏 · 情境信息槽

| 状态 | 展示内容 |
|------|----------|
| 默认 | 当前天气图标 + 简短描述（晴/雾/潮） |
| 特殊日 | 特殊背景插画（如庙会前夜、邪教集会预告） |
| 技能解锁后 | **自然语言查询**输入框（消耗神力） |

**交互**：

- 查询框：输入 → 确认 → 扣神力 → 返回文字线索（非全知剧透）
- MVP：显示占位文案「查询能力尚未解锁」

### 4.4 中央 · 主地图

| 需求 | 规格 |
|------|------|
| 交互 | 鼠标拖拽平移；滚轮缩放；点击地点打开 [Screen D] |
| 地点节点 | 见 §4.5 |
| 事件标记 | 动态刷新；优先级：强硬不幸 > 主线 > 支线 > 日常 |
| NPC 指示 | 可选：小头像/点数表示当前时段在场人数（后期） |

**地点列表（定稿）**：

| 区域 | 地点 |
|------|------|
| 外围 | 山林、沙滩、港口 |
| 公共 | 学校、诊所/医院、警局、寺庙·钟塔、住所 |
| 商业街簇 | 商业街（容器）、书店、广场、咖啡厅 |

商业街在草图中为 **聚合父节点**，子地点可展开或点击商业街后在内层切换。

### 4.5 底栏

| 元素 | 说明 |
|------|------|
| Tab：人物 / 事件 / 缘线 / 业线 | 切换底栏上下文区内容，见 §8 |
| 上下文区 | 随 Tab 与选中对象变化；默认展示当前选中 NPC 摘要 |
| 角色立绘缩略 | 选中 NPC 时显示头像/半身（草图：小美） |
| **推动时间** | 主 CTA；结束当前时段 → 触发模拟与事件结算 |

**推动时间 — 交互流程**：

1. 点击「推动时间」
2. 若本时段有 **未处理关键事件** → 二次确认：「仍有事件未查看，确定推进？」
3. 进入时段结算 → 可能弹出 [Screen C]
4. 更新地图事件标记、资源、NPC 状态

## 5. Screen B：人物面板

**草图**：`幻灯片2.PNG`

### 5.1 用途

让玩家深入了解 **单个 NPC** 的：简介、关系缘线、业线进度、今日行程、基础属性。

### 5.2 布局

```
┌──────────────────────────────────────────────────────────┐
│ 左：简介+状态 │ 中：今日出行（早/午/晚） │ 右：立绘+姓名   │
├──────────────────────────────────────────────────────────┤
│ 缘线区（关系对象 + 强度条） │ 业线区（主线/支线进度条）    │
├──────────────────────────────────────────────────────────┤
│ 当前状态：心智 / 体魄 / 魅力 / 信仰                        │
└──────────────────────────────────────────────────────────┘
```

### 5.3 模块规格

| 模块 | 内容 | 交互 |
|------|------|------|
| **简介** | 2–3 句身份 + 「最近的状态」动态摘要（AI/模板） | 点击展开完整 Agent Memory 摘要（后期） |
| **今日出行** | 早/午/晚 → 地点 + 事件名 | 点击时段 → 地图高亮该地点 |
| **缘线** | 关联 NPC 列表；颜色=关系性质；横条=强度 0–100 | 点击 NPC → 跳转其人物面板 |
| **业线** | 主线 + 支线列表；进度条 + 节点标签 | 点击业线 → 展开节点详情 |
| **立绘** | 角色插画（静态 MVP；Live2D 后期） | — |
| **当前状态** | 心智、体魄、魅力、信仰（数值） | Hover 解释影响范围 |

**缘线视觉编码**（对齐策划）：

| 颜色 | 关系 |
|------|------|
| 红 | 爱情 |
| 金 | 利益 |
| 蓝 | 友情 |
| 灰 | 陌生 |
| 黑 | 仇恨 |

**业线视觉编码**：

| 颜色 | 类型 |
|------|------|
| 金 | 核心主线 |
| 紫/蓝/绿 | 支线 1/2/3 |
| 白 | 机会事件（单次） |

### 5.4 从主界面的关系

- 在地图上点击 NPC 标记 → 打开人物面板（Modal 或底栏扩展）
- 底栏「人物」Tab + 已选中 NPC → 同内容
- 未选中 NPC 时：显示「请选择地图上的居民」空状态

## 6. Screen C：事件推进面板

**草图**：`幻灯片3.PNG`

### 6.1 用途

呈现 **关键事件** 的叙事演出、角色反应、干预结果与收益总结。是 Core Loop 中「观看 → 结算」的核心反馈界面。

### 6.2 布局

```
┌──────────────────────────────────────────────────────────┐
│ 左：参与角色立绘/头像 │ 中：小型动画区 │ 右：事件文本区    │
│      （可多人）      │   角色变化     │  事件名称         │
│                      │               │  字幕流水         │
│                      │               │  [AI输入框-后期]  │
├──────────────────────────────────────────────────────────┤
│                              [确定]  [Auto]              │
└──────────────────────────────────────────────────────────┘
```

### 6.3 状态机

```
[进入] → 展示事件名 + 参与角色
      → 字幕逐段输出（点击 / Auto 推进）
      → 播放小型动画 + 角色表情/状态变化
      → [可选] 玩家干预结果（命运硬币可视化）
      → 收益总结（香火/阴德/阳德/缘线/业线变化）
      → 点击确定 → 关闭，回到主地图
```

### 6.4 交互细则

| 元素 | 行为 |
|------|------|
| 字幕区 | 打字机效果；点击任意处跳下一段；Auto 开启后自动播放 |
| 小型动画 | MVP：立绘位移/淡入 + 简单粒子；后期 Spine/Lottie |
| 角色变化 | 列表展示：「林潮音 信仰 +1」「陈远舟 与林潮音 缘线 +5」 |
| 收益总结 | 表格化：资源 Δ、业线进度 Δ、解锁提示 |
| 确定 | 仅在「收益总结」阶段可点（防误触跳过） |
| Auto | 切换自动/手动；记忆玩家偏好 |

### 6.5 AI 原生输入框（后期技能）

草图标注：**「AI原生，输入东西进去。（后期技能）」**

| 项 | 规格 |
|----|------|
| 解锁 | 玩家能力 Lv3+ 或特定剧情节点 |
| 功能 | 在事件中途输入干预意图（如「让他说出真相」） |
| 限制 | 消耗神力；每日次数上限；结果受模板+LLM 约束 |
| MVP | UI 占位灰态，不可交互 |

## 7. Screen D：地点查询面板

**草图**：`幻灯片4.PNG`

### 7.1 用途

点击地图地点后，展示：**地点简介、实景图、今日早/午/晚有哪些人物与事件**。

### 7.2 布局

```
┌──────────────────────────────────────────────────────────┐
│ 左：简介 │ 右：地点插画/照片                               │
├──────────────────────────────────────────────────────────┤
│ 左：今日出行（早/午/晚 × 人物列表 + 事件） │ 右：待定扩展区 │
└──────────────────────────────────────────────────────────┘
```

### 7.3 模块规格

| 模块 | 内容 |
|------|------|
| **简介** | 地点 lore 1–2 段 + 当前特殊状态（如「庙会筹备中」） |
| **地点图** | 场景概念图/照片（草图示例为学校建筑） |
| **今日出行** | 三个时段，每段：人物列表（可点击）+ 事件摘要 |
| **待定区** | 预留：干预入口、地点专属行动、隐藏线索 |

### 7.4 交互

| 操作 | 结果 |
|------|------|
| 点击人物名 | 跳转 [Screen B] 人物面板 |
| 点击事件 | 若已发生 → [Screen C]；若预告 → 仅详情 |
| 关闭 | 回到主地图，保持缩放位置 |

### 7.5 与人物面板的差异

| 对比 | 人物面板 | 地点面板 |
|------|----------|----------|
| 视角 | 以 **人** 为中心 | 以 **地点** 为中心 |
| 行程 | 该 NPC 全天去向 | 该地点全天有哪些人 |
| 核心问题 | 「他是谁、命运如何」 | 「这里今天会发生什么」 |

## 8. 底栏模式切换（人物 / 事件 / 缘线 / 业线）

草图底栏 4 个 Tab，在主界面底部切换 **上下文区**，不 necessarily 全屏跳转。

| Tab | 上下文区内容 | MVP |
|-----|--------------|-----|
| **人物** | 选中 NPC 摘要 + 快捷打开完整人物面板 | ✅ |
| **事件** | 当前时段事件列表（按优先级排序） | ✅ |
| **缘线** | 全局或选中 NPC 的关系网络缩略图 | ⚠️ 简版列表 |
| **业线** | 全局或选中 NPC 的业线树缩略 | ⚠️ 简版列表 |

**缘线 / 业线完整可视化**（节点图、力导向布局）建议 Phase 2；MVP 用 **可滚动列表 + 颜色条** 降低实现成本。

## 9. 视觉与交互规范（草案）

### 9.1 氛围

- **基调**：海岛小镇、轻度神秘、土地公视角的「俯瞰命运」
- **参考**：低饱和地图 + 缘/业线高饱和强调色
- **避免**：过度 RPG 血条风；应保持「观测仪表盘」感

### 9.2 字体与层级

| 层级 | 用途 | 建议 |
|------|------|------|
| H1 | 屏幕标题（事件名、地点名） | 20–24px |
| H2 | 模块标题（今日出行、缘线） | 16–18px |
| Body | 简介、字幕 | 14–16px |
| Caption | 资源、属性数值 | 12–14px |

### 9.3 输入

| 平台 | 地图 | 面板 |
|------|------|------|
| PC | 左键点击；滚轮缩放；中键/右键拖拽 | Esc 关闭；Tab 切换底栏 |
| 手柄（后期） | 焦点导航 + 缩放键 | B 返回 |

### 9.4 反馈

- 可点击元素：Hover 高亮 + 轻微缩放
- 推动时间：按下后有 0.3s 加载态（「命运流转中…」）
- 资源变化：数字滚动 + 绿/红 Δ 提示

## 10. 数据绑定与系统接口

| UI 字段 | 系统来源 | 刷新时机 |
|---------|----------|----------|
| 第 X 天 / 时段 | `GameTimeService` | 推动时间后 |
| 善意/恶意（或阴德/阳德） | `KarmaResource` | 事件结算、周结算 |
| 神力 / 香火 | `PlayerDivineState` | 干预、日结 |
| 地点列表 / 坐标 | `LocationRegistry` | 静态 + 状态标记 |
| 地图事件标记 | `EventScheduler` | 每时段 |
| NPC 行程 | `NPCScheduleService` | 每时段 |
| 缘线 | `BondGraph` | 事件结算 |
| 业线进度 | `DestinyTrackService` | 节点触发 |
| 事件字幕 | `EventNarrativeProvider`（模板+LLM） | 事件触发 |
| 角色属性 | `NPCStats` | 事件/成长 |

## 11. 异常 / 空 / 加载状态

| 场景 | 表现 |
|------|------|
| 未选中 NPC | 底栏人物区：「点击地图上的居民查看命运」 |
| 当前时段无事件 | 事件 Tab：「本时段风平浪静」 |
| 地点无人物 | 地点面板该时段：「—」 |
| 神力不足干预 | 查询/干预按钮灰态 + Tooltip 说明 |
| 推动时间加载 | 全屏半透明 + -spinner + 文案 |
| LLM 生成失败 | 事件面板回退模板文本 + 「命运迷雾…」提示 |

## 12. MVP 范围与实现优先级

| 优先级 | 内容 |
|--------|------|
