# UI 规格：加载游戏 / 读档

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 0. 全局规范引用

本界面必须遵循：

- `ui-accessibility-guidelines.md`
- `ui-interaction-patterns.md`
- `ui-localization-guidelines.md`

---

## 1. 用途

加载游戏界面用于浏览已有存档，查看存档摘要，并选择一个存档进入游戏。

---

## 2. 入口与出口

| 类型 | 说明 |
|---|---|
| 入口 | 主菜单“加载游戏”、暂停菜单“加载游戏” |
| 成功出口 | 进入主地图 HUD |
| 取消出口 | 返回上一层 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 加载游戏                              │
├──────────────────────────────────────┤
│ [存档 01] 第12天 夜晚  香火63 神力4    │
│ [存档 02] 第3天  下午  香火48 神力2    │
│ [空槽 03]                              │
│                                      │
│ 右侧 / 下方：选中存档详情              │
│                                      │
│ [加载] [删除] [返回]                  │
└──────────────────────────────────────┘
```

---

## 4. 存档卡片字段

| 字段 | 示例 |
|---|---|
| 存档名 | 存档 01 |
| 进度 | 第 12 天 · 夜晚 |
| 资源 | 香火 63 / 神力 4 |
| 路线倾向 | 阳德偏高 / 阴德偏高 / 中立 |
| 最近保存时间 | 2026-06-10 21:30 |
| 缩略图 | 主地图或土地庙截图，可后置 |

---

## 5. 交互规则

- 点击存档卡片后显示详情。
- 点击加载后进入游戏。
- 点击删除存档必须二次确认。
- 空槽不可加载。
- 从暂停菜单进入读档时，加载其他存档前需要提示“当前未保存进度可能丢失”。

---

## 6. 状态与变体

| 状态 | 表现 |
|---|---|
| 无存档 | 显示空状态和“开始新游戏”快捷按钮 |
| 存档损坏 | 卡片标红，加载按钮禁用 |
| 加载中 | 显示“命运织线展开中……” |
| 加载失败 | 显示错误并停留当前界面 |
| 删除确认 | 通用确认弹窗 |

---

```text
LoadGameScreen (Control)

│   │   ├── SaveSlotScroll (ScrollContainer)
│   │   └── SaveDetailPanel
│   └── ButtonRow
│       ├── LoadButton
│       ├── DeleteButton
│       └── BackButton
└── ConfirmDialogLayer
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 存档列表 | SaveSystem | 展示卡片 |
| 存档摘要 | SaveMetadata | 显示详情 |
| 存档有效性 | SaveValidator | 控制加载状态 |
| 当前来源 | NavigationContext | 判断是否来自暂停菜单 |

---

## 9. 验收标准

- [ ] 主菜单和暂停菜单均可进入加载游戏界面。
- [ ] 有存档时显示存档卡片和摘要。
- [ ] 无存档时显示空状态。
- [ ] 空槽不可加载。
- [ ] 删除存档前必须二次确认。
- [ ] 加载成功后进入主地图 HUD。
- [ ] 存档损坏或加载失败时显示错误提示。
