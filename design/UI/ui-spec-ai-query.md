# UI 规格：自然语言查询 UI

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P2  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

自然语言查询 UI 用于让玩家消耗神力，向土地公的神性视角提出有限问题，获得非剧透式线索。该功能属于后置能力，不进入 MVP。

---

## 2. 入口

| 来源 | 触发 |
|---|---|
| 主地图左侧提示区 | 查询能力解锁后显示输入入口 |
| 人物面板 | 对当前人物提问 |
| 地点面板 | 对当前地点提问 |
| 事件推进面板 | 特定事件中解锁查询入口，P2 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 问询命运                              │
├──────────────────────────────────────┤
│ 当前神力：4                            │
│ 本次消耗：2                            │
│                                      │
│ 你想询问什么？                         │
│ [林潮音今晚为什么会去港口？________]    │
│                                      │
│ 提示：命运只会回应你已触及的线索。       │
│                                      │
│ [问询]                  [取消]        │
└──────────────────────────────────────┘
```

---

## 4. 组件清单

| 组件 | 内容 | 交互 |
|---|---|---|
| 标题 | 问询命运 | 无 |
| 神力显示 | 当前神力、本次消耗 | 无 |
| 输入框 | 玩家输入问题 | 文本输入 |
| 提示文案 | 防剧透规则说明 | 无 |
| 问询按钮 | 提交问题 | 神力足够且输入非空时可用 |
| 取消按钮 | 关闭面板 | 返回来源界面 |
| 回答区 | 查询结果 | 提交后显示 |

---

## 5. 交互规则

- 输入为空时，“问询”按钮不可用。
- 神力不足时，输入框可用但问询按钮灰化，并提示神力不足。
- 提交前必须确认神力消耗。
- 查询结果只能基于玩家已解锁信息、已观察事件和系统允许透露的模糊线索。
- 查询失败时返回预设兜底文案，不扣神力或按系统设计退还神力。
- 不允许直接询问最优解、隐藏结局条件或未来完整剧透。

---

## 6. 回答状态

| 状态 | 表现 |
|---|---|
| 查询中 | “香火烟雾正在聚拢……” |
| 成功 | 显示 1–3 句线索 |
| 信息不足 | “这条命运尚未显形。” |
| 神力不足 | “神力不足，无法问询。” |
| 生成失败 | “命运雾气散乱，只留下模糊回声。” |
| 内容不允许 | “这个问题触及尚未显现的命运。” |

---

## 7. Godot 实现建议

```text
AIQueryPanel (PanelContainer)
├── Header
├── CostBlock
├── QueryInput (LineEdit / TextEdit)
├── HintText
├── AnswerPanel
└── ButtonRow
    ├── SubmitButton
    └── CancelButton
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 当前神力 | PlayerResourceSystem | 控制提交状态 |
| 查询消耗 | QueryConfig | 显示成本 |
| 解锁状态 | ProgressionSystem | 控制入口可见性 |
| 已知线索 | KnowledgeSystem | 限制回答范围 |
| 查询上下文 | NavigationContext | 人物 / 地点 / 事件上下文 |
| 查询结果 | AIQueryService / 模板兜底 | 显示回答 |

---

## 9. 验收标准

- [ ] 查询能力未解锁时，入口不可见或灰化。
- [ ] 神力不足时无法提交查询。
- [ ] 输入为空时无法提交查询。
- [ ] 提交查询前必须确认神力消耗。
- [ ] 查询成功后显示非剧透式线索。
- [ ] 查询失败时显示兜底文案且不阻塞游戏。
- [ ] 查询结果不会直接暴露未解锁结局或最优解。
