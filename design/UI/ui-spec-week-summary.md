# UI 规格：周结算界面

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P0 简版 / P1 完整版  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

周结算界面用于在每 7 天结束时总结本周的资源变化、重要事件、人物命运进展和能力解锁。它是中期反馈节点，帮助玩家理解自己这一周的行为造成了什么影响。

---

## 2. 触发时机

| 触发 | 说明 |
|---|---|
| 第 7 / 14 / 21 / 28 / 35 / 42 / 49 / 56 天结束 | 进入周结算 |
| 第 60 天结束 | 不进入普通周结算，进入 60 天总评 |

---

---

## 4. 展示模块

| 模块 | 内容 |
|---|---|
| 周标题 | 第 X 周结算 |
| 资源变化 | 香火、阳德、阴德、神力变化 |
| 重大事件 | 本周影响较大的 3–5 个事件 |
| 人物命运变化 | 关键 NPC 业线进度变化 |
| 缘线变化 | 重要关系增减，可折叠 |
| 能力解锁 | 新干预能力、新查询能力等 |
| 下周提示 | 非剧透式趋势提醒，可选 |

---

## 5. 信息优先级

1. 香火是否安全
2. 阳德 / 阴德路线倾向
3. 是否有重大危险解除或恶化
4. 关键 NPC 命运变化
5. 新能力或新地点解锁

---

## 6. 交互规则

| 操作 | 结果 |
|---|---|
| 点击继续 | 返回主地图 HUD，进入下一时段 / 下一天 |
| 点击人物条目 | 打开人物面板，P1 |
| 点击事件条目 | 打开事件回顾，P2 |
| 点击详情展开 | 展开完整变化列表 |

规则：

- MVP 可只显示摘要，不要求可点击回顾。
- 周结算期间不可打开暂停保存，避免结算状态不一致；可在结算结束后自动保存。
- 若本周无重大事件，显示“这一周归潮镇暂时平静”。

---

## 7. 状态与变体

| 状态 | 表现 |
|---|---|
| 普通周结算 | 标准资源与事件摘要 |
| 路线偏阳德 | 金色 / 暖色强调 |
| 路线偏阴德 | 紫色 / 暗红强调 |
| 香火危险 | 香火数值红色警告 |
| 能力解锁 | 显示新能力卡片 |
| 无重大事件 | 显示平静文案 |

## 9. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 当前周数 | TimeSystem | 标题 |
| 本周资源变化 | WeeklySummarySystem | 资源摘要 |
| 本周重大事件 | EventHistorySystem | 事件列表 |
| 人物业线变化 | DestinySystem | 命运变化 |
| 缘线变化 | BondSystem | 关系变化 |
| 解锁内容 | ProgressionSystem | 能力解锁 |

---

## 10. 验收标准

- [ ] 每 7 天结束后能进入周结算界面。
- [ ] 周结算正确显示本周资源变化。
- [ ] 周结算至少显示本周重大事件或平静空状态。
- [ ] 关键 NPC 业线变化能被列出。
- [ ] 香火危险时有明显警告。
- [ ] 点击继续后返回主地图并进入下一周期。
