# 《命运的织线》UI 初步设计文档 — Godot 像素风版本

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 引擎方向：Godot  
> 美术方向：简约像素风，整体气质参考温暖小镇生活类像素游戏  
> 目标：在低成本前提下，建立可实现、可扩展、可读性强的 UI 框架

---

## 1. UI 总体目标

《命运的织线》的 UI 不应做成复杂的高成本动态图形界面，而应服务于三个核心目标：

1. 让玩家清楚掌握当前小镇状态  
   包括时间、资源、地点、人物、事件、关系和命运进度。

2. 让玩家以“土地公 / 观察者”的身份俯瞰小镇  
   默认体验是观察、判断、轻微干预，而不是直接操控角色。

3. 用低成本像素风表达温暖小镇与命运压力  
   UI 应简洁、稳定、易读，避免复杂 3D、复杂 Live2D 或高成本动态演出。

---

## 2. 视觉方向

### 2.1 基调

- 主体风格：2D 像素风
- 氛围：温暖小镇 + 轻微神秘 + 命运压力
- 画面倾向：简约、清晰、柔和，不追求华丽复杂
- 推荐表现：
  - 像素边框面板
  - 木牌、纸张、布卷、庙宇纹样等轻量装饰
  - 小图标表达资源和状态
  - 少量颜色区分关系与事件优先级

### 2.2 UI 材质语言

| UI 类型 | 建议表现 |
|---|---|
| 主 HUD | 半透明暗色像素面板或木质横条 |
| 地点按钮 | 小镇地图上的像素建筑图标 |
| 人物信息 | 像素头像 + 简短文字卡片 |
| 事件提示 | 小旗帜、感叹号、铃铛、香火烟雾 |
| 缘线 | 简化为彩色线条或列表色条 |
| 业线 | 简化为进度条、节点列表或小型树状图 |
| 结算变化 | 数字跳动 + 小图标变化 |

### 2.3 色彩建议

| 用途 | 颜色方向 |
|---|---|
| 普通 UI 底色 | 暖棕、米白、深蓝灰 |
| 阳德 / 善向 | 金色、暖黄、浅绿 |
| 阴德 / 恶向 | 紫色、暗红、深灰 |
| 神力 | 蓝紫色或青蓝色 |
| 香火 | 橙色、红橙色 |
| 普通事件 | 白色 / 浅黄标记 |
| 危险事件 | 红色 / 深紫标记 |
| 已完成内容 | 灰化或低饱和 |

## 3. 信息架构

游戏 UI 以“主地图”为中心，其他界面都围绕主地图展开。

```text
主地图 HUD
├── 顶栏：时间、资源、设置
├── 中央：归潮镇地图
├── 左侧：当前情境 / 天气 / 提示
├── 底栏：人物、事件、缘线、业线
│
├── 人物面板
├── 地点面板
├── 事件推进面板
├── 缘线视图
├── 业线视图
└── 设置界面
```

设计原则：

- 玩家大多数时间停留在主地图。
- 其他面板尽量以弹窗、侧栏或底栏展开，不频繁切换全屏。
- MVP 阶段优先保证信息读得懂，而不是追求复杂可视化。
- 缘线、业线先用列表和进度条实现，后续再升级为网络图。

## 4. 核心界面清单

### 4.1 主界面 HUD

用途：玩家观察小镇、选择地点、查看事件、推动时间。

核心元素：

| 区域 | 内容 |
|---|---|
| 顶栏 | 第几天、早/午/晚、香火、神力、阳德、阴德、设置 |
| 中央 | 像素风小镇地图 |
| 左侧 | 天气、当前提示、特殊日提醒 |
| 底栏 | 人物 / 事件 / 缘线 / 业线 Tab |
| 右下或底栏 | “推动时间”主按钮 |

MVP 表现：

- 地图可先做成固定 2D 背景图。
- 地点是可点击按钮节点。
- 事件用小图标贴在地点旁。
- 不强制实现地图缩放和平移，除非地图尺寸确实需要。

---

### 4.2 人物面板

用途：查看单个 NPC 的身份、状态、行程、关系和业线。

核心元素：

| 模块 | 内容 |
|---|---|
| 人物头像 | 像素头像或半身像 |
| 基础信息 | 姓名、身份、简介 |
| 今日行程 | 早 / 午 / 晚所在地点 |
| 当前状态 | 心智、体魄、魅力、信仰等 |
| 缘线 | 与其他 NPC 的关系列表 |
| 业线 | 主业线与支线进度 |
| 操作 | 返回、定位到地图、查看相关事件 |

低成本建议：

- 不做复杂关系网。
- 缘线以“对象 + 颜色条 + 数值/等级”展示。
- 业线以“进度条 + 当前节点描述”展示。

---

### 4.3 地点面板

用途：查看地点信息，以及该地点今天会出现的人物和事件。

核心元素：

| 模块 | 内容 |
|---|---|
| 地点名 | 如学校、寺庙、港口、商业街 |
| 地点图 | 像素小场景图 |
| 简介 | 1–2 段地点说明 |
| 今日人物 | 早 / 午 / 晚人物列表 |
| 今日事件 | 当前或预告事件 |
| 操作 | 查看人物、查看事件、返回地图 |

MVP 表现：

- 一个地点一张静态像素图。
- 人物列表用头像小图标 + 名字。
- 事件用简短标题，不展开长文。

---

### 4.4 事件推进面板

用途：呈现事件文本、角色反应、干预结果和结算反馈。

核心元素：

| 模块 | 内容 |
|---|---|
| 事件标题 | 当前事件名 |
| 参与角色 | 头像列表 |
| 事件文本 | 分段文本，可点击推进 |
| 小演出区 | 简单立绘 / 图标 / 像素动画 |
| 选择或干预区 | 可选干预按钮 |
| 结算区 | 香火、神力、阳德、阴德、缘线、业线变化 |
| 操作 | 继续、Auto、确认 |

低成本建议：

- 事件演出采用视觉小说式布局。
- 不做复杂动画。
- 用头像表情切换、抖动、淡入淡出、图标飘字表达反馈。
- Godot 中可用 `RichTextLabel` 做逐字显示。

---

### 4.5 缘线视图

用途：查看人物之间的关系状态。

MVP 版本：

```text
[林潮音]
  - 陈远舟：友情 Lv.3
  - 顾沉舟：利益 Lv.2
  - 小美：陌生 Lv.1
```

视觉规则：

| 关系 | 颜色 |
|---|---|
| 爱情 | 红 / 粉 |
| 友情 | 蓝 |
| 利益 | 金 |
| 陌生 | 灰 |
| 仇恨 | 黑 / 暗红 |

后续升级：

- 可做成简化节点图。
- 但不建议 MVP 一开始实现复杂力导向关系网。

---

### 4.6 业线视图

用途：查看人物命运线、主线进度、支线进度和关键节点。

MVP 版本：

```text
林潮音 — 主业线：寻找自我
[■■■■□□] 65%

当前节点：
- 已触发：与陈远舟争吵
- 当前目标：重新审视自己的选择
- 潜在风险：夜晚前往港口
```

视觉规则：

- 主业线：金色
- 支线：蓝 / 绿 / 紫
- 危险节点：红色
- 已完成节点：灰化或打勾

---

## 5. Godot UI 实现建议

### 5.1 推荐节点结构

主界面可采用：

```text
MainGameUI (Control)
├── TopBar (HBoxContainer)
├── MapArea (Control / TextureRect)
│   ├── LocationButtons
│   └── EventMarkers
├── LeftInfoPanel (PanelContainer)
├── BottomPanel (PanelContainer)
│   ├── TabBar
│   └── ContextContent
├── TimeAdvanceButton (Button)
└── ModalLayer (CanvasLayer)
    ├── CharacterPanel
    ├── LocationPanel
    └── EventPanel
```

### 5.2 Godot 控件建议

| 功能 | Godot 控件 |
|---|---|
| 顶栏 | `HBoxContainer`, `Label`, `TextureRect` |
| 底栏 Tab | `TabContainer` 或自定义 `Button` 组 |
| 面板 | `PanelContainer` |
| 滚动列表 | `ScrollContainer` + `VBoxContainer` |
| 人物头像 | `TextureRect` |
| 事件文本 | `RichTextLabel` |
| 地图节点 | `TextureButton` |
| 弹窗层 | `CanvasLayer` |
| 简单动效 | `Tween` / `AnimationPlayer` |

### 5.3 低成本实现原则

- 优先使用 Godot 原生 Control UI。
- UI 图形尽量复用九宫格像素边框。
- 避免每个界面单独做复杂特效。
- 事件演出优先用文本、头像、图标、轻量 Tween。
- 缘线、业线先做列表，不做复杂图编辑器。
- 所有 UI 文案、资源数值、地点、人物数据尽量数据驱动。

## 6. UI 交互规范

### 6.1 输入方式

当前建议以 PC 键鼠为主：

| 操作 | 输入 |
|---|---|
| 点击地点 | 鼠标左键 |
| 打开人物 | 点击人物头像或名字 |
| 切换底栏 Tab | 鼠标点击 / 数字快捷键可选 |
| 关闭面板 | Esc / 关闭按钮 |
| 推进文本 | 鼠标点击 / Space |
| 推动时间 | 点击主按钮 |
| 滚动列表 | 鼠标滚轮 |

手柄支持可后置，不作为 MVP 必须项。

### 6.2 反馈规范

| 行为 | 反馈 |
|---|---|
| Hover 可点击元素 | 轻微高亮 / 边框变亮 |
| 点击按钮 | 按钮下压 1–2 像素 |
| 资源变化 | 数字跳动 + `+/-` 飘字 |
| 事件出现 | 地点旁出现小图标并轻微闪烁 |
| 危险事件 | 红色图标 + 轻微脉冲 |
| 神力不足 | 按钮灰化 + 提示文字 |
| 推动时间 | 短暂遮罩：“命运流转中……” |

## 7. 状态设计

| 状态 | 表现 |
|---|---|
| 无选中人物 | “请选择一位居民查看命运” |
| 当前无事件 | “此刻风平浪静” |
| 地点无人 | 对应时段显示 “无人停留” |
| 神力不足 | 干预按钮不可点击 |
| 数据加载中 | 简短 loading 文案 |
| 事件已完成 | 图标灰化或打勾 |
| 危险事件未处理 | 推动时间前二次确认 |
| 结算失败 / AI 失败 | 使用预设模板文本兜底 |

---

## 8. MVP UI 范围

第一阶段建议只做以下内容：

### P0 必做

- 主地图 HUD
- 顶栏时间与资源显示
- 地点按钮
- 事件标记
- 推动时间按钮
- 人物面板
- 地点面板
- 事件推进面板
- 简版缘线列表
- 简版业线进度

### P1 可做

- 底栏事件列表
- Auto 文本播放
- 总地图缩略图
- 资源变化飘字
- 危险事件提醒

### P2 后置

- 复杂缘线网络图
- 复杂业线树
- 自然语言查询 UI
- 事件内 AI 输入框
- 高级动画演出
- 手柄完整适配

## 9. 现有 UI 草案审阅结论

现有 `design/UI/ui-spec.md` 的优点：

- 主界面、人物、事件、地点四个核心界面已经明确。
- 信息架构比较完整。
- 已考虑时间、资源、缘线、业线、事件结算等核心系统。
- 有异常状态和验收标准雏形。
- 草图与文档对应关系清楚。

需要调整的地方：

1. 引擎方向需从 Unity 改为 Godot。
2. 视觉方向需从偏“命运仪表盘”改为“像素小镇 + 神明观察”。
3. 缘线 / 业线可视化需要降低 MVP 实现复杂度。
4. AI 输入框、自然语言查询、Live2D、复杂动画应后置。
5. 主地图交互不必一开始实现复杂缩放平移。
6. 资源命名需统一，建议对外使用“阳德 / 阴德 / 神力 / 香火”。
7. UI 文档后续应拆分为：
   - 主 HUD 设计
   - 人物面板设计
   - 地点面板设计
   - 事件面板设计
   - UI 组件规范
   - 像素风视觉规范

## 10. 初步验收标准

- [ ] 玩家进入游戏后能在主界面看到当前天数、时段和核心资源。
- [ ] 玩家能点击地点查看该地点的人物和事件。
- [ ] 玩家能点击人物查看简介、状态、行程、缘线和业线。
- [ ] 玩家能点击事件进入事件推进面板并阅读事件文本。
- [ ] 事件结束后能展示资源、关系或业线变化。
- [ ] 推动时间后，时段、事件和人物位置会刷新。
- [ ] 所有可点击元素都有 hover 或点击反馈。
- [ ] 神力不足或无事件等异常状态有明确提示。
- [ ] MVP 不依赖复杂动画、复杂关系网或 AI 输入框即可运行。
