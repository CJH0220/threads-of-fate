# UI 规格：结局回顾 / 图鉴

> 文档状态：初稿  
> 更新日期：2026-06-11  
> 所属阶段：P2  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

结局回顾 / 图鉴用于记录玩家已解锁的最终结局、角色结局和重要事件回顾。它提供长期收集目标，也方便玩家复盘不同路线。

---

## 2. 入口

| 来源 | 触发 |
|---|---|
| 最终结局界面 | 点击“结局回顾” |
| 主菜单 | 后续可加入“回顾”入口，P2 |
| 60 天总评 | 点击角色结局或事件回顾 |

---

## 3. 布局规格

```text
┌──────────────────────────────────────┐
│ 结局回顾                       [返回] │
├──────────────────────────────────────┤
│ 分类：[最终结局] [角色结局] [重大事件] │
│                                      │
│ [已解锁] 天界善神结局                 │
│ [已解锁] 中性结局                     │
│ [未知]   ？？？？                     │
│                                      │
│ 右侧 / 下方详情：                     │
│ 结局插图、解锁条件提示、回顾文本        │
└──────────────────────────────────────┘
```

---

## 4. 分类

| 分类 | 内容 |
|---|---|
| 最终结局 | 失败、邪神、正神、中性、传奇等 |
| 角色结局 | 各 NPC 的好 / 坏 / 中性 / 特殊结局 |
| 重大事件 | 已经历的关键事件回顾，P2 |
| 成就式记录 | 可后置，不作为 MVP |

---

## 5. 条目状态

| 状态 | 表现 |
|---|---|
| 已解锁 | 显示标题、缩略图、回顾文本 |
| 未解锁 | 显示问号、模糊剪影 |
| 部分解锁 | 显示标题但隐藏条件细节 |
| 新解锁 | 显示“新”标记 |

未解锁条目不应剧透完整条件，只能给模糊提示。

---

## 6. 交互规则

| 操作 | 结果 |
|---|---|
| 切换分类 | 显示对应条目 |
| 点击已解锁条目 | 展示详情 |
| 点击未解锁条目 | 显示模糊提示 |
| 返回 | 回到来源界面 |

示例模糊提示：

```text
这条命运仍未显现。
也许需要在善与恶之间走得更远。
```

---

## 8. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 已解锁最终结局 | GallerySystem | 显示最终结局条目 |
| 已解锁角色结局 | GallerySystem | 显示角色结局 |
| 已经历事件 | EventHistoryArchive | 显示重大事件 |
| 未解锁提示 | GalleryConfig | 模糊提示 |
| 缩略图 / 插图 | AssetDatabase | 显示图鉴图像 |

---

## 9. 验收标准

- [ ] 最终结局界面可进入结局回顾。
- [ ] 结局回顾能按最终结局、角色结局、重大事件分类。
- [ ] 已解锁条目显示标题和详情。
- [ ] 未解锁条目不剧透完整条件。
- [ ] 新解锁条目有明显标记。
- [ ] 返回按钮能回到来源界面。
