# UI 组件规格：危险事件警告

> 文档状态：初稿  
> 更新日期：2026-06-10  
> 所属阶段：P0  
> 引擎方向：Godot  
> 美术方向：简约像素风

---

## 1. 用途

危险事件警告用于提示玩家即将错过重大事件、触发强硬不幸结局，或进入不可逆风险状态。它必须足够醒目，但不能剧透完整结果。

---

## 2. 使用场景

| 场景 | 示例 |
|---|---|
| 未查看危险事件时推动时间 | “仍有危险征兆尚未查看” |
| 强硬不幸结局逼近 | “寺庙的香火出现异常” |
| 关键 NPC 进入高风险状态 | “林潮音今晚可能遭遇危险” |
| 不可逆操作 | “这次干预可能改变多人命运” |

---

## 3. 表现形式

危险警告有三种层级：

| 层级 | 形式 | 用途 |
|---|---|---|
| 轻度 | HUD 图标 / Toast | 普通风险提示 |
| 中度 | 左侧提示区红色条目 | 当前时段重要风险 |
| 高度 | 确认弹窗 / 强警告面板 | 推动时间、不可逆操作前 |

---

## 4. 视觉规格

- 主色：暗红、橙红、深紫。
- 图标：感叹号、裂纹香炉、暗色铃铛、红色旗帜。
- 动效：轻微脉冲；减少动画开启时改为静态高亮。
- 文案必须明确但不剧透。

示例文案：

```text
仍有危险征兆尚未查看。
如果继续推动时间，可能错过干预机会。
```

---

## 5. 交互规则

- 点击 HUD 危险提示应定位到相关地点、人物或事件。
- 推动时间前的高度警告必须要求玩家确认。
- 危险提示不应无限闪烁，避免疲劳。
- 已查看风险后，提示可降级但不完全消失，直到风险解除。

---

## 6. Godot 实现建议

```text
DangerWarningItem (PanelContainer)
├── WarningIcon
├── WarningText
└── OptionalActionButton
```

高度警告复用 `ConfirmDialog`，但使用危险样式。

---

## 7. 数据需求

| 数据 | 来源 | 用途 |
|---|---|---|
| 风险等级 | RiskSystem | 决定显示层级 |
| 关联目标 | Event / NPC / Location | 点击定位 |
| 是否已查看 | KnowledgeSystem | 控制提示降级 |
| 是否不可逆 | EventRiskConfig | 决定是否强确认 |

---

## 8. 验收标准

- [ ] 存在危险事件时，HUD 能显示危险提示。
- [ ] 点击危险提示能定位到相关事件或地点。
- [ ] 未查看危险事件时推动时间会出现确认警告。
- [ ] 危险提示文案不直接剧透结果。
- [ ] 减少动画开启时不使用脉冲闪烁。
- [ ] 风险解除后提示消失或降级。
