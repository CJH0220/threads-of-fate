# 游戏概念：《命运的织线》

> 版本：v2.0  
> 更新日期：2026-06-12  
> 类型：2D 像素风命运干预 / 小镇叙事模拟  
> 引擎方向：Godot  
> 当前状态：概念与系统策划阶段

---

## 1. 概述

《命运的织线》是一款以土地公视角展开的 60 天小镇命运干预游戏。玩家扮演归潮镇的土地公，被岛民称为“岛爷”，在天界绩效压力和地府诱惑之间，通过一台“命运织线器”观察居民的缘线、业线与关键事件，并用有限神力间接影响小镇命运。

玩家不能直接控制人类，也不能改写已经发生的过去，只能通过托梦、赐福、牵引机缘、遮蔽灾兆、显灵警示等方式，在关键节点轻轻拨动命运。60 天后，归潮镇将迎来灾难与结算，玩家此前的所有干预都会汇聚成最终路线。

---

## 2. 玩家幻想

> **“我不能替他们活，但我可以在命运拐弯前推一把。”**

玩家体验的不是全能神，而是一位香火衰微、能力有限、还有绩效压力的小神。你能看见凡人看不见的线索，但不能粗暴命令人类。你每次干预都像在命运线上轻轻打一个结：有时救人，有时误导，有时只是让一个人多说一句话。

核心体验包括：

- 观察居民生活，理解他们的欲望、压力和关系。
- 在有限资源下选择是否干预、干预谁、干预到什么程度。
- 看见小选择如何累积成大后果。
- 在正神、邪神、中性、传奇或失败路线之间形成自己的神明道路。

## 3. 世界观基准

### 3.1 归潮镇

归潮镇是一座海岛小镇，居民不多，靠港口、海鲜铺、学校、寺庙、酒屋、诊所和商业街维持日常生活。镇上仍保留土地公信仰，但现代生活让香火逐年减少。

小镇表面平静，内部却积累了家庭矛盾、升学压力、寺庙腐败、邪教渗透、外来者调查和即将到来的天灾。

### 3.2 土地公 / 岛爷

玩家是归潮镇土地公。岛民私下称其为“岛爷”。他不是威严的上神，而是一个有些懒散、会吐槽、但仍与这片土地有关系的小神。

当前困境：

- 香火下降，天界准备撤回神格。
- 天界给出 60 天最后考核。
- 地府趁机诱惑玩家走向阴德路线。
- 归潮镇即将发生大难。

### 3.3 命运织线器

命运织线器是天界试点设备，也是玩家的核心 UI 与系统入口。

它可以显示：

- 居民基础信息
- 今日行程
- 缘线关系
- 业线进度
- 关键事件
- 风险征兆
- 干预入口
- 历史记录

## 4. 核心玩法循环

```text
观察命运
→ 查看人物 / 地点 / 事件
→ 选择是否使用神力干预
→ 推动时间
→ 系统结算 NPC 行为与事件结果
→ 更新资源、缘线、业线和历史记录
→ 周结算 / 终局结算
```

玩家每个时段主要做三件事：

1. 看见当前有什么人、地点、事件值得关注。
2. 判断是否消耗神力进行间接干预。
3. 推动时间并承受结果。

## 5. 核心系统

### 5.1 时间系统

- 总时长：60 天
- 每天时段：早 / 午 / 晚
- 每 7 天进行周结算
- 第 60 天进入终局结算

### 5.2 资源系统

| 资源 | 用途 |
|---|---|
| 香火 | 神位与表层胜负指标，过低会失败 |
| 神力 | 用于观察强化和干预行为 |
| 阳德 | 正向干预、救助、守护积累 |
| 阴德 | 操控、遮蔽、牺牲、邪路行为积累 |

### 5.3 干预系统

玩家不能直接控制 NPC，只能使用有限神力影响环境、梦境、机缘或信息可见性。

示例干预：

- 托梦
- 赐福
- 牵引机缘
- 遮蔽灾兆
- 显灵警示

### 5.4 NPC 与事件系统

NPC 拥有：

- 基础性格
- 当前状态
- 日程地点
- 缘线关系
- 业线进度
- 关键标签
- 事件参与条件

事件由规则触发，AI 文本只在边界内润色描述，不负责改变结算结果。

### 5.5 结局系统

可能路线包括：

- 正神路线
- 邪神路线
- 中性路线
- 传奇路线
- 失败 / Game Over

结局由香火、阳德、阴德、关键 NPC 命运、邪教状态、天灾应对、证据链等共同决定。

## 7. 美术与 UI 方向

### 7.1 美术方向

- 简约 2D 像素风
- 温暖小镇与民俗神明结合
- 土地庙、香炉、海雾、港口、学校、酒屋等场景反复使用
- 克制表达邪教、天灾和失败，不做跳脸恐怖

### 7.2 UI 方向

UI 以命运织线器为概念外壳，但实现上保持低成本：

- 主地图 HUD
- 人物面板
- 地点面板
- 事件面板
- 缘线 / 业线列表
- 时间推进与结算
- 周总结与终局总结

## 8. 技术方向

| 项目 | 当前方向 |
|---|---|
| 引擎 | Godot |
| 类型 | 2D 像素风叙事模拟 |
| UI | Godot Control 系列节点 |
| 数据 | 本地 JSON / CSV / Resource 优先 |

MVP 阶段不依赖复杂数据库、实时大模型推演、复杂命运图渲染或高成本动态演出。

---

## 9. MVP 范围

MVP 应优先验证：

- 60 天时间推进是否成立
- 香火、神力、阳德、阴德是否能形成路线压力
- 至少 6 名核心 NPC 的事件与结局是否可玩
- 事件触发、干预、结算、历史记录是否闭环
- UI 是否能支撑主地图、人物、地点、事件和时间推进
- 第 60 天总评与至少一版结局是否可展示

MVP 不优先做：

- 复杂关系网动画
- 高自由自然语言查询
- 全量 14 人复杂 Agent 推演
- 多周目大型分支树
- 高成本过场动画
